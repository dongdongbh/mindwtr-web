# 匯入 TickTick

如水可匯入 TickTick 備份，讓你不必手動重建系統即可遷移。

支援的來源：

- TickTick **CSV** 備份
- 含 CSV 匯出檔的 TickTick **ZIP** 備份

桌面版與行動版都可從**設定 -> 資料 -> 從 TickTick 匯入**執行。

## 從 TickTick 匯出

TickTick 透過網頁版建立備份：

1. 在瀏覽器中開啟 TickTick 並登入。
2. 按一下左上角的大頭貼。
3. 開啟**設定 -> 帳號 -> 備份與匯入**。
4. 選擇**產生備份**。
5. 儲存 TickTick 下載的 CSV 或 ZIP 檔案。

接著在如水中開啟**從 TickTick 匯入**並選擇該檔案。

## 匯入內容

如水以 GTD 優先的原則，將 TickTick 備份對應至如水的模型：

- TickTick 資料夾成為**如水領域**
- TickTick 清單成為**如水專案**
- 父任務與子資料列可成為**檢查清單任務**
- 檢查清單內容保留為檢查清單項目
- TickTick 標籤成為**如水標籤**
- 若有優先順序則加以對應
- 能安全讀取時，保留截止日期、開始日期、全天日期與時區
- 受支援的重複規則會匯入為如水的重複規則
- 若資料含已完成與已歸檔狀態，則加以保留
- 任務筆記／內容保留在描述中

匯入的使用中任務會保留供 GTD 處理，不會被迫套用如水無法從備份判斷的狀態。

## 匯入流程

1. 開啟**從 TickTick 匯入**
2. 選擇 TickTick CSV 或 ZIP 備份
3. 檢查預覽摘要
4. 確認匯入

在支援的環境中，如水會在匯入前保存目前本機資料的復原快照。

匯入後：

- 視需要建立新領域與專案
- 從備份加入任務、檢查清單、標籤、日期與重複規則
- 略過的資料列、不支援的封存檔項目或無法安全對應的欄位會顯示警告

## TickTick 備份注意事項

TickTick 備份會在真正的 CSV 標頭前放置中繼資料列。如水會自動偵測真正的標頭，不必先編輯檔案。

如水會略過：

- 巢狀 ZIP 檔案
- 封存檔中的非 CSV 檔案
- 無法安全解析的格式錯誤資料列

TickTick 備份不含應用程式的所有狀態。附件與部分 TickTick 專屬的呈現細節可能需要手動搬移。

## 提示

- 確認匯入結果前請保留原始 TickTick 備份
- 可從完整備份開始，但請利用預覽確認對應結果是否正確
- 重複匯入同一備份可能產生重複任務
- 若其他應用程式需要替代遷移方式，請參閱[從其他應用程式匯入資料](/zh-Hant/import/)

另請參閱[資料與同步](/zh-Hant/data-sync/)及[備份與還原](/zh-Hant/data-sync/backup-restore)。
