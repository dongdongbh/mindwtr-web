# Contexts and tags

Contexts and tags help you choose work without scanning every project.

## Contexts

Contexts describe the condition needed to do an action: place, tool, person, energy, or mode.

Examples:

- `@home`
- `@office`
- `@errands`
- `@phone`
- `@low-energy`
- `@deep-work`

Use contexts when the label changes whether the action is available right now.

## Tags

Tags are looser labels for grouping, searching, and filtering.

Examples:

- `#finance`
- `#travel`
- `#client`
- `#writing`

Use tags when the label helps retrieval but does not define action availability.

## Contexts vs areas

An area is a responsibility you maintain over time, such as Health, Family, Work, or Home. A context is a condition for doing a specific action. A contractor task might live in a Work area, belong to a Renovation project, and use an `@phone` context.

## Practical guidance

Start with a small set. Add a context only when it changes a real decision in Focus or Next Actions. Use tags for search and lightweight grouping.
