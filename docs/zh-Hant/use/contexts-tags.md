# 情境與標籤

情境會根據你所在的位置、手邊的工具或所需的心理狀態，協助你看見**現在**能做的事情。

---

## 什麼是情境？

在 GTD 中，情境是完成任務所需的條件，通常是地點、工具或人物。

依情境篩選時，只會看見你在目前處境下真正能做的任務。

## 情境與領域的差異

這是剛接觸 GTD 時最容易混淆的概念之一。

| 概念       | 回答的問題                                           | 經常變動？ | 範例                          |
| ---------- | ---------------------------------------------------- | ---------- | ----------------------------- |
| **情境**   | 考量所在位置與手邊資源，我現在能做什麼？             | 是         | `@computer`、`@phone`、`@home` |
| **領域**   | 這屬於哪一項持續承擔的責任？                         | 否         | 工作、家庭、健康、財務        |

主要差異：

- **情境**關乎*執行限制*，會在一天之中改變。
- **領域**是*責任分類*，只要生活或工作的該部分存在，就會持續保留。

### 範例：承包商或客戶工作

如果你服務多位客戶，為每位客戶各設一個**領域**完全合理。

- **領域：**客戶 A
- **專案：**後端遷移
- **任務：**檢查 API 規格
- **情境：**`@computer`

這能回答兩個不同的問題：

- 想查看某位客戶的所有相關事項時，依**領域**篩選。
- 想查看此刻能做的事情，而不區分客戶時，依**情境**篩選。

直接建立或編輯領域的方式，請參閱[領域與人員](/zh-Hant/use/areas-people)。

## 地點情境（@）

Mindwtr 內建以下地點情境：

| 情境        | 適用情況                               |
| ----------- | -------------------------------------- |
| `@home`     | 必須在家執行的任務                     |
| `@work`     | 辦公室／工作場所的任務                 |
| `@errands`  | 外出時執行的任務（購物、約會）         |
| `@agendas`  | 會議或通話中要討論的事項               |
| `@computer` | 需要電腦的任務                         |
| `@phone`    | 需要打電話的任務                       |
| `@anywhere` | 沒有地點限制的任務                     |

### 範例

- `Call mom @phone`
- `Fix leaky faucet @home`
- `Buy office supplies @errands`
- `Deploy update @computer @work`

---

## 標籤（#）

Mindwtr 內建依心理狀態、工作模式或主題篩選的標籤：

| 標籤         | 適用情況                         |
| ------------ | -------------------------------- |
| `#focused`   | 需要專心投入的深度工作           |
| `#lowenergy` | 疲倦時仍能處理的簡單任務         |
| `#creative`  | 腦力激盪或構思                   |
| `#routine`   | 重複性／機械性的任務             |

### 範例

- `Write proposal #focused @computer`
- `File receipts #lowenergy @home`
- `Brainstorm campaign ideas #creative`
- `Process expenses #routine @computer`

### 為什麼使用標籤？

你的生產力會隨一天的時間而變化：
- **早上：**專注力高，處理 #focused 任務
- **午餐後：**精力較低，處理 #lowenergy 或 #routine 任務
- **創意時間：**有靈感時處理 #creative 任務

## 在 Mindwtr 中使用情境

### 加入情境

**快速新增語法：**
```
Task title @context1 @context2
Research topic #focused @computer
```
情境與標籤名稱可包含 Unicode 字母與數字（包括中日韓文字及帶重音字元）。

**編輯任務：**
1. 開啟任務編輯器
2. 在「情境」欄位加入情境（以逗號分隔）

### 依情境篩選

**桌面版：**
1. 前往**下一步行動**或**情境**檢視
2. 按一下情境方塊進行篩選

**行動版：**
1. 前往**下一步行動**分頁，或從導覽抽屜開啟**情境**
2. 點選情境進行篩選

### 依情境或標籤分組

任務清單工具列也包含**分組**控制項。掃描較長的收集箱、等待中、將來/也許、參考資料、已完成或下一步行動清單時，可依情境、領域、專案或標籤分組。

如果偏好以標籤為中心的工作流程，並以標籤作為主要分類層，依標籤分組會很實用。

## 自訂情境

你可以建立自己的情境：

- `@Bob`：要和 Bob 討論的事項
- `@waiting-on-vendor`：等待供應商回覆
- `@car`：要在車上處理的事情
- `@morning`：早晨例行事項

### 建立自訂情境

新增任務時直接輸入新的情境：
```
Review contract @legal
```

系統會加入該情境，之後即可用來篩選。

### 管理已儲存的情境與標籤

想清理或重新命名可重複使用的中繼資料時：

- 開啟**設定 → 管理**
- 在同一處編輯**情境**、**標籤**、**領域**與**人員**
- 可用來合併重複項目、統一命名，或移除不再使用的值

領域與人員也能在指派時建立。在**領域**選擇器或**指派給**欄位輸入內容；找不到完全相符的項目時，選擇**建立**或**新增人員**。

### 批次編輯情境

桌面版可一次更新多項任務的情境：

1. 在清單檢視進入**選取**模式
2. 選取要編輯的任務
3. 使用批次動作列中的**加入情境**或**移除情境**

在每週回顧或專案規劃後，想重新分類一批任務時，此功能很實用。

## 階層式情境與標籤

使用斜線標記法，以巢狀結構整理情境：

| 語法              | 任務範例                    |
| ----------------- | --------------------------- |
| `@work/meetings`  | Meeting prep @work/meetings |
| `@home/garage`    | Fix shelf @home/garage      |
| `#health/fitness` | Morning run #health/fitness |
| `#work/admin`     | File reports #work/admin    |

### 篩選父層時包含子層

依父層情境篩選時，會包含所有子層：

| 篩選條件  | 顯示內容                                     |
| --------- | -------------------------------------------- |
| `@work`   | `@work`、`@work/meetings`、`@work/calls`     |
| `#health` | `#health`、`#health/fitness`、`#health/diet` |

如此既能維持具體分類，也能從較高層級進行篩選。

### 優點

- **整理：**將相關情境歸在一起，避免雜亂
- **彈性：**可廣泛或精確地篩選
- **向後相容：**簡單情境仍會照常運作

## 情境最佳做法

### 從簡單開始

先從少數幾個情境開始：
- @home
- @work
- @errands
- @computer

需要時再增加。

### 保持一致

使用相同的拼法與格式：
- ✓ `@home`（一律如此）
- ✗ `@Home`、`@house`、`home`

### 組合情境

一項任務可有多個情境：
- `@computer @work`：需要在工作場所使用電腦
- `@phone @anywhere`：可在任何地方撥打的電話
- `#focused @home`：在家執行的深度工作

### 人物情境

為議程事項加入人物情境：
```
Discuss project timeline @Sarah
Ask about budget @manager
```

見到 Sarah 時，搜尋 `@Sarah` 即可找到所有相關事項。

## 情境工作流程

### 早晨規劃

1. 確認今天會在哪些地方
2. 依相關情境篩選「下一步行動」
3. 選擇專注任務

### 更換地點

移動至其他地點時：
1. 依新情境篩選（@work → @home）
2. 查看能在這裡完成的事情
3. 選擇下一項任務

### 意外互動

有人突然來電或到訪時：
1. 搜尋對方的姓名／情境
2. 檢視議程事項
3. 處理等待中項目

## 情境與標籤的差異

在 Mindwtr 中，`@contexts` 與 `#tags` 都能篩選任務，但回答的問題不同：

| 符號 | 回答的問題               | 一般用途                       | 範例                     |
| ---- | ------------------------ | ------------------------------ | ------------------------ |
| `@`  | 可在何處／用什麼完成？   | 地點、工具或人物限制           | @home、@work、@phone     |
| `#`  | 這件事關於什麼？         | 主題、精力、模式或標記         | #focused、#lowenergy     |

### 實用建議

- 以 **`@` 情境**作為*執行篩選條件*（所需的地點、工具或人物）。
- 以 **`#` 標籤**進行*分類*（精力程度、主題或跨專案分組）。
- 兩者都支援**階層**（例如 `@work/meetings`、`#health/fitness`）。
