# 使用指南：桌面版

Mindwtr 桌面版以 [Tauri v2](https://tauri.app/) 建置，在 Windows、macOS 與 Linux 上提供輕巧、快速的使用體驗。

## 概覽

桌面版以側邊欄導覽所有 GTD 檢視，並在主要內容區域顯示內容。

---

## 互動模式

- **按一下**切換任務詳細資料。
- 在任務上**按兩下**會開啟完整編輯模式。若只想就地重新命名標題，請從任務的 `⋯` 選單選擇**重新命名任務**（Enter 儲存，Esc 取消）。
- 任務編輯器開啟且尚未變更任何內容時，**按一下外部**即可關閉；欄位一經編輯，就必須使用「儲存」、「取消」或 Esc 關閉。
- **按一下右鍵**開啟快顯選單（專案、任務）。
- **鍵盤優先**：全域快速鍵與 Vim/Emacs 模式針對桌面工作流程最佳化。

若不斷將任務截止日期往後延，第三次延後後，任務列會在截止日期旁顯示小沙漏徽章與延後次數。這是在提醒你：任務可能需要重新釐清、委派或放棄，而不是再次延後。

大多數任務清單共用同一列控制項，用來選取任務、篩選、排序、分組、顯示詳細資料及切換緊湊密度。可使用**分組**，依情境、領域、專案或標籤等中繼資料整理收集箱、下一步行動、等待中、將來/也許、參考資料、已完成及其他清單檢視。

## 專注模式

使用「專注模式」隱藏側邊欄，並將清單置中（最大寬度 800px）。

- Windows/Linux 按 **Ctrl+\\**，macOS 按 **Cmd+\\** 切換
- 很適合深度工作或回顧

### 系統匣

Mindwtr 會在背景執行，以處理自動同步。

- **關閉視窗**會將應用程式縮至系統匣，而不是結束。
- **按一下系統匣圖示**顯示／隱藏視窗。
- **在系統匣圖示上按一下右鍵**，即可完全結束應用程式。

## 檢視

### 📥 收集箱

收集區域。所有新任務都會先進入這裡，再進行處理。

- 使用底部輸入欄**新增任務**
- 支援**快速新增語法**（請參閱[開始使用](/zh-Hant/start/getting-started)）
- 若新任務應直接進入**今日專注**，可使用「快速新增」輸入欄旁的星號。任務編輯器標題列也有相同星號。加星號視同完成釐清：加星號的收集項目會建立為**下一步行動**，而不是收集箱項目；將加星號任務移回收集箱時會移除星號。
- 使用釐清工作流程**處理收集箱**
- 想進行引導式收集而不是面對空白輸入欄時，**思緒清理**會依常見生活／工作領域逐步提示

### 🎯 專注

供每日執行使用的整合儀表板。「專注」不是所有 `next` 狀態任務的完整清單，而會將範圍縮小至目前可執行的工作。

- **今日專注**：加星號的優先任務，數量不超過設定的專注上限。
- **逾期**：已過截止日期的項目。
- **今日到期**：今天到期的任務。
- **下一步行動**：沒有截止日期、已可執行且依情境篩選的任務。
- **待回顧**：有提醒日期的項目。

「專注」會隱藏未來才開始的任務，以及循序專案中較後面的任務。需要查看更完整的任務清單時，請使用**情境**、**專案**或**搜尋**。

**預設下一步行動順序：**「專注」會將即將到期的行動排在最前，無日期行動居中，較遠的未來到期行動排在最後。同組內會依序使用已啟用的優先順序、開始時間、建立日期、標題與 id。完整邏輯請參閱[Mindwtr 的 GTD 工作流程](/zh-Hant/use/gtd-workflow#「專注」如何排序可用行動)。

**功能：**
- **情境篩選條件**：依情境（例如 @home、@work）或標籤篩選下一步行動。篩選選擇器會將情境與標籤分組，讓較長的中繼資料清單仍易於瀏覽。
- **依情境分組**：以清單分組控制項，依主要情境為下一步行動分組；沒有情境的任務會放在**無情境**。
- **已儲存篩選條件**：儲存可重複使用的專注條件，例如專案、情境、標籤、優先順序、精力程度與預估時間。
- **禪模式**：切換後隱藏「今日專注」任務以外的所有內容。
- **番茄鐘（選用）**：在**設定 → GTD → 功能 → 番茄鐘**中啟用，以顯示專注／休息計時面板（15/3、25/5、50/10，另加一組選用的自訂預設）。可保留為**只使用計時器**，或開啟**將計時器連結至任務**，顯示「計時任務」選擇器與**將任務標示為已完成**動作。

### 📁 專案

包含相關任務、需要多個步驟的成果。

- **循序模式**：只有專案中第一項可用任務會顯示在「專注」
- **平行模式**：所有可用專案任務都能顯示在「專注」
- **循序範圍**：循序專案可依整個專案或逐分區推進；分區範圍會顯示各分區中第一項可用任務。
- **狀態**：使用中、等待中、將來、已歸檔
- **專注領域**：依較高層級領域（例如工作、健康）為專案分組，讓側邊欄保持整齊。將專案拖曳至領域即可移動；拖曳期間，收合及空白領域（以及**無領域**）也會顯示為放置目標。
- **拖曳以移動任務**：從開啟專案的清單，將任務拖至側邊欄中的另一個專案即可移入（會排在目標現有任務之後）；拖至領域標題則成為領域的直接任務。所有排序模式皆適用；**已移至……**通知提供一鍵復原，且已歸檔專案不接受放置。
- **專案標籤**：為專案加上標籤並依標籤篩選
- **支援筆記**：加入規劃筆記與參考資料
- **分區**：專案內選用的任務群組（階段、里程碑、子工作流）。分區是單一專案內的標題，不是子任務或獨立專案。
- **回顧日期**：設定專案回顧提醒日期
- **歸檔**：完成專案即將它歸檔，剩餘任務也會一併完成；**重新啟用**可復原

任務編輯器的**專案分區**欄位會將任務指派至目前專案中的分區。只有任務已屬於設有分區的專案時，此欄位才有實際用途。

### 🏷️ 情境

依地點或工具情境篩選任務：

- `@home`、`@work`、`@errands`、`@agendas`
- `@computer`、`@phone`、`@anywhere`

### 🏷️ 標籤

依精力程度、模式或主題篩選任務：

- 精力：`#focused`、`#lowenergy`、`#creative`
- 主題：`#health`、`#finance`

### ⏳ 等待中

追蹤已委派的項目，或正在等待外部事件的任務。

### 💭 將來/也許

孵化日後可能想進行的想法。

### 🗓️ 行事曆

依時間顯示有截止日期或開始時間的任務。

在較寬的版面中，「行事曆」會為所選日期顯示**規劃下一步行動**面板。它會呈現尚未排程的下一步行動與已到期但未排程的任務，讓你不必離開「行事曆」即可將它們放入空閒時段。若想為日／週格線騰出更多空間，可收合面板。

### 📋 看板檢視

可拖放的看板式欄位：

- **收集箱**：尚未處理的項目
- **下一步行動**：已可執行
- **等待中**：已委派項目
- **將來/也許**：已延後項目
- **已完成**：完成的任務

在同一欄內拖曳卡片可設定手動看板順序。將任務移至其他狀態時，會清除舊欄順序，以便在新欄中重新排序。

以看板搜尋欄依標題、筆記、專案、情境或標籤縮小可見卡片。篩選按鈕會開啟情境、標籤、截止日期與專案的快速篩選；**清除篩選條件**會同時移除搜尋文字及使用中的篩選條件。

### ✅ 已完成

最近完成的任務。將此視為每日或每週回顧使用的短期完成記錄。

若實際完成時間早於標示時間，請按一下任務的**完成時間**修正。也可在任何已開啟任務的完成按鈕上按一下右鍵，將它標示為在較早時間完成。當完成時間會影響記錄或「完成後重複」排程時，此功能很實用。

### 📦 已歸檔

已收存的完成任務。已歸檔任務不會顯示在一般任務清單，但可以在此搜尋或還原。按一下任務的**完成時間**即可修正。

選擇**選取**以挑選任務，或使用**全選**。可將選取項目還原至收集箱，或移至垃圾桶。Mindwtr 只會從垃圾桶永久刪除任務。

### 🗑️ 垃圾桶

已刪除的任務與專案，以最新項目優先。可還原、永久刪除，或使用**清空垃圾桶**一次清除所有內容。選擇**選取**可挑選多項任務與專案（或使用**全選**），再一步完成還原或永久刪除。永久刪除一定會要求確認。

### 📝 每週回顧

引導式 GTD 回顧精靈包含以下步驟：

1. 處理收集箱
2. 回顧行事曆
3. 跟進等待中
4. 回顧專案
5. 回顧將來/也許

在回顧中使用**處理收集箱**動作，先清除收集項目，再繼續回顧行事曆、等待中、專案與將來項目。

詳細指南請參閱[每週回顧](/zh-Hant/use/weekly-review)。

## AI 助理（選用）

在**設定 → AI 助理**中啟用：

- **釐清**：將含糊任務變成具體下一步行動
- **拆分**：為大型任務產生檢查清單步驟
- **回顧分析**：在回顧期間標示停滯任務
- **Copilot**：輸入時提供情境／標籤／時間建議

AI 是選用功能，只會在你要求時執行。

## 可重複使用的清單

以檢查清單作為範本：

- **複製任務**：複製主清單（打包、行前準備）
- **重設檢查清單**：取消勾選所有內容以再次使用（採買）

## 任務編輯器（檢視 + 編輯）

- 按一下任務，開啟包含所有詳細資料的**唯讀檢視**。
- 按**編輯**（或 `e`）切換至編輯模式。
- 編輯器一開始保持精簡。使用**更多選項**顯示進階欄位。
- **專案分區**是設有分區之專案內的任務整理欄位。收集箱任務、未指派任務或沒有分區的專案請留空。
- 狀態、優先順序與精力程度使用膠囊控制項而非下拉選單，讓鍵盤及指標選取更快。
- 日期欄位提供**今天**、**明天**、**+3 天**、**下週**、**下個月**與**無日期**快速方塊。
- 在**設定 → GTD → 任務編輯器版面**中，選擇桌面編輯器要以行內側邊預覽，還是置中快顯方式開啟。

## 行事曆整合

Mindwtr 可在「行事曆」檢視疊加外部行事曆。macOS 可透過 EventKit 讀取 Apple 行事曆；所有桌面平台都能讀取直接 ICS 訂閱 URL。
需要 ICS 訂閱時，請在**設定 → 行事曆**加入 **ICS URL**。
外部事件為唯讀，但可在「行事曆」中從事件建立另一項 Mindwtr 任務；Mindwtr 會在可用時複製事件標題、日期／時間、地點、描述與行事曆名稱。
以「行事曆」中的**規劃下一步行動**將所選日期變成規劃介面：檢視尚未排程的工作、選擇任務並安排至空閒時段，同時讓截止日期維持期限用途。
在 macOS 上，**將任務推送至行事曆**也能將已排程／到期的 Mindwtr 任務寫入所選的可寫入 Apple 行事曆。設定詳情請參閱[行事曆整合](/zh-Hant/use/calendar-integration)。

### 🔍 全域搜尋

使用運算子快速找到任何內容。

**開啟：**按 `/`、`Ctrl/Cmd + K`，或按一下搜尋圖示。

#### 搜尋運算子

| 運算子      | 範例               | 說明                       |
| ----------- | ------------------ | -------------------------- |
| `status:`   | `status:next`      | 依任務狀態篩選             |
| `-status:`  | `-status:done`     | 排除狀態                   |
| `context:`  | `context:@home`    | 依情境篩選                 |
| `tag:`      | `tag:#focused`     | 依標籤篩選                 |
| `assigned:` | `assigned:Tom`     | 依委派受指派者篩選         |
| `project:`  | `project:HomeReno` | 依專案名稱或 ID 篩選       |
| `location:` | `location:office`  | 依任務地點篩選             |
| `where:`    | `where:office`     | 任務地點別名               |
| `id:`       | `id:abc123`        | 尋找確切任務 ID            |
| `-id:`      | `-id:abc123`       | 排除確切任務 ID            |
| `due:`      | `due:today`        | 在該日到期的任務           |
| `due:<=`    | `due:<=7d`         | 未來 7 天內到期的任務      |
| `start:`    | `start:>=tomorrow` | 自該日開始的任務           |
| `created:`  | `created:>=30d`    | 過去 30 天建立的任務       |
| `OR`        | `@home OR @work`   | 符合任一條件               |

**日期格式：**`today`、`tomorrow`、`7d`（7 天）、`2w`（2 週）、`1m`（1 個月）、`2025-01-15`

#### 已儲存搜尋

「已儲存搜尋」是可重複使用之搜尋查詢的捷徑：

1. 使用運算子輸入搜尋查詢
2. 按一下**「儲存搜尋」**按鈕
3. 為搜尋命名（例如「即將到期的工作任務」）
4. 從側邊欄的**已儲存搜尋**區段開啟

**刪除已儲存搜尋：**開啟已儲存搜尋，再按一下垃圾桶圖示。

「已儲存搜尋」與「專注」中的**已儲存篩選條件**彼此獨立。「專注」的已儲存篩選條件會以篩選條件方式同步，並從「專注」篩選控制項編輯。

## 快速收集

### 全域快速鍵

不必切換視窗，即可從桌面任何位置收集任務：

1. 按下全域快速鍵（`Ctrl+Alt+M`）
2. 使用快速新增語法輸入任務
3. 按 Enter 加入收集箱
4. 繼續原本的工作

可在**設定 → 一般 → 輸入 → 全域快速新增快速鍵**中變更或停用。
預設值遵循平台慣例：
- macOS：`Ctrl+Option+M`（預設）、`Ctrl+Option+N`、`Ctrl+Option+Q`，或舊版 `Cmd+Shift+A`
- Linux：`Ctrl+Alt+M`（預設）、`Ctrl+Alt+N`、`Ctrl+Alt+Q`，或舊版 `Ctrl+Shift+A`
- Windows：預設停用，以免 Mindwtr 默默搶走其他應用程式的按鍵組合；啟用時建議選擇 `Ctrl+Alt+M`
- 避免舊版 `Ctrl+Shift+A` 選項：Chrome、Word 與 Excel 都會使用，而全域快速鍵會在整個系統中覆寫它們

**Flatpak/Wayland 注意事項：**部分 compositor 不允許應用程式註冊全域快速鍵。此時請改為將桌面快速鍵綁定至 `flatpak run tech.dongdongbh.mindwtr --quick-add`。

### 系統匣圖示

按一下系統匣圖示即可立即收集：

- 顯示快速新增輸入欄
- 使用自然語言語法
- 任務直接進入收集箱

### 新增任務對話框快速鍵

「新增任務」對話框完全可用鍵盤操作：

- `Enter`：儲存並關閉
- `Shift+Enter`：儲存並保持對話框開啟，以輸入下一項任務（批次收集）
- `Ctrl+Enter`（macOS 為 `⌘Enter`）：儲存並在完整編輯器中開啟任務
- `Esc`：不儲存並關閉
- 建議快顯視窗開啟時（`@`、`#`、`+`、`%`、`!`、`/`），`↑`/`↓` 可移動選取項目，`Enter` 或 `Tab` 則插入反白項目

### 快速新增語法

Mindwtr 會在新增任務時解析自然語言：

| 語法              | 範例                         | 結果                                      |
| ----------------- | ---------------------------- | ----------------------------------------- |
| `@context`        | `@home`                      | 加入情境                                  |
| `#tag`            | `#focused`                   | 加入標籤                                  |
| `+Project`        | `+HomeReno`                  | 指派至專案                                |
| `+Multi Word`     | `+New Project`               | 指派至 "New Project"                      |
| `+"Quoted Name"`  | `+"New Project" call Bob`    | 引號界定句中多字名稱（領域也可用 `!"Area Name"`） |
| `!Area`           | `Plan roadmap !Work`         | 指派至領域                                |
| `%Person`         | `Ask %Jim for budget`        | 設定「指派給」（委派／等待對象）；新建多字名稱使用 `%"Full Name"` |
| `/area:<name>`    | `/area:Personal`             | 指派至領域（不能有空格）                  |
| `/due:date`       | `/due:friday`                | 設定截止日期                              |
| `/energy:<level>` | `/energy:medium`             | 設定精力程度（`low`、`medium`、`high`）   |
| `/note:text`      | `/note:call back`            | 加入描述                                  |
| `/status`         | `/next`, `/waiting`, `/someday`, `/done`, `/archived`, `/inbox` | 設定狀態 |

**日期格式：**today、tomorrow、friday、next week、in 3 days、2025-01-15

## 音訊收集與轉錄

使用 AI 轉錄，以語音收集任務。

### 設定

1. 前往**設定 → AI 助理**。
2. 啟用**語音轉文字**。
3. 選擇**供應商**：
   - **OpenAI / Gemini**：需要 API 金鑰（雲端）。
   - **離線（Whisper）**：在裝置本機執行。按一下**下載**，取得一次模型。
4. 設定**處理模式**：
   - **智慧解析**：從語音中擷取日期（`tomorrow`）、優先順序及專案。
   - **僅逐字稿**：將內容逐字轉錄至任務。

### 使用音訊收集

- **快速新增**：在「快速新增」列中將收集模式切換為**音訊**（麥克風圖示）。
- **錄音**：按一下麥克風開始錄音，再自然說出任務。
- **完成**：按下停止以轉錄，文字會填入輸入欄。
- **附件**：在**設定 → 一般**啟用「儲存音訊附件」，保留原始語音筆記。

## 任務管理

### 建立任務

1. 使用任何清單檢視底部的輸入欄
2. 從任何位置使用全域快速鍵
3. 按一下系統匣圖示快速收集
4. Flatpak/Wayland 無法使用內建快速鍵時，將 `flatpak run tech.dongdongbh.mindwtr --quick-add` 綁定為自訂快速鍵
5. 使用快速新增語法輸入任務標題
6. 按 Enter 加入

### 編輯任務

- 按一下任務開啟編輯面板
- 編輯：標題、狀態、情境、標籤、描述、地點
- 設定：截止日期、開始日期、回顧日期、預估時間、重複規則
- 管理檢查清單項目
- 描述 Markdown 支援無序清單與任務核取方塊（`- item`、`[ ] item`、`[x] item`）
- 開啟清單的**詳細資料**切換鈕後，任務列會將描述第一行轉譯為 Markdown；展開任務列可閱讀完整描述
- 在任務描述或專案筆記輸入 `[[`，從浮動選擇器連結另一項任務或專案
- 這些連結只供導覽，不會同步任務之間的完成狀態
- 描述中的 Markdown 核取方塊只屬於筆記，不會影響任務檢查清單
- 將多行文字貼入檢查清單項目，會逐行建立一個項目（可辨識項目符號、編號與 `[x]` 標記）
- 指派至專案

### 任務屬性

| 屬性              | 說明                                                |
| ----------------- | --------------------------------------------------- |
| **狀態**          | inbox、next、waiting、someday、done、archived       |
| **優先順序**      | low、medium、high、urgent                           |
| **情境**          | 地點／工具標籤（例如 @home、@work）                 |
| **標籤**          | 精力／模式標籤（例如 #focused、#lowenergy）         |
| **截止日期**      | 任務何時到期                                        |
| **開始日期**      | 何時開始處理                                        |
| **回顧日期**      | 供回顧使用的提醒日期                                |
| **預估時間**      | 5min、10min、15min、30min、1hr、2hr、3hr、4hr、4hr+ |
| **重複**          | 每天、每週、每月、每年 + 策略                       |
| **檢查清單**      | 多步驟任務的子項目                                  |
| **描述**          | 可預覽的 Markdown 格式筆記                          |
| **附件**          | 附加至任務的檔案與連結                              |
| **地點**          | 實體地點                                            |
| **受指派者／人員** | 「等待中」與 `assigned:` 搜尋所用的委派對象        |
| **專案**          | 上層專案指派                                        |
| **分區**          | 專案內的選用群組                                    |

**附件：**「新增連結」欄位同時接受 URL 與本機檔案路徑（例如 `/home/user/doc.pdf`、`C:\Users\you\file.txt` 或 `file://...`）。
同步、清理與音訊筆記詳情請參閱[附件](/zh-Hant/use/attachments)。

### 重複任務

完成重複任務時，Mindwtr 會自動建立更新日期後的下一個實例。

- **固定排程**（預設）：維持固定排程節奏。
  範例：每 5 天一次會固定在規劃週期。
- **完成後重複**：從完成時間起推算下一個截止日期。
  範例：現在完成，下一次於 5 天後到期。
- **結束：永不／於日期／N 次後**：選擇系列永久執行、在特定日期停止，或在固定發生次數後停止。
  `After N occurrences` 會將目前任務計入系列總數。

Mindwtr 只保留一個使用中的重複任務實例。有截止日期或開始時間時，「行事曆」會顯示目前實例；未來發生時間不會預先建立，直到目前任務完成。若啟用**在行事曆顯示下一次發生時間**，則會顯示僅供規劃的預覽。

可在任務編輯器的重複欄位切換**完成後重複**。
若系列需要自動停止，也可在同一個重複設定面板中設定結束日期或發生次數。

## 批次動作

選取多項任務進行批次操作：

1. 按一下清單標題中的**「選取」**按鈕
2. 按一下任務以選取／取消選取
3. 使用動作列：
   - **移動**：變更所有選取任務的狀態
   - **指派領域**：將所有選取任務移至特定領域下（或清除領域）
   - **加入標籤**：為所有選取任務加入標籤
   - **移除標籤**：從所有選取任務移除標籤
   - **加入情境**：為所有選取任務加入情境
   - **移除情境**：從所有選取任務移除情境
   - **刪除**：刪除所有選取任務
4. 按一下**「完成」**離開選取模式

### 排序

使用排序下拉選單，依以下條件排列任務：
- 預設（依狀態）
- 截止日期
- 開始日期
- 回顧日期
- 標題（字母順序）
- 建立時間（最早／最新）

## 鍵盤快速鍵

Mindwtr 支援**標準**（Gmail/Todoist 風格）、**Vim** 與 **Emacs** 按鍵綁定預設。可在設定中變更。

完整清單請參閱[桌面版鍵盤快速鍵](/zh-Hant/use/keyboard-shortcuts)。

**快速參考（標準）：**
- `/`：開啟搜尋
- `?`：顯示快速鍵說明
- `gi`：前往收集箱
- `gn`：前往下一步行動
- `gf`：前往專注
- `j/k`：向下／向上移動選取項目
- `Enter`：開啟選取任務
- `Shift+Enter`：編輯選取任務
- `e`：標示為已完成／未完成
- `x`：選取／取消選取任務以執行批次動作
- `#`：刪除任務
- `z`：復原上一次完成／刪除

**快速參考（Vim）：**
- `j/k`：向下／向上移動選取項目
- `Enter`：開啟選取任務
- `e`：編輯選取任務
- `x`：切換完成狀態
- `dd`：刪除任務

所有預設都可用 `Ctrl+Z` / `Cmd+Z` 復原上一次任務完成或刪除。

在所有預設中，先按 `s` 再按一個字母，可直接設定選取任務的狀態（`si` 收集箱、`sn` 下一步、`sw` 等待中、`ss` 將來、`sd` 已完成、`sa` 已歸檔），並顯示復原通知。`Insert` 會跳至新增任務輸入欄（若清單沒有輸入欄，則開啟快速新增）。

## 設定

從側邊欄開啟設定。

### 一般
- **外觀**：淺色、深色或跟隨系統
- **語言**：英文、簡體中文、繁體中文、印地文、西班牙文、阿拉伯文、法文、葡萄牙文、俄文、德文、日文、越南文、土耳其文、韓文、義大利文、波蘭文、荷蘭文、捷克文
- **鍵盤快速鍵**：標準、Vim 或 Emacs 預設
- **登入時啟動**：登入後自動啟動 Mindwtr

### 通知

**任務提醒：**
- 啟用／停用截止日期與開始時間的任務通知

**每日摘要：**
- **早間簡報**：今日到期、逾期與專注任務摘要
- **晚間回顧**：提醒回顧並結束一天
- 設定時間（例如上午 9:00、晚上 8:00）

**每週回顧：**
- **提醒**：每週收到開始回顧的通知
- **回顧日期／時間**：自訂希望收到提醒的時間

### GTD
- **自動歸檔**：在設定天數後自動將已完成任務移至已歸檔（預設 7 天）；選擇**永不**可將完成任務保留在「已完成」
- **專注任務上限**：選擇可標示為今日專注的任務數量
- **功能**：需要時可啟用的選用訊號：
  - **優先順序**：在任務上顯示優先順序旗標
  - **預估時間**：加入供時間區塊使用的持續時間欄位
- **收集箱處理**：保留引導式收集箱處理以學習 GTD，或將預設切換為**快速**模式，採用精簡的單畫面流程
  - 選擇是否顯示 2 分鐘捷徑、提早詢問專案、包含情境／標籤、允許排程，以及在處理期間提供參考資料選項
- **任務編輯器版面**：選擇預設顯示的欄位、將欄位移至不同區段並重新排序
  - 選擇編輯器開啟方式：使用**側邊預覽**進行行內編輯，或使用**快顯**專注編輯
  - 可將**截止日期**等欄位移入**排程**，並選擇預設展開的區段
- **管理**：在同一處維護可重複使用的中繼資料
  - 編輯**領域**、**人員**、**情境**與**標籤**，不必逐一尋找任務
  - 人員會保留可重複使用的受指派者名稱、筆記與參考連結，供委派工作及建議使用
  - 指派時建立領域與人員的位置，請參閱[領域與人員](/zh-Hant/use/areas-people)

### 資料與同步

**同步後端：**
- **雲端同步**：支援組建中的 Dropbox，以及可用時 macOS 上的 iCloud
- **資料夾／檔案同步**：透過共用 JSON 檔案／資料夾進行檔案同步（Google Drive、Syncthing、OneDrive 等）
- **進階／自訂伺服器**：WebDAV 或自行託管 Mindwtr Cloud

**設定同步選項：**
- 選擇要跨裝置同步的偏好（主題、語言／日期格式、GTD 預設、外部行事曆 URL、AI 設定、已儲存篩選條件）
- API 金鑰與本機模型路徑永遠不會同步

**同步狀態：**
- 側邊欄頁尾顯示上次同步時間及線上／離線狀態

WebDAV 設定：
- 伺服器 URL（資料夾 URL；Mindwtr 會將 `data.json` 存在其中）
- 使用者名稱與密碼

詳細設定請參閱[資料與同步](/zh-Hant/data-sync/)。
