# 可重複使用的清單（範本）

Mindwtr 支援可重複使用的檢查清單，適合沒有嚴格排程但會反覆執行的任務（例如打包、採買或行前準備）。

## 兩種做法

### 複製（範本）

每次執行都需要一份全新副本時使用。

範例：**打包清單**

1. 在「將來/也許」中保留一項主範本任務
2. 規劃旅行時複製它
3. 編輯副本而不影響主範本

複製時會重設：

- 所有檢查清單項目恢復為未勾選
- 清除排程日期與回顧日期
- 清除重複規則
- 副本成為全新任務，並取得新的檢查清單與附件 ID

### 重設檢查清單

想再次使用同一份清單時使用。

範例：**採買清單**

1. 購物時逐項勾選
2. 點選**重設檢查清單**
3. 下次繼續使用

重設會：

- 取消勾選每個檢查清單項目
- 直接更新原任務，而不建立第二份副本
- 若任務已標示完成，則重新開啟

## 在哪裡操作

- **行動版：**任務編輯器 → **複製任務**或**重設檢查清單**
- **桌面版：**任務動作選單

## 如何選擇

- 旅程準備、新手引導清單，或每次執行都應保留各自筆記與時程的事項，請使用**複製任務**。
- 購物、例行維護，或同一項任務能長期沿用的清單，請使用**重設檢查清單**。
- 若清單還包含日期、重複規則或專案專屬筆記，複製通常較安全。

## 提示

- 將主範本放在**將來/也許**或干擾較少的專案中，以免塞滿日常清單。
- 經常複製的範本任務可取名為 `打包清單（範本）` 等一眼可辨的名稱。
- 若清單中的每一項開始需要各自的截止日期或重複規則，請拆成獨立任務，不要讓一份可重複清單承擔過多用途。
