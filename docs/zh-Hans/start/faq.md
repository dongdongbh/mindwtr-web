## 通用问题

### Mindwtr 是什么？

Mindwtr 是一款跨平台的 Getting Things Done（GTD）效率应用，可帮助你记录、明确、整理和执行任务。支持桌面端（Windows、macOS、Linux）和移动端（Android、iOS）。

### Mindwtr 免费吗？以后也会保持免费吗？

是的。Mindwtr 是一款开源（AGPL-3.0）且可免费使用的应用。同步或其他任何功能都没有付费墙、订阅或高级层级。我们计划一直如此：无论经济状况如何，每个人都应该能够使用优秀的软件；项目应由自愿支持它的人来提供支持，而不是依靠付费墙。

项目确实有持续成本：每年的 Apple 开发者费用、托管、开发工具，以及投入功能开发、问题修复和社区支持的时间。[捐赠](https://mindwtr.app/donate)让项目得以持续发展。

需要坦诚说明一点：目前捐赠尚不足以覆盖成本，差额由维护者承担。如果这个缺口有朝一日大到难以维持，项目可能会增加可选的付费服务（例如为不想自行托管的用户提供托管同步服务器）来弥补成本。无论如何，应用本身都会保持免费和开源。你今天可以使用的任何功能都不会转到付费墙后。

### Mindwtr 会维护多久？

Mindwtr 是一个出于热爱的项目，也是维护者用来管理自己生活的工具，因此他会维护它，因为他自己也需要它得到维护。提交历史如实展现了这种维护在实践中的样子。

任何个人项目都无法承诺“永远”存在，因此 Mindwtr 选择从设计上确保最坏情况仍然安全：它采用本地优先架构，你的数据以开放格式存储在你控制的文件中，代码采用 AGPL 许可，并可通过独立渠道进行可复现构建。即使开发停止，应用仍可离线工作，你的数据依然可读，任何人也都可以复刻代码。你可以放心把自己的工作流交给它，也可以随时带着完好的数据离开。

### Mindwtr 是由 AI 生成的吗？项目对 AI 持什么立场？

Mindwtr 由 AI 辅助开发，而不是由 AI 生成。这个区别很重要。

Mindwtr 背后的架构、产品方向、功能设计和 GTD 理念均由维护者设计并负责。AI 工具协助执行工作：更快地起草实现、减少样板代码、改进措辞并加快问题调查。每项发布的更改仍会经过明确说明、审查和测试，并由维护者对结果负责。

如今，AI 辅助开发是正常的软件工程实践，大型科技公司和开源领域也不例外。AI 是一种工具，就像 IDE、自动补全、文档和 Stack Overflow。使用 AI 并不会取代开发者的工作，而是将更多工作转向方向把握、判断、审查、集成和验证。

Mindwtr 由一位拥有 10 多年软件工程经验的独立开发者维护。正是这些经验让 AI 变得有用而非危险：知道该构建什么、不该构建什么，何时输出有误，以及如何保持系统连贯。AI 帮助如此广泛的项目由一个人维持，但它不会决定设计，也不对结果拥有决定权。

议题和讨论回复均由维护者撰写。AI 可能用于润色英文措辞，但没有代理会自动分类或自动回复议题。

有关贡献，请参阅 [CONTRIBUTING.md 中的 LLM 辅助编码章节](https://github.com/dongdongbh/Mindwtr/blob/main/docs/CONTRIBUTING.md#llm-assisted-coding-vibe-coding)。

### 为什么有这么多提交和议题报告？

Mindwtr 是一款跨平台的终端用户应用，而不是只有单一狭窄命令界面的小型 CLI 或库。它包含桌面端和移动端客户端、本地优先持久化、同步后端、导入、通知、快速记录、小组件、翻译、安装渠道和 GTD 工作流决策。此类产品自然会产生许多小型后续提交，因为界面打磨、平台差异、打包修复和边缘情况报告都是正常维护的一部分。

公开议题的范围也有意保持宽泛。GitHub 议题包括功能请求、用户体验改进、特定平台的边缘情况、发布打包报告、文档缺口和已确认的问题。许多报告不会阻碍主要的记录、整理和回顾工作流，但仍会公开跟踪，让用户看到已知问题和变更内容。快速跟进修复是维护模式的一部分，并不表示议题遭到忽视或隐藏。

### 我的数据存储在哪里？

所有数据都存储在你的设备本地：
- **桌面端数据（Linux）：**`~/.local/share/mindwtr/data.json`
- **桌面端配置（Linux）：**`~/.config/mindwtr/config.toml`
- **Windows/macOS：**`%APPDATA%/mindwtr/` 或 `~/Library/Application Support/mindwtr/`
- **移动端：**应用内部存储

详情请参阅[数据与同步](/zh-Hans/data-sync/)。

### 有云同步吗？

Mindwtr 支持文件同步、WebDAV、自托管云同步以及 Dropbox 同步（在支持的构建中）。请参阅[数据与同步](/zh-Hans/data-sync/)。

### 哪种同步方式最快：本地文件夹还是 WebDAV？

在硬件和网络距离相近的情况下，**本地文件夹同步通常比 WebDAV 更快**。文件夹同步通过普通文件系统调用读写快照，而 WebDAV 的每次请求都会增加 HTTP 往返、身份验证和服务器处理。处理大型附件时差异最明显，因为 WebDAV 会将附件作为单独的上传和下载进行传输。不过，瓶颈通常是文件夹背后的存储提供方（例如在后台同步的 Syncthing 或云盘客户端），而不是 Mindwtr 本身。请选择适合你的后端；有关同步时实际传输的内容，请参阅[数据生命周期](/zh-Hans/data-sync/data-lifecycle)。

### Mindwtr 直接支持 iCloud 同步吗？

是的，在支持的 Apple 构建中可以使用。

- **iPhone / iPad：**在**设置 → 同步**中可以使用原生 **iCloud** 后端
- **macOS 桌面端：**在**设置 → 同步**中可以使用原生 **iCloud** 后端
- **Android / Windows / Linux：**没有原生 iCloud 后端

请参阅 [iCloud 同步](/zh-Hans/data-sync/icloud)和[数据与同步](/zh-Hans/data-sync/)。

### 如何使用 OneDrive 同步（尤其是在 Android 上）？

Mindwtr 已经可以**通过文件同步**配合 OneDrive 使用：

- **Windows/macOS：**将 Mindwtr 的 `data.json` 放入 OneDrive 文件夹。OneDrive 会自动处理同步。
- **Android：**官方 OneDrive 应用**不**提供真正的双向文件夹同步。
  请使用 **OneSync (Autosync for OneDrive)** 或 **FolderSync** 等辅助“桥接”应用，让本地文件夹保持同步。
  然后在 Mindwtr 的**设置 → 同步**中指向该本地文件夹（Mindwtr 使用其中的 `data.json`）。

这与 Obsidian 等本地优先应用采用的方式相同。

### 为什么 Mindwtr 没有“使用 OneDrive / Google Drive 登录”按钮？

Mindwtr 采用本地优先和离线优先架构。

Mindwtr 现在已在支持的构建中提供原生 **Dropbox OAuth** 同步，但**不**提供原生 OneDrive/Google Drive OAuth 集成。
扩展到每一个提供方都会增加大量维护和合规成本。

对于 OneDrive/Google Drive，文件同步既能让数据由你掌控，也能避免巨大的维护和安全负担。

### Mindwtr 可以与电子邮件（Gmail/Outlook）集成或接收转发的邮件吗？

不能直接实现。构建完整的电子邮件客户端需要：

- Gmail/Outlook 的 OAuth 访问权限（如今需要昂贵的安全审核）
- 稳健的 MIME/HTML 解析和附件处理
- 针对不同提供方持续维护

**当前替代方案：**
- **桌面端：**在任务中粘贴 `message://` 或邮件链接，或者在支持此功能的客户端中将邮件拖入任务备注。
- **移动端：**使用分享面板将选中的邮件内容发送到 Mindwtr。

Mindwtr **不**提供 `add@mindwtr.com` 收件箱，因为这需要中央服务器接收和存储你的电子邮件。

## 功能

### GTD 是什么？

Getting Things Done（GTD）是 David Allen 创立的一套效率方法。它包括五个步骤：记录、明确、整理、回顾和执行。请参阅 [GTD 概览](/zh-Hans/use/gtd-overview)。

### GTD 视野层级如何映射到 Mindwtr？

Mindwtr 原生表示较低的视野层级：

- **视野 0（行动）：**下一步行动和任务列表。
- **视野 1（项目）：**明确的项目实体。
- **视野 2（领域）：**领域将相关项目归组。

视野 3–5（目标、愿景、使命）目前还没有专用实体。大多数用户使用以下方式跟踪：

- **参考**清单（或“目标”领域）
- 项目备注以及指向这些参考条目的链接
- 每周回顾清单

如果你依赖明确的目标/愿景对象，请创建议题，说明你期望的工作流和回顾频率。

### 项目分区有什么用途？

**项目分区**是一个项目内带标签的分组。你可以使用分区让较长的项目保持易读，例如在**发布网站**项目中设置**设计**、**开发**和**内容**分区。

分区不是子任务，也不是独立项目。它们只是同一项目成果内任务的标题。

任务上的**项目分区**字段会将该任务分配到当前项目的某个分区。只有当任务已属于一个设有分区的项目时，它才会发挥作用。如果任务不属于任何项目，或项目没有分区，请将其留空。

### 为什么一个项目可以有多个“下一步”任务？

在 Mindwtr 中，**下一步**是一种任务状态：任务已经明确并且可以执行。它并不总是等同于项目中“唯一的下一步行动”。

项目类型控制哪些任务会显示出来：

- **顺序：**一次只显示一个可执行的“下一步”任务。后面的“下一步”任务仍留在项目中，等待轮到它们。
- **并行：**可以同时显示多个彼此独立的“下一步”任务，因为它们可以按任意顺序完成。

顺序项目中的后续步骤并不是**参考**。参考用于不可执行的支持材料、备注和文档。

### 如何将项目标记为完成？

**将其归档**——在 Mindwtr 中，归档就是完成项目的方式。打开项目并使用**归档**按钮（桌面端位于项目标题中，移动端位于项目详情的**操作**下）。任何未完成的任务都会随项目一起完成；如果归档得太早，使用**重新激活**可恢复所有内容。完成项目的最后一个行动时，“下一步行动是什么？”提示也会直接提供**完成项目**选项，因此你很少需要特意寻找该按钮。否则，即使项目的所有任务都已完成，项目仍会保持活动状态，直到你亲自结束该成果；这在每周回顾时是一项很有用的提示。

### 可以把任务转为项目吗？

可以。最简单的时机是处理收集箱：当引导流程询问某一条目是否需要多个步骤时，回答“是”，该记录就会变成项目，你可以为其命名并定义第一个下一步行动。之后添加的任何行动都会回到收集箱，并已关联该项目，以便每项行动分别经过明确处理。请参阅 [GTD 工作流](/zh-Hans/use/gtd-workflow)。

### 为什么任务没有显示在“专注”中？

“专注”有意只显示你当前可以执行的任务，因此任务可能因以下几个原因而隐藏：

- 它设置了**未来开始日期**，会在日期到来时重新出现。
- 它是**顺序项目中靠后的任务**，因此只会显示第一个可执行任务。
- 活动的**情境筛选器**将其排除。
- 它的状态不可执行（收集箱、将来、等待中）。

使用**项目**、**情境**或**搜索**查看完整任务清单。请参阅[“专注”如何排列可执行行动](/zh-Hans/use/gtd-workflow#焦点如何对可执行行动排序)。

### Mindwtr 支持开始日期或提醒日期吗？

二者都支持：

- **开始日期**会推迟任务：未来的开始日期会将其从“专注”和“下一步行动”中隐藏；日期到来时，它会以原有状态重新出现。“专注”和“下一步行动”列表都会隐藏推迟的任务直至开始日；想提前查看时，请使用**项目**或**搜索**。
- **回顾日期**就是提醒日期：日期到来时，Mindwtr 会显示该任务，供你重新考虑。在你做出决定前，不会发生任何变化。
- **开始提前量**将开始日期与截止日期关联（例如在截止日期前两天变为可见）。

请参阅[日期与状态](/zh-Hans/use/gtd-workflow#日期与状态)。

### 如何启用“优先级”或“预计时间”？

Mindwtr 采用渐进式披露，因此默认隐藏可选字段。

在以下位置控制任务编辑字段：

**设置 -> GTD -> 任务编辑器布局**

在其中开启**优先级**和**预计时间**（并根据需要重新排列字段）。隐藏字段仍会显示在**更多**下，或者在任务的该字段已有内容时显示。


### 已完成任务与已归档任务有什么区别？

对近期完成的任务使用**已完成**。已完成任务会保留完成日期，继续显示在“已完成”视图中，并且在每日或每周回顾时查看已完成的内容很有用。

对希望收起保存的已完成任务使用**已归档**。已归档任务会从普通任务列表中隐藏，并继续保留在“已归档”视图中，供搜索、恢复或永久删除。归档不会删除任务。

在实践中，可以将“已完成”视为短期完成记录，将“已归档”视为长期历史。**自动归档**设置可以在选定天数后自动将已完成任务移到已归档，也可以设为**永不**，以便将所有完成记录保留在“已完成”中。


### 重复任务如何工作？

Mindwtr 支持两种重复策略：

- **严格**（固定日程）：下一日期依据日程模式本身计算。
  示例：即使你延迟完成，每 5 天一次仍会保持这一频率。
- **完成后重复**（灵活）：下一日期根据你实际完成任务的时间计算。
  示例：今天完成，则下一次截止日期为从今天起 5 天后。

在任务编辑器中设置重复（每天/每周/每月/每年）；如果你希望采用灵活方式，请启用**完成后重复**。

Mindwtr 只保留一个活动的重复任务实例。它不会为整个系列预先填充未来的日历条目；只有当前任务完成后，才会创建下一个任务实例。

### 可以在 Google Calendar 或 Apple Calendar 中看到 Mindwtr 中的任务吗？

可以，支持单向推送：

- **Android/iOS：**将带日期的任务推送到设备日历。Android 已通过 Google Calendar 验证；在 iOS 上，请使用 Apple Calendar 中已有的日历。最好在该账户中使用名为 `Mindwtr` 的专用日历。
- **macOS 桌面端：**通过 EventKit 推送到 Apple Calendar。

重复任务的*规则*不会导出为原生日历重复事件。只会推送具体实例。请参阅[日历集成](/zh-Hans/use/calendar-integration)。

### Mindwtr 与 Obsidian 集成吗？

是的，桌面端支持：导入库后，Mindwtr 会保留返回原始笔记的深层链接，因此任务可以直接跳转到 Obsidian 中的源笔记。请参阅 [Obsidian](/zh-Hans/power-users/obsidian)。

### 如何收集用于问题报告的日志？

日志默认关闭。仅在需要报告问题时启用。

**桌面端（Tauri）：**
1. 前往**设置 → 数据**。
2. 启用**调试日志**。
3. 重现问题。
4. 复制**日志文件**路径，并将该文件附加到 GitHub 议题中。

Linux 日志位置（默认）：`~/.local/share/mindwtr/logs/mindwtr.log`

**移动端：**
1. 前往**设置 → 数据**。
2. 启用**调试日志**。
3. 重现问题。
4. 点按**分享日志**，并将其附加到 GitHub 议题中。

日志仅保存在本地，并在写入前对常见凭据（密码/令牌）进行脱敏。

### 可以使用自然语言添加任务吗？

可以！Mindwtr 支持快速添加语法：
- `@context`：添加情境
- `#tag`：添加标签
- `!Area` 或 `/area:<name>`：分配到领域
- `%Person`：设置“分配给”的人员（新建多词姓名时使用 `%"Full Name"`）
- `/due:date`：设置截止日期（today、tomorrow、friday、next week 等）
- `/energy:low`、`/energy:medium` 或 `/energy:high`：设置精力水平
- `/note:text`：添加描述
- `/status`：设置状态（`/next`、`/waiting`、`/someday`、`/done`、`/archived`、`/inbox`）
- `+ProjectName`：分配到项目

示例：`Call client /due:friday @phone`

### 什么是情境？

情境是表明你可以在何处或借助什么完成任务的标签。例如：`@home`、`@work`、`@phone`、`@computer`。按情境筛选，只查看你现在可以完成的任务。请参阅[情境与标签](/zh-Hans/use/contexts-tags)。

### 情境与领域有什么区别？

它们回答的问题不同：

- **领域**回答：*“这项责任与我的生活或工作的哪一部分相关？”*（工作、家庭、健康、某个客户……）
- **情境**回答：*“考虑到我所在的位置和拥有的工具，现在可以做什么？”*（`@computer`、`@errands`……）

一项任务可以同时拥有三者：**领域：客户 A**、**项目：网站改版**、**情境：@computer**。请参阅 [GTD 概览](/zh-Hans/use/gtd-overview)。

### 如何快速记录任务？

**桌面端：**
- 使用全局快捷键，从任何位置打开快速添加
- 在 Flatpak/Wayland 上，如果内置快捷键不可用，请将 `flatpak run tech.dongdongbh.mindwtr --quick-add` 绑定为自定义系统快捷键
- 单击托盘图标即可即时记录
- 在任意视图的输入字段中键入内容

**移动端：**
- 使用分享面板从任何应用记录
- 添加主屏幕小组件，实现一键记录
- 使用“收集箱”选项卡的输入字段

## 桌面端

### 有哪些键盘快捷键？

Mindwtr 支持 Vim 和 Emacs 键位预设。按 `?`（Vim）或 `Ctrl-h`（Emacs）可查看所有快捷键。请参阅[桌面端键盘快捷键](/zh-Hans/use/keyboard-shortcuts)。

### 如何更改主题？

前往“设置 → 外观”。选择浅色、深色或跟随系统。

### 如何与手机同步？

1. 在“设置”中配置同步文件夹（指向 Dropbox、Syncthing 等）
2. 在移动端的“设置 → 同步”中选择同步文件夹
3. 两个平台都会在数据发生变化以及切换应用时自动同步

请参阅[数据与同步](/zh-Hans/data-sync/)。

### 支持通知吗？

支持！Mindwtr 会针对以下内容发送桌面通知：
- 截止日期提醒
- 开始时间提醒
- 重复任务提醒

你可以将通知稍后提醒。

首次启用通知时，**macOS** 会请求权限。在 **Linux** 上，请确保通知守护程序正在运行。

## 移动端

### 支持哪些平台？

- **Android：**通过 Google Play 或下载 APK 提供完整支持
- **iOS：**可从 App Store 获取，也可通过 TestFlight 测试版使用。维护 App Store 版本仍需每年支付 Apple Developer 费用，因此赞助支持有助于保持 iOS 版本的持续提供。

### 为什么桌面端和移动端的编辑方式不同？

Mindwtr 遵循各平台的惯例：
- **桌面端：**单击切换详情，双击打开完整编辑模式（通过任务的 `⋯` 菜单就地重命名标题），右键单击打开上下文菜单。
- **移动端：**单击打开编辑模式，滑动操作用于快速更改。

这些模式让应用在各个平台上都保持快速且符合使用习惯。

### 如何在 Android 上安装？

从 Google Play 安装，或从 [GitHub Releases](https://github.com/dongdongbh/Mindwtr/releases) 下载 APK。如果出现提示，请启用“安装未知来源应用”。请参阅[移动端安装](/zh-Hans/start/mobile-installation)。

### 如何从其他应用记录？

使用**分享面板**！在任何应用（浏览器、电子邮件、笔记）中查看内容时，点按“分享”并选择 Mindwtr。Mindwtr 会打开记录页面，将共享内容填入备注，以便你添加标题、调整字段并将其保存到收集箱。

### 有小组件吗？

有！将 Mindwtr 小组件添加到主屏幕，即可快速记录并查看专注事项。

### AI 助手是必需的吗？

不是。AI 助手是可选功能，默认关闭。没有它，Mindwtr 也能完整运行。
启用后，它会使用你自己的 API 密钥（BYOK）。请参阅 [AI 助手](/zh-Hans/power-users/ai-assistant)。

### 滑动操作如何工作？

在收集箱中，向右滑动任务可将其标记为已完成。其他视图可能有不同的滑动操作。

### 如何与桌面端同步？

1. 将备份导出到同步文件夹（Google Drive、Syncthing 等）
2. 在“设置 → 同步”中选择该文件夹
3. 应用会在数据发生变化以及进入后台时自动同步

请参阅[数据与同步](/zh-Hans/data-sync/)。

### 移动应用会发送通知吗？

会！Mindwtr 会针对以下内容发送推送通知：
- 截止日期提醒
- 开始时间提醒
- 每日摘要提示
- 每周回顾提醒

你可以直接从通知中选择稍后提醒。
点按通知正文会打开**回顾**页面。

## 同步与数据

### 同步会导致数据丢失吗？

不会。Mindwtr 使用“最后写入者优先”合并，保留每个条目的最新版本。软删除的条目会在设备之间正常同步。

### 可以使用多个同步服务吗？

我们建议只使用一个同步文件夹以避免冲突。请选择一项服务（Dropbox、Google Drive、Syncthing）并始终使用它。

### 如何备份数据？

**桌面端：**使用**设置 → 数据 → 导出备份**，或备份应用数据文件夹中的 `data.json`。
**移动端：**使用**设置 → 数据 → 导出备份**保存副本。

请参阅[备份与恢复](/zh-Hans/data-sync/backup-restore)。

### 可以恢复已删除的任务吗？

目前还没有恢复单个任务的界面，但你可以从之前的备份或恢复快照还原本地数据。

请参阅[备份与恢复](/zh-Hans/data-sync/backup-restore)。

### 可以从 TickTick 导入吗？

可以。Mindwtr 可以从**设置 → 数据 → 从 TickTick 导入**中导入 TickTick CSV 和 ZIP 备份。

- TickTick 文件夹会成为 Mindwtr 中的领域
- TickTick 清单会成为 Mindwtr 中的项目
- 在可能的情况下，会保留清单内容、标签、日期、优先级和受支持的重复规则

请参阅 [TickTick 导入](/zh-Hans/import/ticktick)。

### 可以从 Todoist 导入吗？

可以。Mindwtr 可以从**设置 → 数据 → 从 Todoist 导入**中导入 Todoist CSV 导出和 ZIP 备份。

- Todoist 项目会成为 Mindwtr 中的项目
- 子任务会成为清单条目
- 已分配项目的活动任务会留在相应项目中；未分配的活动任务则保留供收集箱处理

请参阅 [Todoist 导入](/zh-Hans/import/todoist)。

### 可以从 DGT GTD 导入吗？

可以。Mindwtr 可以从**设置 → 数据 → 从 DGT GTD 导入**中导入 DGT GTD JSON 导出和 ZIP 归档。

- DGT 文件夹会成为 Mindwtr 中的领域
- DGT 项目会成为 Mindwtr 中的项目
- DGT 清单会成为清单任务
- DGT 情境和标签会得到保留

不受支持的 DGT 重复模式会导入一次，原始重复文本会保留在描述中。

请参阅 [DGT GTD 导入](/zh-Hans/import/dgt-gtd)。

### 可以从 OmniFocus 导入吗？

可以。Mindwtr 可以从**设置 → 数据 → 从 OmniFocus 导入**中导入 OmniFocus CSV 导出和 Omni Automation JSON / ZIP 导出。

- 如果元数据可用，OmniFocus 文件夹可以成为 Mindwtr 中的领域
- 必要时，OmniFocus 项目会成为 Mindwtr 中的项目
- 独立的 OmniFocus 行动不会归入项目
- 会保留 JSON 路径中受支持的备注、标签、推迟日期、截止日期、完成状态和重复规则
- 简单的嵌套任务可以成为清单条目，更深的层级则会展平，并保留原始路径

如果重复规则很重要，请使用 Omni Automation JSON / ZIP 导出，而不是 CSV。当 Mindwtr 没有对应字段时，计划日期和时长文本会保留在描述中。

请参阅 [OmniFocus 导入](/zh-Hans/import/omnifocus)。

### 可以从 Apple 提醒事项导入吗？

可以，在 iPhone 和 iPad 上。Mindwtr 可以从**设置 → 数据 → 从 Apple 提醒事项导入**中导入未完成的 Apple 提醒事项。

- 选择要从中导入的提醒事项列表
- 导入的提醒事项会成为 Mindwtr 中的任务
- Mindwtr 确认导入后，你可以选择从 Apple 提醒事项中删除这些提醒事项

Apple 提醒事项导入是单向导入，而不是同步。请参阅 [Apple 提醒事项导入](/zh-Hans/data-sync/#apple-提醒事项导入-ios)。

## 故障排除

### 应用无法启动（Linux 桌面端）

确保已安装 WebKitGTK：
```bash
# Arch
sudo pacman -S webkit2gtk-4.1

# Debian/Ubuntu
sudo apt install libwebkit2gtk-4.1-0
```

### 为什么 AUR 安装或构建目录这么大？

除非你明确希望从源码构建，否则请在基于 Arch 的发行版上使用 `mindwtr-bin`：

```bash
yay -S mindwtr-bin
```

`mindwtr-bin` 安装预构建的 GitHub 发布软件包，应当是体积小、速度快的 AUR 安装方式。源码软件包 `mindwtr` 会在本地构建桌面应用，必须下载 Tauri、Rust、Bun 和 React 应用的构建依赖。这可能会在构建期间占用多得多的磁盘空间。

源码软件包应获取发布标签的归档，而不是完整的 Git 历史。如果某个 AUR 助手似乎下载了非常大的 Git 检出，请确认你安装的是用于二进制软件包方式的 `mindwtr-bin`；或者报告源码软件包的行为，以便修正 AUR 配方。

### 应用启动时崩溃（移动端）

尝试清除应用数据：
1. 前往“设置 → 应用 → Mindwtr”
2. 点按“存储 → 清除数据”
3. 重新打开应用

注意：此操作会删除本地数据。

### 任务没有同步

1. 检查同步文件夹是否可以访问
2. 确认同步服务正在运行
3. 检查文件权限
4. 尝试在“设置”中手动同步

### 通知无法使用

**桌面端：**
- 检查系统通知设置
- 确保应用拥有通知权限

**移动端：**
- 在设备设置中授予通知权限
- 检查应用通知设置
