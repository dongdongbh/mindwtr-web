# Listes réutilisables (modèles)

Mindwtr propose des listes de contrôle réutilisables pour les tâches que vous répétez sans calendrier strict, par exemple préparer une valise, faire les courses ou organiser un voyage.

## Deux méthodes

### Dupliquer (modèles)

À utiliser lorsque chaque occurrence doit être une nouvelle copie.

Exemple : **Liste pour la valise**

1. Conservez une tâche principale dans Un jour/Peut-être.
2. Dupliquez-la lorsque vous préparez un voyage.
3. Modifiez la copie sans toucher à l’original.

La duplication réinitialise les éléments suivants :

- les éléments de la liste de contrôle sont tous décochés ;
- les dates de planification et de revue sont effacées ;
- la récurrence est supprimée ;
- la copie devient une nouvelle tâche avec de nouveaux identifiants de liste de contrôle et de pièces jointes.

### Réinitialiser la liste de contrôle

À utiliser lorsque vous souhaitez reprendre la même liste.

Exemple : **Courses**

1. Cochez les articles pendant vos achats.
2. Touchez **Réinitialiser la liste de contrôle**.
3. Réutilisez-la la prochaine fois.

La réinitialisation :

- décoche tous les éléments ;
- met à jour la tâche existante au lieu d’en créer une copie ;
- rouvre la tâche si elle avait déjà été marquée comme terminée.

## Où trouver ces actions

- **Mobile :** éditeur de tâche → **Dupliquer la tâche** ou **Réinitialiser la liste de contrôle**
- **Ordinateur :** menu des actions de la tâche

## Laquelle choisir ?

- Utilisez **Dupliquer la tâche** pour préparer un voyage, créer des listes d’accueil ou toute activité dont chaque occurrence doit avoir ses propres notes et dates.
- Utilisez **Réinitialiser la liste de contrôle** pour les courses, l’entretien régulier ou les listes dont la même tâche reste utile dans le temps.
- Si la liste possède aussi des dates, une récurrence ou des notes propres à un projet, la duplication est généralement plus sûre.

## Conseils

- Conservez la version principale dans **Un jour/Peut-être** ou dans un projet discret afin qu’elle n’encombre pas les listes quotidiennes.
- Donnez aux tâches modèles des noms explicites comme `Liste pour la valise (modèle)` si vous les dupliquez souvent.
- Si chaque élément d’une liste de contrôle commence à nécessiter ses propres échéances ou règles de récurrence, transformez-les en tâches distinctes au lieu de surcharger une seule liste réutilisable.
