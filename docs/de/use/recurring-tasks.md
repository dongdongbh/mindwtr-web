# Wiederkehrende Aufgaben

Verwenden Sie wiederkehrende Aufgaben für Arbeit, die nach ihrem Abschluss erneut ansteht: Durchsichten, Hausarbeiten, Routinen, Verlängerungen und Wartungsaufgaben.

Die Wiederholung wird im Aufgabeneditor festgelegt. Wenn Sie die Schnellerfassung verwenden, erstellen Sie zuerst die Aufgabe und öffnen Sie sie anschließend zur Bearbeitung.

## Eine wiederkehrende Aufgabe einrichten

1. Öffnen Sie die Aufgabe.
2. Bearbeiten Sie die Aufgabe.
3. Suchen Sie im Planungsabschnitt das Feld **Wiederholung**.
4. Wählen Sie **Täglich**, **Wöchentlich**, **Monatlich** oder **Jährlich**.
5. Passen Sie die Wiederholungsdetails an.
6. Speichern Sie die Aufgabe.

Die Wiederholungssteuerung erscheint auf dem Desktop und auf Mobilgeräten in der Aufgabenbearbeitung.

## Wiederholungsoptionen

| Option | Verwendung |
| --- | --- |
| **Täglich** | Gewohnheiten und tägliche Routinen. |
| **Wöchentlich** | Wochenrückblicke, Besprechungen, Besorgungen oder Wartung. |
| **Monatlich** | Rechnungen, monatliche Durchsichten, Abonnements oder monatliche Checklisten. |
| **Jährlich** | Jährliche Verlängerungen, Steuern, Registrierungen und Jahrestage. |

Bei jeder Wiederholungsart können Sie mit **Alle … wiederholen** Intervalle überspringen, zum Beispiel alle 2 Wochen oder alle 3 Monate.

Für wöchentliche Aufgaben lassen sich bestimmte Wochentage auswählen. Monatliche Aufgaben können am gleichen Tag jedes Monats oder über **Benutzerdefiniert** nach einem Muster wie einem bestimmten Wochentag wiederholt werden.

## Fester Zeitplan oder nach Abschluss

Standardmäßig behält Mindwtr einen festen Zeitplan bei. Verwenden Sie ihn für Fristen oder Verpflichtungen, die an den Kalender gebunden bleiben sollen.

Beispiele:

- Jeden Monat die Miete bezahlen.
- Am ersten Samstag die Finanzen prüfen.
- Jedes Jahr die Versicherung verlängern.

Aktivieren Sie **Nach Abschluss wiederholen**, wenn die nächste Aufgabe davon abhängen soll, wann Sie die aktuelle Aufgabe abschließen.

Beispiele:

- Pflanzen 3 Tage nach dem letzten Gießen erneut gießen.
- Filter 2 Wochen nach dem Abschluss erneut reinigen.
- 1 Monat nach der vorherigen Nachfrage erneut nachfragen.

Wenn bei einer Wiederholung nach Abschluss zusätzlich Wochentage oder ein Monatstag gewählt sind, fällt die nächste Aufgabe auf den ersten passenden Tag in der letzten Woche (oder im letzten Monat) des Intervalls. Wenn Sie „alle 2 Wochen am Sonntag“ an einem Dienstag abschließen, erscheint die Aufgabe am übernächsten Sonntag erneut, nicht am kommenden Sonntag.

Wenn Sie eine Aufgabe früher erledigt haben, sie aber erst später als erledigt markieren, schließen Sie sie stattdessen mit dem tatsächlichen Zeitpunkt ab (Rechtsklick auf die Abschlussschaltfläche auf dem Desktop oder langes Drücken auf **Erledigt** im mobilen Statusmenü). Das nächste Vorkommen wird dann ab dem tatsächlichen Abschlusszeitpunkt geplant.

## Eine Wiederholung beenden

Mit **Endet** legen Sie fest, wann die Serie endet:

| Einstellung | Bedeutung |
| --- | --- |
| **Nie** | Nach jedem Abschluss weiterhin die nächste Aufgabe erstellen. |
| **Am Datum** | Beenden, sobald das nächste Vorkommen nach dem ausgewählten Datum liegen würde. |
| **Nach** | Nach der ausgewählten Anzahl von Vorkommen beenden. |

## Kalendervorschau

Mindwtr hält für eine wiederkehrende Serie jeweils eine aktive Aufgabe vor. Die nächste tatsächliche Aufgabe wird erstellt, wenn Sie die aktuelle abschließen.

Aktivieren Sie **Nächstes Vorkommen im Kalender anzeigen**, wenn Sie eine Planungsvorschau des nächsten Vorkommens sehen möchten. Diese Vorschau hilft bei der Vorausplanung, ist aber keine zweite aktive Aufgabe.

## Was beim Abschluss geschieht

Wenn Sie eine wiederkehrende Aufgabe abschließen, erstellt Mindwtr die nächste Aufgabe mit demselben Titel, Projekt, Bereich, denselben Kontexten, Tags, Notizen, Erinnerungen und anderen Aufgabendetails. Checklistenpunkte werden für das nächste Vorkommen zurückgesetzt.

Besitzt die Aufgabe ein Fälligkeits-, Start- oder Wiedervorlagedatum, verschiebt Mindwtr diese Felder entsprechend der Wiederholungsregel. Reine Datumswerte bleiben reine Datumswerte.

Die nächste Aufgabe erscheint erst in „Fokus“ und unter „Nächste Aktionen“, wenn ihr Datum erreicht ist. Ein zukünftiges Startdatum stellt sie wie gewohnt zurück. Eine wiederkehrende Aufgabe ohne Startdatum wird bis zu ihrem nächsten Datum zurückgestellt (dem früheren von Fälligkeits- und Wiedervorlagedatum). Dadurch erscheint das gerade erstellte nächste Vorkommen nicht unmittelbar nach Abschluss der aktuellen Aufgabe wieder in der Liste. Zurückgestellte Vorkommen bleiben unter **Projekte** und in der **Suche** sichtbar.
