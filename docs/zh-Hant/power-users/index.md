# 進階使用者整合

Mindwtr 能與周遭工具連接，同時維持在地優先模式。本頁依你想完成的事情整理各項整合。每項整合都用四個問題，說明它需要你給予多少信任：

- 它能**讀取哪些**資料
- 它能**變更哪些內容**
- 它**在哪裡執行**
- **哪些資料會離開你的裝置**

這裡的所有整合都是選用功能，完成設定前一律保持關閉。完整說明請參閱各連結頁面。

## 隨處收集

### Apple 捷徑

從 Siri、「捷徑」app、動作按鈕或自動化，將任務送入收集箱，也能直接跳至 GTD 清單。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 無。它只會收集任務及開啟清單。 |
| 它能變更哪些內容 | 新增收集箱項目。無法編輯、完成或刪除任務。 |
| 它在哪裡執行 | 在你的 iPhone 或 iPad 上。 |
| 哪些資料會離開你的裝置 | 無。收集動作會經由 Mindwtr 的一般儲存及同步流程執行；Swift 絕不會直接寫入任務。 |

[Apple 捷徑指南](/zh-Hant/power-users/apple-shortcuts)

### 電子郵件收集

讓桌面版監看郵件資料夾，每封進入該資料夾的郵件都會成為收集箱任務。在手機上，則可將電子郵件分享至 Mindwtr。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 只有你指定的郵件資料夾；不會碰觸信箱的其餘部分。 |
| 它能變更哪些內容 | 建立收集箱任務。絕不會修改、移動或刪除郵件。 |
| 它在哪裡執行 | 在桌面電腦上，透過 IMAP 直接與郵件供應商通訊，沒有第三方中繼服務。 |
| 哪些資料會離開你的裝置 | 不會送往第三方。自行託管的作法只會將你所對應的欄位送至自己的伺服器。 |

[電子郵件收集指南](/zh-Hant/power-users/email-capture)

## 連接個人工具

### Obsidian

從 Obsidian vault 讀取任務，在檔案變更時保持更新，並能回到 Obsidian 開啟來源筆記。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 桌面版中，你所允許資料夾內的 vault 檔案。 |
| 它能變更哪些內容 | 對受支援格式切換核取方塊或 TaskNotes 狀態。不會重寫你的筆記。 |
| 它在哪裡執行 | 僅限桌面版。它不是 Mindwtr 的同步後端。 |
| 哪些資料會離開你的裝置 | 無。筆記會留在你的 vault 中。 |

[Obsidian 指南](/zh-Hant/power-users/obsidian)

## 在自己的電腦上自動化及編寫指令碼

### 本機 API

提供給指令碼、捷徑及小型工具的 REST API，讓它們讀寫你的任務。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 所有任務資料，以及專案與領域的唯讀存取權。 |
| 它能變更哪些內容 | 建立、更新、完成、封存、軟刪除及還原任務。專案與領域維持唯讀。 |
| 它在哪裡執行 | 在桌面版內部執行，繫結至 `127.0.0.1`，且每項要求都必須提供 bearer token。 |
| 哪些資料會離開你的裝置 | 無。要求只會在 localhost 上傳送。 |

[本機 API 指南](/zh-Hant/power-users/local-api)

## 連接 AI agent

### MCP 伺服器

讓 Claude、Codex 或 Gemini 等 AI 用戶端透過 Model Context Protocol 存取你的 Mindwtr 資料。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 任務、專案、分區、領域及人員。 |
| 它能變更哪些內容 | 預設為唯讀。使用 `--write` 後可新增、更新、完成及刪除。 |
| 它在哪裡執行 | 作為本機子程序，透過 SQLite 檔案運作；或連接你自行託管的 Cloud 伺服器。 |
| 哪些資料會離開你的裝置 | 只有你所連接的 AI 用戶端收到的資料。Cloud 模式下，編輯內容會透過經過驗證的 API 傳至你自己的伺服器。 |

[MCP 伺服器指南](/zh-Hant/power-users/mcp)

## 應用程式內的 AI

### AI 助理

選用的自備金鑰功能，可協助釐清及拆解任務，並檢視過時項目。

| 問題 | 答案 |
| --- | --- |
| 它能讀取哪些資料 | 你所執行動作需要的任務資料，不會超出所需範圍。 |
| 它能變更哪些內容 | 只有你已檢視並核准的內容。 |
| 它在哪裡執行 | 透過你設定的服務供應商，也可以是 llama.cpp 或 Ollama 等本機端點。 |
| 哪些資料會離開你的裝置 | 只將限定範圍的資料送至你選擇的供應商。若供應商在本機執行，則不會送出任何資料。 |

[AI 助理指南](/zh-Hant/power-users/ai-assistant)
