# 每日回顾

每日回顾是一套轻量的重整流程，帮助你在两次每周回顾之间保持现状清晰。桌面端可在“回顾”视图中使用，移动端则有单独的每日回顾页面。

重点不在于把列表收拾得整齐。花几分钟让清单上的内容与当下真实情况重新一致，之后一整天你都可以信赖清单，而不必在脑中惦记尚未闭合的事项。

## 流程

默认顺序如下：

1. 今天与日历
2. 收集箱，仅在其中存在当前任务时显示
3. 等待中
4. 今日专注（启用时）
5. 完成

收集箱和“等待中”有意排在专注之前：先明确新记录的事项；你等待的事项解除阻塞后，也可以切换为“下一步”，这样选择今日专注时就能把它们纳入候选。空步骤会自动跳过。

## 设置

专注步骤由以下设置控制：

```typescript
settings.gtd.dailyReview.includeFocusStep
```

桌面端和移动端都在 GTD 设置中将其显示为“包含专注步骤”。该步骤默认显示。将值设为 `false` 即可使用更短的“今天/收集箱/等待中”流程。

相关设置：

| 设置 | 用途 |
| --- | --- |
| `settings.gtd.defaultScheduleTime` | 预填手动安排时间字段。 |
| `settings.weekStart` | 控制日历中的每周布局。 |
| `settings.calendar.viewMode` | 保存应用其他位置所用的日历视图模式。 |

## 桌面端行为

回顾弹窗进行期间，桌面端会把当前步骤保存在本地存储中。关闭并重新打开弹窗会从同一步骤继续；完成回顾后则会清除保存的步骤。

回顾中的任务行使用常规的快速操作和任务详情功能，包括可用时的领域与项目操作。

## 移动端行为

移动端使用同一个 GTD 设置控制专注步骤，并将回顾呈现为独立页面。它与桌面端流程的各步骤意图相同，但采用移动端专用的导航控件。
