# 移动端用户指南

Mindwtr 移动应用使用 React Native 和 Expo 构建。Android 已获得完整支持；iOS 版可从 App Store 获取，也可通过 TestFlight 测试版使用。

## 概览

移动应用使用底部标签页承载核心流程，并通过“菜单”页面提供其他视图。

---

## 交互方式

- **轻点**以打开并编辑任务。
- **滑动**以执行快捷操作（请参阅下方的“滑动操作”一节）。
- **共享表单**可将条目直接添加到收集箱。

## 导航

### 底部标签页

| 标签页            | 说明                          |
| -------------- | ------------------------------------ |
| 📥 **收集箱**    | 收集并处理传入的条目   |
| 🎯 **专注**    | 每日仪表板和下一步行动     |
| ➕ **快速收集** | 快速添加任务或音频笔记 |
| 📝 **回顾**  | 每日 + 每周回顾                |
| ☰ **菜单**     | 访问项目、看板、日历等。 |

### 菜单标签页

轻点**菜单**标签页可访问其他视图：

 - 📋 **看板**：支持手动拖动排序、搜索以及情境/标签/日期/项目筛选的看板视图
 - 🗓️ **日历**：基于时间的视图
 - 📁 **项目**：需要多个步骤才能达成的成果
 - 🏷️ **情境**：按情境筛选
 - ⏳ **等待中**：已委派的条目
 - 💭 **将来/也许**：未来的想法
 - 📚 **参考资料**：参考材料
 - ✅ **已完成**：最近完成的任务
 - 📦 **已归档**：已完成且从常规列表中收起的任务和项目
 - 🗑️ **废纸篓**：已删除的任务和项目
 - ⚙️ **设置**：应用偏好设置

打开**已完成**并轻点**选择**以挑选任务，然后使用**移动 → 已归档**提前将其归档。在**已归档**中，轻点**选择**或**全选**，可将任务移回“已完成”且不更改完成时间、恢复到收集箱，或移至废纸篓。切换到**项目**即可打开、恢复归档项目，或将其移至废纸篓。Mindwtr 仅从废纸篓中永久删除条目。

打开**废纸篓**并轻点**选择**，即可管理多个已删除的任务和项目。轻点**全选**或选择条目，然后一步完成恢复或永久删除。永久删除始终会要求确认。

## 全局搜索

轻点标题栏中的**搜索图标**以打开全局搜索。

搜索默认隐藏已完成和已归档条目。当隐藏条目与查询匹配时，会出现可轻点的“N 个其他结果位于已完成和已归档中”提示。轻点该提示即可包含它们，或在搜索筛选器中启用**包含已完成和已归档的任务**。

### 搜索运算符

使用运算符进行强大的筛选：

| 运算符    | 示例            | 说明             |
| ----------- | ------------------ | ----------------------- |
| `status:`   | `status:next`      | 按任务状态筛选   |
| `-status:`  | `-status:done`     | 排除某个状态        |
| `context:`  | `context:@home`    | 按情境筛选       |
| `tag:`      | `tag:#focused`     | 按标签筛选           |
| `assigned:` | `assigned:Tom`     | 按受托人筛选      |
| `project:`  | `project:HomeReno` | 按项目筛选       |
| `location:` | `location:office`  | 按任务地点筛选 |
| `where:`    | `where:office`     | 任务地点的别名 |
| `id:`       | `id:abc123`        | 查找确切的任务 ID   |
| `-id:`      | `-id:abc123`       | 排除确切的任务 ID |
| `due:`      | `due:today`        | 在某日到期的任务       |
| `due:<=`    | `due:<=7d`         | 7 天内到期的任务 |
| `start:`    | `start:>=tomorrow` | 从某日开始的任务 |
| `created:`  | `created:>=30d`    | 最近 30 天内创建的任务 |
| `OR`        | `@home OR @work`   | 匹配任一条件  |

### 已保存的搜索

已保存的搜索是可重复使用的搜索查询快捷方式。

1. 输入搜索查询
2. 轻点**“保存搜索”**
3. 为搜索命名
4. 从**菜单 → 已保存的搜索**访问

**删除方法：**打开已保存的搜索，然后轻点标题栏中的废纸篓图标。

已保存的搜索与“专注”中的**已保存筛选器**相互独立。专注筛选器会为专注视图保存项目、情境、标签、优先级、精力和预计用时等条件。

## 快速收集

Mindwtr 提供多种在移动端快速收集任务的方式。

收集界面以输入为先。语法帮助收在一个小巧的“?”开关后方，以保持界面简洁。

需要连续收集时，请打开收集表单中的**继续添加**开关：每次按 Enter 都会保存任务，同时保持表单打开和键盘弹出，以便立即输入下一项。该开关会在多次收集之间保持开启，直到你将其关闭。

### 共享表单

使用共享表单从任何应用收集任务：

1. 在任何应用（浏览器、电子邮件、笔记）中，找到想要收集的内容
2. 轻点**共享**按钮
3. 从共享选项中选择 **Mindwtr**
4. Mindwtr 会打开收集界面，并将共享内容附加为笔记
5. 添加标题、调整任何字段，然后将其保存到收集箱

共享文件（PDF、图像、视频或录音）会将其复制到 Mindwtr 中，作为新任务的附件，并以文件名作为初始标题。一次最多可共享六个文件。

特别适合：
- 保存文章供稍后阅读
- 将电子邮件收集为任务
- 添加网页浏览中的链接
- 将 PDF 或照片归入收集箱任务，供稍后处理

### 主屏幕小组件

将 Mindwtr 小组件添加到主屏幕，以便快速访问：

1. 长按主屏幕
2. 选择**小组件**
3. 找到并添加 **Mindwtr** 小组件
4. 轻点小组件以打开快速收集或查看专注条目

### iOS 锁定屏幕小组件

在 iOS 16 及更高版本中，Mindwtr 提供锁定屏幕小组件，无需解锁手机即可显示当前专注任务：

1. 长按锁定屏幕，然后轻点**自定**
2. 选择小组件区域，并添加一个**Mindwtr 专注**小组件
3. 选择矩形小组件（顶部专注任务并带有“专注”说明文字）、时钟上方的单行内联小组件，或圆形专注任务计数器

轻点其中任意一个都会打开专注界面。锁定屏幕小组件会遵循系统的单色锁定屏幕样式。

### Android 快速设置磁贴

在 Android 上，将 Mindwtr 收集磁贴添加到快速设置，即可通过一次滑动收集到收集箱：

1. 打开 Android 快速设置编辑模式。
2. 添加 **Mindwtr** 磁贴。
3. 轻点磁贴以打开快速收集。

### Android 通知栏快速收集

如需最快的收集方式，只需滑动一次且可从锁定屏幕使用，请固定一条常驻的**快速收集**通知：

1. 打开**设置 → 通知**。
2. 开启**通知栏中的快速收集**。
3. 随时轻点通知以打开快速收集。

该通知不会发出声音，并会安静地停留在通知栏底部。关闭开关即可移除。（仅限 Android；iOS 不支持常驻通知，因此请在那里使用小组件或“快捷指令”。）

### Android 语音应用操作

Android 构建版本会向受支持的助理公开收集操作，包括通过 Android App Actions 路由的 Gemini/Assistant 界面。语音创建的收集内容会打开 Mindwtr 的确认流程，以便你在保存前检查标题和笔记。

### Android 情境自动化 Intent

Tasker、MacroDroid 或 Phone Profiles 等自动化应用可以激活 Mindwtr 中的情境。对于仅在后台触发的操作，请使用 Android 广播形式。激活后，Mindwtr 会检查当前可用且匹配的 `/next` 行动，并且仅在有内容可显示时发送通知。轻点该通知会打开匹配的“情境”视图。

Android 广播形式：

| 字段 | 值 |
| --- | --- |
| Package | `tech.dongdongbh.mindwtr` |
| Class | `tech.dongdongbh.mindwtr.ContextAutomationReceiver` |
| Target | Broadcast Receiver |
| Activate action | `tech.dongdongbh.mindwtr.action.ACTIVATE_CONTEXT` |
| Deactivate action | `tech.dongdongbh.mindwtr.action.DEACTIVATE_CONTEXT` |
| String extra | `context=parents` or `context=@parents` |

ADB 示例：

```bash
adb shell am broadcast -n tech.dongdongbh.mindwtr/.ContextAutomationReceiver -a tech.dongdongbh.mindwtr.action.ACTIVATE_CONTEXT --es context parents
adb shell am broadcast -n tech.dongdongbh.mindwtr/.ContextAutomationReceiver -a tech.dongdongbh.mindwtr.action.DEACTIVATE_CONTEXT --es context parents
```

URL 形式：

| URL | 操作 |
| --- | --- |
| `mindwtr://contexts?token=%40parents&contextAction=activate` | 激活 `@parents` |
| `mindwtr://contexts?token=%40parents&contextAction=deactivate` | 停用 `@parents` |

URL 示例：

```bash
adb shell am start -a android.intent.action.VIEW -d 'mindwtr://contexts?token=%40parents&contextAction=activate' tech.dongdongbh.mindwtr
```

注意：
- URL 启动可能会打开 Mindwtr。当自动化应留在后台时，请使用广播接收器形式。
- 情境名称会规范化为 `@context`，因此 `parents` 和 `@parents` 都会匹配 `@parents`。
- 分层情境会向下匹配所选情境，因此 `@parents` 也会匹配 `@parents/errands`。
- 如果没有当前可用的 `/next` 行动与情境匹配，Mindwtr 会保持静默。
- 停用目前是一项静默的空操作。它会确认自动化退出触发器，并为未来的活动情境状态预留；它不会删除、隐藏或更改任务。
- 在 Android 上，情境自动化 URL 和 Intent 处理完毕后会将 Mindwtr 送回后台。想要打开匹配的“情境”视图时，请轻点通知。
- Mindwtr 本身不检测地点或设备状态；触发器由自动化应用负责。

### Apple 快捷指令

在 iPhone 和 iPad 上，Mindwtr 提供原生 Apple 快捷指令操作，用于收集到收集箱以及打开专注、等待中、将来、项目、回顾和日历等 GTD 视图。有关设置、示例和 v1 限制，请参阅 [Apple 快捷指令](/zh-Hans/power-users/apple-shortcuts)。

### URL Scheme 快速收集（iOS 快捷指令 / Android 自动化）

Mindwtr 注册了 URL scheme `mindwtr://`，因此你可以从 iOS 快捷指令、Tasker 或其他自动化工具收集任务。

支持的 URL：

| URL | 操作 |
| --- | --- |
| `mindwtr://capture?title=Buy%20groceries` | 打开带标题的收集界面 |
| `mindwtr://capture?title=Buy%20groceries&note=From%20store` | 打开带标题和笔记的收集界面 |
| `mindwtr://capture?title=Buy%20groceries&project=Shopping&tags=errands,home` | 打开带项目和标签的收集界面 |

注意：
- `title` 为必填项（别名：`text`）。
- `note` 为可选项（别名：`description`）。
- `project` 按标题匹配现有的活动项目（不区分大小写），或创建该项目。
- `tags` 以逗号分隔，并在保存前规范化为 `#tag` 格式。

iOS 快捷指令示例：
1. 打开**快捷指令**并创建一个快捷指令。
2. 添加**要求输入**（提示：任务标题）。
3. 添加**打开 URL**，内容为：`mindwtr://capture?title=[Provided Input]`。
4. 运行快捷指令；Mindwtr 会打开收集界面，以便你检查并保存任务。

### 快速添加语法

Mindwtr 会在添加任务时解析自然语言：

| 语法       | 示例           | 结果             |
| ------------ | ----------------- | ------------------ |
| `@context`   | `@home`           | 添加情境       |
| `#tag`       | `#focused`        | 添加标签           |
| `+Project`   | `+HomeReno`       | 分配到项目 |
| `+Multi Word` | `+New Project`    | 分配到 "New Project" |
| `+"Quoted Name"` | `+"New Project" call Bob` | 引号界定句子中间的多词名称（也支持 `!"Area Name"`） |
| `!Area`       | `Plan roadmap !Work` | 分配到领域       |
| `%Person`     | `Ask %Jim for budget` | 设置受托人（委派 / 等待中的负责人）；新建多词名称使用 `%"Full Name"` |
| `/area:<name>` | `/area:Personal` | 分配到领域（不含空格） |
| `/due:date`  | `/due:friday`     | 设置截止日期      |
| `/energy:<level>` | `/energy:medium` | 设置精力水平（`low`、`medium`、`high`） |
| `/note:text` | `/note:call back` | 添加描述   |
| `/status`    | `/next`, `/waiting`, `/someday`, `/done`, `/archived`, `/inbox` | 设置状态 |

**日期格式：** today、tomorrow、friday、next week、in 3 days

标题中不带令牌的日期短语（如 "next week"）会被转换为日期，这由**识别自然语言日期**（设置 → GTD，默认开启）控制。关闭后，这类短语会保留为标题文字；显式令牌（如 `/due:friday`）始终有效。该设置会在设备间同步。

## 音频收集

使用由 AI 驱动的转写功能，通过语音收集任务。

### 设置

1. 前往**菜单 → 设置 → AI 助手**。
2. 启用**语音转文字**。
3. 选择一个**提供商**：
   - **OpenAI / Gemini**：基于云端（需要 API 密钥）。
   - **离线（Whisper）**：在本地运行。你可以直接在设置中下载模型（例如 Tiny 或 Base）。
4. 如果偏好音频优先，请在**设置 → 常规**中设置**默认收集方式**。

### 使用音频收集

- **快速添加**：轻点快速收集界面中的**音频**标签页。
- **录音**：轻点麦克风以开始。
- **转写**：停止录音以处理音频。
- **智能解析**：启用后，应用会自动提取日期和字段。

## 收集箱

用于快速输入任务的收集区域。

当你希望通过针对工作、家庭、人物、跑腿事项和未闭环事务的引导提示进行收集，而不是从空白输入框开始时，请使用**思绪清扫**。

### 添加任务

1. 轻点底部的输入字段
2. 使用其他应用的共享表单
3. 轻点主屏幕小组件
4. 使用快速添加语法输入任务
5. 轻点添加按钮或按 Enter

### 处理收集箱

轻点**处理收集箱**以开始明确流程：

1. **这可行动吗？**
   - 是 → 继续
   - 否 → 移至废纸篓或将来/也许

2. **能否在 2 分钟内完成？**
   - 是 → 立即执行，标记为已完成
   - 否 → 继续

3. **应该由谁执行？**
   - 我来执行 → 添加情境，移至下一步行动
   - 委派 → 移至等待中

4. **你会在哪里执行？**
   - 选择情境（@home、@work 等）
   - 添加自定义情境

5. **分配到项目？**（可选）
   - 选择项目或跳过
   - 或者，当收集内容包含多个行动时，选择**是，将其设为项目**：为项目命名并定义其下一步行动。你随后输入的其他行动都会回到收集箱，并已附加该项目，准备各自完成明确流程

## 专注

用于执行工作的主要仪表板。“专注”是一个执行仪表板，并非状态为 `next` 的所有任务的完整清单。

### 分区

| 分区      | 内容                                                                 |
| ------------ | ----------------------------------------------------------------------- |
| **今天**    | 今天专注、今天到期/已逾期或今天开始的任务           |
| **下一步**     | 未被阻塞或推迟的可用下一步行动                 |

“专注”会隐藏未来才开始的任务和顺序项目中靠后的任务，从而将列表限制在你现在可以采取行动的内容上。需要查看更广泛的任务清单时，请使用**情境**、**项目**或**搜索**。

下一步行动的默认顺序为：即将到期的行动优先，无日期的行动其次，截止日期在很久以后的行动最后。在同一分组内，启用优先级时，“专注”会依次使用优先级、开始时间、创建日期、标题和 id 排序。完整逻辑请参阅[Mindwtr 中的 GTD 工作流](/zh-Hans/use/gtd-workflow#焦点如何对可执行行动排序)。

### 功能

- **情境筛选器**：轻点情境标签以筛选“下一步”列表。
- **已保存筛选器**：保存可重复使用的专注条件，例如项目、情境、标签、优先级、精力水平和预计用时。
- **滑动以专注**：向右滑动任务以切换“专注”状态（将其移至“今天”）。
- **快速状态**：轻点状态徽章以更改状态。
- **番茄钟（可选）**：在**设置 → GTD → 功能 → 番茄计时器**中启用，以显示紧凑的专注/休息计时器。将其保留为**仅计时器**，或开启**将计时器关联到任务**以显示计时器任务选择器和**将任务标记为已完成**操作。

## 回顾

回顾任务并更新其状态。

- 查看任务详细信息（描述、开始时间、截止日期、情境）
- 快速将任务标记为已完成
- 在任务之间导航
- **选择模式**：批量整理、移动、添加标签、共享或删除任务
- 在每周回顾期间，使用**处理收集箱**步骤清空已收集条目，然后再继续进行日历、等待中、项目和将来回顾。

## 任务编辑器（任务 + 查看）

任务编辑器有两种模式：

- **任务**：编辑字段、清单、日期、标签和情境
- **查看**：简洁的只读摘要，清单可轻点

向左/向右滑动可在**任务**和**查看**之间切换。

以清单为主的任务默认使用“查看”模式，以便更快勾选。

编辑器初始保持简洁。轻点**更多选项**以显示高级字段；已有内容的任何字段都会保持可见。

描述 Markdown 支持无序列表和任务复选框（`- item`、`[ ] item`、`[x] item`）。
在任务描述或项目笔记中输入 `[[`，可从链接选择表单中链接另一个任务或项目。
这些链接仅用于导航；不会在任务之间同步完成状态。
描述中的 Markdown 复选框仅作为笔记；不会影响任务清单。
将多行文本粘贴到清单条目中会按行创建条目（可识别项目符号、编号和 `[x]` 标记）。
**受托人 / 人物**字段存储用于“等待中”、建议和 `assigned:` 搜索的受托人物。可从**设置 → 管理**中管理已保存的人物、笔记和参考链接。

重复任务支持两种策略：
- **严格**（固定周期）
- **完成后重复**（下一日期从完成时间起算）
- **结束：永不 / 指定日期 / N 次后**

Mindwtr 会保留重复任务的一个活动实例。当该实例有截止日期或开始时间时，“日历”会显示它；除非启用**在日历中显示下一次发生**以进行仅供规划的预览，否则在当前实例完成前，不会预先填充未来发生项。

使用任务编辑器中的重复字段，然后根据需要切换**完成后重复**或**在日历中显示下一次发生**。
同一表单还允许你在目标日期或达到固定的总发生次数后停止系列。

### 附件

你可以从编辑器向任务附加文件或链接。启用**保存音频附件**后，音频笔记可保存为附件。

有关同步和清理的详细信息，请参阅[附件](/zh-Hans/use/attachments)。

## 日历集成

Mindwtr 可以叠加外部日历，并将 Mindwtr 中带日期的任务推送到设备日历。详细设置请参阅[日历集成](/zh-Hans/use/calendar-integration)。

要将任务推送到 Android/Google Calendar 或 iOS 上的 Apple Calendar：

1. 前往**设置 -> 日历**

3. 授予日历权限
4. 展开**同步目标**
5. 选择专用的 `Mindwtr` 日历或其他可写日历目标

有关 Android Google Calendar 和 iOS Apple Calendar 目标设置，请参阅[日历集成](/zh-Hans/use/calendar-integration)。

要使用 ICS 订阅叠加外部日历：

1. 前往**设置 -> 日历**
2. 添加你的 **ICS URL**
3. 刷新以获取事件

外部事件仅供查看，不会同步回其来源。轻点外部事件并选择**创建任务**，即可在 Mindwtr 中创建一项独立任务；Mindwtr 会复制事件的标题、日期/时间、地点、描述以及日历名称（如有）。

## 日历

具备日程安排功能的基于时间的视图。

### 视图

- **月视图**：带截止日期任务的概览
- **日视图**：包含已安排任务和外部事件的详细时间线

### 安排任务

1. 在日历的日视图中，轻点**安排任务**
2. 从下一步行动（优先显示）中选择，或搜索待办任务
3. Mindwtr 会找到最早的空闲时段（避开与外部事件的冲突）
4. 任务会根据其预计用时获得开始时间

安排面板是移动端的规划界面：当你正在查看某一天，并希望将未安排的下一步行动或已到期但未安排的工作转化为具体时间块时，请使用它。想专注于日历网格时，可将其折叠。

### 拖动以重新安排

- 长按已安排的任务块
- 拖动到新的时段（吸附到 5 分钟间隔）
- 松开以更新开始时间

### 外部日历（iCal/ICS）

订阅外部日历，以便在任务旁查看事件：

1. 前往**设置 → 高级 → 日历**
2. 输入日历 URL（ICS/webcal 格式）
3. 为其命名，然后轻点**添加**
4. 外部事件会在日视图中显示为灰色块

## 项目

管理需要多个步骤才能达成的成果。

从**菜单 → 项目**打开项目。

### 项目列表

- 查看所有活动项目
- 查看每个项目的任务数
- 轻点以查看项目详细信息

### 项目详细信息

- 查看项目中的所有任务
- 添加新任务
- 在项目内使用**分区**对任务进行分组。分区是一个项目内部的标题，而不是子任务或独立项目。
- 在任务编辑器中轻点任务，为其分配**分区**
- 编辑项目设置（名称、颜色、笔记）
- 分配**关注领域**（例如工作、个人）
- 添加**项目标签**以供筛选
- 设置顺序或并行模式
- 设置回顾日期
- 启用自定义排序后，使用拖动手柄重新排列项目任务。将任务放到不同的分区标题下会把它移入该分区，将它放到第一个标题上方则会清除其分区
- 归档项目（项目通过这种方式完成——剩余任务会随之完成，而**重新激活**可撤销此操作）

任务编辑器中的**项目分区**字段会将任务分配到其当前项目的某个分区。只有在任务属于具有分区的项目后，该字段才有意义；否则请将其留空。

### 顺序与并行

| 模式           | 行为                                             |
| -------------- | ---------------------------------------------------- |
| **顺序** | 只有项目中第一个可用任务会出现在“专注”中 |
| **并行**   | 所有可用的项目任务都可以出现在“专注”中        |

顺序项目可以按整个项目运行，也可以逐分区运行。按分区排序时，每个项目分区都会显示第一个可用任务，因此不同阶段或工作流可以并行推进，而无需显示所有任务。

## 滑动操作

使用滑动手势快速管理任务：

| 视图             | 向右滑动 | 结果             |
| ---------------- | ----------- | ------------------ |
| **收集箱**        | 完成        | 将任务标记为已完成 |
| **专注**        | 专注       | 切换专注状态 |

完成任务时会记录当前时间。如果实际上更早完成，请长按任意**完成**控件以选择真实完成时间：滑动后显示的“完成”按钮、状态菜单中的“完成”选项、任务编辑器中的“完成”标签，或任务预览中的状态徽章。之后也可以轻点已完成或已归档任务上的**完成时间**时间戳进行更正。

## 通知与提醒

Mindwtr 会发送通知，帮助你保持进度。

### 通知类型

- **截止日期提醒**：任务到期时发出提醒
- **开始时间提醒**：提醒你何时该开始
- **重复任务提醒**：重复条目的通知

轻点通知正文可直接跳转到**回顾**界面。

提醒在手机本身进行安排，不使用推送服务器。如果你在另一台设备上完成或重新安排任务（例如，在桌面端勾选每日重复任务），该更改一经同步到手机，手机就会立即更新其闹钟，因此请让手机在下次提醒时间之前完成同步（打开应用一次即可）。

### 权限

确保已启用通知：
1. 前往设备的**设置 → 应用 → Mindwtr**
2. 启用**通知**
3. 根据需要允许提醒和声音

## 设置

### 常规

- **外观**：跟随系统、浅色或深色
- **语言**：英语、简体中文、繁体中文、印地语、西班牙语、阿拉伯语、法语、葡萄牙语、俄语、德语、日语、越南语、土耳其语、韩语、意大利语、波兰语、荷兰语、捷克语

### 通知

**任务提醒：**
- 启用/禁用截止日期和开始时间通知

**每日摘要：**
- **晨间简报**：汇总今天到期、已逾期和专注任务
- **晚间回顾**：提示回顾并结束一天
- 分别配置时间

**每周回顾：**
- **提醒**：获取开始回顾的每周通知
- **时间/日期**：自定义希望回顾的时间（例如周五下午 4 点）

### GTD

自定义 Mindwtr 如何配合你的 GTD 工作流：

**功能（可选）：**
- **优先级**：在任务上显示优先级标志
- **预计用时**：添加用于时间块安排的时长字段

**预计用时预设：**
- 选择任务编辑器中显示哪些预计用时
- 选项：5m、10m、15m、30m、1h、2h、3h、4h、4h+
- 默认：10m、30m、1h、2h、3h、4h、4h+

**自动归档：**
- 在设定天数后自动将已完成任务移至已归档（默认：7 天）。缩短延迟时间会立即归档现有“已完成”任务中已超过新期限的任务。
- 设置为“永不”，可将完成项无限期保留在“已完成”列表中

**收集箱处理：**
- 移动端保留基于卡片的收集箱处理流程
- 相同的共享设置可以隐藏或显示 2 分钟快捷方式、项目优先提示、情境/标签分区、日程安排分区和参考资料选项

**任务编辑器布局：**
- 轻点字段以切换可见性（隐藏字段在有值时仍会显示）
- 长按拖动手柄以重新排列字段
- 在**基本**、**日程安排**、**组织**和**详细信息**等分区之间移动字段
- 选择默认展开哪些可折叠分区
- 隐藏字段可通过编辑器中的**更多**按钮显示

**管理：**
- 使用**设置 → 管理**编辑已保存的**领域**、**人物**、**情境**和**标签**
- 这里是清理重复项或重命名可重复使用元数据的最快位置
- 有关在分配时从何处创建领域和人物，请参阅[领域和人物](/zh-Hans/use/areas-people)

### 数据与同步

有关同步设置，请参阅[数据与同步](/zh-Hans/data-sync/)。

**同步后端：**
- **云同步**：受支持的构建版本中使用 Dropbox，iOS 上还可在可用时使用 iCloud
- **文件夹 / 文件同步**：通过共享 JSON 文件/文件夹进行文件同步（Google Drive、Syncthing、OneDrive 等）
- **高级 / 自定义服务器**：WebDAV 或自托管 Mindwtr Cloud

**其他选项：**
- **同步**：手动触发同步
- **上次同步状态**：查看数据上次同步的时间
- **同步历史**：默认折叠；轻点以展开最近条目
- **导出备份**：将数据保存到文件
- **Apple 提醒事项导入**：选择一个“提醒事项”列表，并将未完成的提醒事项导入收集箱。已导入的提醒事项仍会保留在 Apple 提醒事项中，且已导入、已完成或无标题的提醒事项会被跳过。
- **设置同步选项**：选择哪些偏好设置跨设备同步（主题、语言/日期格式、GTD 默认值、外部日历 URL、AI 设置和已保存筛选器）。API 密钥和本地模型路径永远不会同步。

**GTD 选项：**
- **专注任务上限**：选择可标记为今日专注的任务数量。

### 高级

**AI 助手：**
- 可选的 BYOK 助手，用于明确和拆解任务

**日历（ICS/iCal）：**
- **添加日历**：输入名称和 URL
- **启用/禁用**：切换每个日历的可见性
- **移除**：删除订阅
- **测试**：验证日历能否正确加载
