# 番茄专注

Mindwtr 在桌面端和移动端的**专注**视图中提供可选的番茄计时器面板。

该功能默认**关闭**，以保持专注界面简洁、无干扰。

---

## 启用番茄计时器

### 桌面端

1. 打开**设置**。
2. 前往 **GTD**。
3. 在**功能**下开启**番茄计时器**。
4. 打开**专注**视图。

### 移动端

1. 打开**菜单 → 设置**。
2. 前往 **GTD**。
3. 在**功能**下开启**番茄计时器**。
4. 打开**专注**标签页。

---

## 工作方式

- 可以从当前专注候选中选择一项**计时任务**，也可以保留为**仅计时器**。
- 选择预设：`15/3`、`25/5`、`50/10`，或在**设置 → GTD**中配置的一项可选自定义预设。
- 开始专注时段。
- 专注时段结束后，Mindwtr 会切换到休息并暂停，让你主动决定何时开始。
- 休息结束后，会切换回专注。

任务关联是可选功能，默认关闭。如果希望在专注界面中显示计时任务选择器和**将任务标记为完成**操作，请开启**设置 → GTD → 番茄计时器 → 将计时器关联到任务**。

桌面端可以折叠面板：点击标题栏的 V 形按钮，面板会缩成只显示剩余时间和阶段的细行，计时器仍继续运行。系统会记住折叠状态；按任务的播放按钮会重新展开面板。

控件：

- **计时器：**开始/暂停、重置、切换阶段
- **任务更新：**将任务标记为完成，即完成所选计时任务

任务选择器决定番茄面板关联哪项任务（如果有）。计时器控件只影响计时器。只有关联计时任务时才会显示**将任务标记为完成**；该操作会把所选任务移至已完成，并从今日专注中移除。

## 已用时间

任务编辑器中，“时间预估”旁有可选的**已用时间**值。已用时间需要 v1.1.0 或更高版本；v1.0.5 等早期版本没有此功能。

已用时间需主动启用，默认关闭：只有同时启用**番茄计时器**和**将计时器关联到任务**时，相关界面才会出现，包括桌面端和移动端的编辑器字段与任务行徽标、移动端的完成对话框，以及桌面端的快速开始按钮。关闭后，所有位置都不显示时间相关内容（已记录的值会保留，重新启用后再次出现）。在任务编辑器布局中隐藏“时间预估”也会一并隐藏“已用时间”。

- 每次完成与计时任务关联的专注时段，都会把该时段的专注分钟数加到任务总计中。
- 也可在编辑器中手动设置或修正总计。
- 有已记录时间的任务会在桌面端和移动端任务列表中显示小徽标，总计会像其他任务字段一样跨设备同步。

该功能有意不提供逐次时段日志、报告界面或自由运行的秒表。每项任务只有一个总数。如需外部报告，可通过 Cloud API 或 MCP 服务器读取 `timeSpentMinutes`。

## 从任务开始时段（桌面端）

同时启用番茄功能和**将计时器关联到任务**后，将鼠标悬停在任务行上会显示播放按钮：

- 点击即可关联该任务并立即开始专注时段，无需在计时任务下拉列表中查找。
- 按钮还会显示你已为该任务完成多少个专注时段。

## 专注与下一步行动

**专注**视图并非完整“下一步行动”列表的一比一副本，而是一块执行仪表板：

- **今日专注：**你明确标记为今天处理的任务，不超过已配置的专注数量上限
- **今天：**今天截止、已逾期或今天开始的下一步任务
- **下一步行动：**当前可执行的下一步任务
- **待回顾：**需要留意的等待中/提醒事项

专注会有意隐藏未来开始的任务，以及顺序项目中靠后的任务，让视图只保留当下可以执行的工作。如需查看更广泛的任务清单（包括未来开始的任务），请使用**情境**、**项目**或**搜索**。

---

## 何时使用

- 想在不离开当前 GTD 专注流程的情况下使用轻量计时器时启用。
- 如果希望专注保持纯列表界面、不显示计时器，则保持关闭。
- 把它当作执行下一步行动的辅助工具，而不是另一套规划系统。

## 实用模式

- 用 `15/3` 清理收集箱、处理小型事务或打破拖延。
- 用 `25/5` 作为普通下一步行动的默认日常节奏。
- 当项目任务已足够明确、可以开始时，用 `50/10` 进行深度工作。
- 如果需要不同节奏，可使用一项自定义预设，但请保持简单。
- 只有关联的计时任务确实完成时才使用**将任务标记为完成**；否则请暂停或切换阶段，让任务继续保留。

---

## 说明

- 番茄计时器可作为普通计时器运行；如果希望直接从面板完成任务，也可关联一项计时任务。
- 面板有意设计为需主动启用，让偏好纯净 GTD 专注页面的用户可以保持隐藏。
- 内置预设固定且简单。Mindwtr 只允许增加一项可选自定义预设，避免专注界面变成计时器设置页。
