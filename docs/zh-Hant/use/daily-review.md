# 每日回顧

每日回顧是在兩次每週回顧之間，用來掌握現況的輕量重整流程。桌面版可在「回顧」檢視中找到，行動版則有專屬的每日回顧路徑。

重點不是把清單整理得好看。花幾分鐘消除清單內容與當下真實狀況之間的落差，你便能在一天剩餘時間裡信任清單，不必把尚未結束的事情全放在腦中。

## 流程

預設順序如下：

1. 今天與行事曆
2. 收集箱（僅在有尚未處理的收集箱任務時顯示）
3. 等待中
4. 今日專注（已啟用時）
5. 完成

收集箱與「等待中」刻意排在專注之前：先釐清新收集的事項；原先等待的項目若已解除阻礙，也可改為「下一步」，成為挑選今日專注項目時的候選。沒有內容的步驟會自動略過。

## 設定

專注步驟由以下設定控制：

```typescript
settings.gtd.dailyReview.includeFocusStep
```

桌面版與行動版都在 GTD 設定中將它顯示為「包含專注步驟」。此步驟預設顯示；將值設為 `false`，即可使用較短的「今天／收集箱／等待中」流程。

相關設定：

| 設定 | 用途 |
| --- | --- |
| `settings.gtd.defaultScheduleTime` | 預先填入手動排程欄位。 |
| `settings.weekStart` | 控制行事曆一週的版面配置。 |
| `settings.calendar.viewMode` | 儲存應用程式其他位置使用的行事曆檢視模式。 |

## 桌面版行為

回顧對話框進行期間，桌面版會把目前步驟保存在本機儲存空間。關閉後再開啟對話框，會從同一步驟繼續；完成回顧則會清除已儲存的步驟。

回顧中的任務列沿用一般的快速動作與任務詳情操作，包括可用時顯示的領域與專案動作。

## 行動版行為

行動版與桌面版使用相同的 GTD 設定控制專注步驟，並將回顧呈現為路徑層級的畫面。兩者各步驟的目的相同，但行動版使用適合行動裝置的導覽控制項。
