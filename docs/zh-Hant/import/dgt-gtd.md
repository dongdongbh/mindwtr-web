# 匯入 DGT GTD

如水可匯入 DGT GTD 匯出資料，讓你不必手動重建系統即可遷移。

支援的來源：

- DGT GTD 的 **JSON** 匯出檔
- 含匯出 JSON 檔案的 DGT GTD **ZIP** 封存檔

桌面版與行動版都可從**設定 → 資料 → 從 DGT GTD 匯入**執行。

---

## 匯入內容

如水以 GTD 優先的原則，將 DGT GTD 匯出資料對應至如水的模型：

- DGT 資料夾成為**如水領域**
- DGT 專案成為**如水專案**
- DGT 檢查清單成為**檢查清單任務**
- 檢查清單項目仍為**檢查清單項目**
- DGT 情境成為**情境**
- DGT 標籤成為**標籤**

若如水能安全表示原狀態，任務就會保留對應狀態。獨立的 DGT 任務可維持在專案之外，讓你日後再整理，不會在匯入時被迫加入多餘結構。

---

## 支援的 DGT 資料

- 任務標題
- 筆記／描述
- 優先順序
- 截止日期
- 檢查清單項目
- 資料夾、專案、情境與標籤
- 已完成任務
- 受支援的重複規則，例如簡單的每日／每週／每月／每年排程，以及部分間隔式重複

不支援的 DGT 重複模式只會匯入一次，原始重複文字會保留在描述中，供你在如水內手動調整。

---

## 匯入流程

1. 開啟**從 DGT GTD 匯入**
2. 選擇 DGT GTD JSON 或 ZIP 檔案
3. 檢查預覽摘要
4. 確認匯入

在支援的環境中，如水會在匯入前保存目前本機資料的復原快照。

匯入後：

- 視需要建立新領域與專案
- 獨立任務保持可用，供日後整理
- 若重複規則對應會損失資訊或封存檔項目遭略過，系統會顯示警告

---

## ZIP 匯出檔注意事項

如水會讀取封存檔中第一個有效的 DGT JSON 匯出檔。

如水會略過：

- 巢狀 ZIP 檔案
- 封存檔中的非 JSON 檔案
- 無法安全解析的格式錯誤 JSON 檔案

---

## 提示

- 若想先確認對應結果，可從較小的 DGT GTD 匯出檔開始
- 確認匯入結果正確前，請保留復原快照
- 重複匯入同一匯出檔可能產生重複任務

另請參閱[資料與同步](/zh-Hant/data-sync/)及[備份與還原](/zh-Hant/data-sync/backup-restore)。
