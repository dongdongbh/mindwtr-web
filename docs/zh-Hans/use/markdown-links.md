# Markdown 链接

Mindwtr 支持在笔记中使用内部 Markdown 链接交叉引用任务和项目。

## 支持的语法

请使用稳定的 Mindwtr ID，而不是纯文本标题：

```md
[[task:task-id|季度回顾]]
[[project:project-id|网站发布]]
```

- `task:` 链接通过 ID 指向任务。
- `project:` 链接通过 ID 指向项目。
- `|` 后的文字是编辑器和预览中显示的标签。

渲染时，Mindwtr 还会把这些标记规范化为普通 Markdown 链接：

```md
[季度回顾](mindwtr://task/task-id)
[网站发布](mindwtr://project/project-id)
```

## 创建链接

编辑受支持的 Markdown 字段时，输入 `[[` 并开始搜索。

- 搜索会匹配任务和项目标题。
- 桌面端在光标附近显示浮动建议弹窗；下方空间不足时会翻到插入点上方。
- 移动端在键盘上方的底部面板中显示相同建议。
- 任务编辑器会排除当前编辑的任务，避免意外链接到自身。
- 插入的链接始终使用稳定的 `[[task:...|label]]` 或 `[[project:...|label]]` 标记。
- 行内代码和围栏代码块不会被改动。

## 可用位置

- 桌面端和移动端的任务描述
- 桌面端和移动端的项目备注
- 桌面端的只读预览、展开的任务详情，以及专注/列表中的“详情开启”渲染
- 移动端任务/项目编辑器内的预览渲染

## 不会执行的操作

- Markdown 链接只是导航引用。
- 它们不会创建依赖关系图、自动完成所链接的任务，也不会跨任务绑定检查清单状态。

## 导航行为

- 有效的任务链接会打开正确的 Mindwtr 视图并高亮该任务。
- 有效的项目链接会打开“项目”视图并选中该项目。
- 外部链接仍支持 `http`、`https`、`mailto` 和 `tel`。

## 已删除项目

如果所链接的任务或项目已删除：

- Mindwtr 会为标签显示删除线样式。
- 如果已删除项目仍以删除标记形式存在于本地数据中，桌面端会显示**恢复**操作。
- 如果删除标记也已消失，链接会保留为不可交互的已删除标签。

## 示例

```md
为 [[project:project-123|网站发布]] 准备发布说明

- [ ] 起草简介文案
- [ ] 检查 [[task:task-456|首页检查清单]]
```
