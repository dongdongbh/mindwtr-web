# 用户指南：桌面端

Mindwtr 桌面应用使用 [Tauri v2](https://tauri.app/) 构建，可在 Windows、macOS 和 Linux 上提供轻量、快速的体验。

## 概览

桌面应用采用侧边栏导航，其中包含所有 GTD 视图，另有一个主要内容区域。

---

## 交互方式

- **单击**可切换任务详情的显示状态。
- **双击**任务可打开完整编辑模式。如果只想原地重命名标题，请从任务的 `⋯` 菜单中选择**重命名任务**（Enter 保存，Esc 取消）。
- 在未更改任何内容时，**单击打开的任务编辑器之外**可将其关闭；一旦编辑了某个字段，就需要使用“保存”“取消”或 Esc 才能关闭。
- **右键单击**可打开上下文菜单（项目、任务）。
- **键盘优先**：全局快捷键以及 Vim/Emacs 模式已针对桌面工作流进行优化。

如果你不断将任务的截止日期往后推迟，在第三次推迟后，其所在行会在截止日期旁边显示一个小沙漏徽章，并标明推迟次数。这个徽章是在提醒你：与其再次推迟，该任务可能需要重新澄清、委派或放弃。

大多数任务列表共用同一控制栏，用于选择任务、筛选、排序、分组、显示详情以及切换紧凑密度。使用**分组**可以按情境、领域、项目或标签等元数据整理收件箱、下一步、等待、将来/也许、参考资料、已完成及其他列表视图。

## 专注模式

使用专注模式可隐藏侧边栏，并让列表居中显示（最大宽度为 800px）。

- 使用 **Ctrl+\\**（Windows/Linux）或 **Cmd+\\**（macOS）切换
- 非常适合深度工作或回顾

### 系统托盘

Mindwtr 会在后台运行，以处理自动同步。

- **关闭窗口**会将应用最小化到系统托盘，而不是退出。
- **单击托盘图标**可显示/隐藏窗口。
- **右键单击托盘图标**可彻底退出应用。

## 视图

### 📥 收件箱

你的收集区。所有新任务都会先进入这里，等待处理。

- 使用底部的输入框**添加任务**
- 支持**快速添加语法**（请参阅[开始使用](/zh-Hans/start/getting-started)）
- 如果新任务应直接进入**今日专注**，请使用“快速添加”输入框旁边的星标。任务编辑器的标题栏中也有同样的星标。加星会被视为完成澄清：加星的收集项会创建为**下一步行动**，而不是收件箱项目；将加星任务移回收件箱时，会移除其星标。
- 使用澄清工作流**处理收件箱**
- 当你想进行有引导的收集、而不是面对空白输入框时，**思绪清扫**会提示你逐一思考常见的生活/工作领域。

### 🎯 专注

用于每日投入的统一仪表板。专注视图并不是状态为 `next` 的每项任务的完整清单；它会将列表缩小到当前可执行的工作。

- **今日专注**：用于深度工作的加星优先任务，数量不超过你配置的专注上限。
- **已逾期**：已过截止日期的项目。
- **今日到期**：今天到期的任务。
- **下一步行动**：没有截止日期、已准备就绪的任务，可按情境筛选。
- **应回顾**：已设置提醒日期的项目。

专注视图会隐藏未来才开始的任务，以及顺序项目中排在后面的任务。如果需要查看范围更广的任务清单，请使用**情境**、**项目**或**搜索**。

**下一步行动的默认顺序：**专注视图会将即将到期的行动排在最前，没有日期的行动排在其后，截止日期在很久以后的行动排在最后。在同一分组内，启用优先级时会先按优先级排序，然后依次按开始时间、创建日期、标题和 id 排序。完整逻辑请参阅[Mindwtr 中的 GTD 工作流](/zh-Hans/use/gtd-workflow#焦点如何对可执行行动排序)。

**功能：**
- **情境筛选器**：按情境（例如 @home、@work）或标签筛选下一步行动。筛选器选择器会将情境和标签分组，让很长的元数据列表仍易于浏览。
- **按情境分组**：使用列表分组控件按主要情境对下一步行动进行分组；没有情境的任务会集中在**无情境**下。
- **已保存的筛选器**：保存可复用的专注条件，例如项目、情境、标签、优先级、精力水平和预计用时。
- **禅模式**：切换后隐藏除今日专注任务之外的所有内容。
- **番茄钟（可选）**：在**设置 → GTD → 功能 → 番茄计时器**中启用，以显示专注/休息计时器面板（15/3、25/5、50/10，外加一个可选的自定义预设）。可以保持为**仅计时器**，也可以启用**将计时器关联到任务**，以显示计时器任务选择器和**将任务标记为完成**操作。

### 📁 项目

包含相关任务的多步骤成果。

- **顺序模式**：只有项目中第一个可执行的任务会出现在专注视图中；在项目视图中，可执行任务会高亮显示，排队等待的任务会淡化显示
- **并行模式**：项目中所有可执行的任务都可以出现在专注视图中
- **顺序范围**：顺序项目可以按整个项目推进，也可以逐个分区推进；分区范围会显示每个分区中第一个可执行的任务。
- **状态**：进行中、等待、将来、已归档
- **专注领域**：按更高层级的领域（例如工作、健康）对项目分组，使侧边栏井然有序。将项目拖到某个领域上即可移动；拖动期间，已折叠和空白的领域（以及**无领域**）也会显示为放置目标。
- **通过拖动移动任务**：将任务从已打开项目的列表拖到侧边栏中的另一个项目，即可将其移到那里（它会排在目标项目现有任务之后）；也可以拖到领域标题上，使其成为直属于该领域的任务。此功能适用于所有排序模式；**已移动到……**提示消息提供一键撤销，已归档项目不接受放置。
- **项目标签**：为项目添加标签并按标签筛选
- **支持性笔记**：添加规划笔记和参考材料
- **分区**：项目内部可选的任务分组（阶段、里程碑、子工作流）。分区是一个项目内的标题，不是子任务或独立项目。
- **回顾日期**：设置项目回顾的提醒日期
- **归档**：完成项目意味着将其归档——剩余任务会随之完成，而**重新激活**可撤销此操作

任务编辑器中的**项目分区**字段会将任务分配到其当前项目的某个分区。只有当任务属于一个包含分区的项目后，该字段才会有可用的值。

### 🏷️ 情境

按地点或工具情境筛选任务：

- `@home`、`@work`、`@errands`、`@agendas`
- `@computer`、`@phone`、`@anywhere`

### 🏷️ 标签

按精力水平、模式或主题筛选任务：

- 精力：`#focused`、`#lowenergy`、`#creative`
- 主题：`#health`、`#finance`

### ⏳ 等待

跟踪已委派的事项，或正在等待外部事件的任务。

### 💭 将来/也许

孵化你以后可能想要推进的想法。

### 🗓️ 日历

按时间显示有截止日期或开始时间的任务。

在宽屏布局中，日历包含一个针对所选日期的**规划下一步行动**面板。它会显示未排期的下一步行动以及已到期但未排期的任务，让你无需离开日历即可将其安排到空闲时间段。如果想为日/周网格腾出更多空间，可以折叠该面板。

### 📋 看板视图

看板式拖放面板包含以下列：

- **收件箱**：未处理的项目
- **下一步行动**：已准备好执行的项目
- **等待**：已委派的项目
- **将来/也许**：已推迟的项目
- **已完成**：已完成的任务

在同一列内拖动卡片可设置手动看板顺序。将任务移至另一状态时，会清除其在旧列中的顺序，以便在新列中排序。

使用看板搜索框，可按标题、笔记、项目、情境或标签缩小可见卡片的范围。筛选按钮会打开情境、标签、截止日期和项目的快速筛选器；**清除筛选器**会同时移除搜索文本和已启用的筛选器。

### ✅ 已完成

最近完成的任务。将其用作每日或每周回顾时的短期完成记录。

如果你完成任务的时间早于将其标记为“已完成”的时间，请单击其**完成时间**时间戳进行修正。你也可以右键单击任何已打开任务的完成按钮，将其标记为在更早的时间完成。如果完成时间对你的记录或“完成后重复”排期很重要，此功能会很有帮助。

选择**选择**以挑选任务，或选择**全选**，然后使用**移动 → 已归档**提前将所选任务归档。

### 📦 已归档

已收纳归档的完成任务。已归档任务不会出现在普通任务列表中，但你可以在这里搜索或恢复它们。单击任务的**完成时间**时间戳可进行修正。顶部的“任务 | 项目”切换可改为显示已归档项目：恢复会重新激活项目（并找回随它一起完成的任务），删除则将其移入回收站。

选择**选择**以挑选任务，或选择**全选**。你可以将所选任务移回“已完成”且不更改完成时间、恢复到收件箱，或移至回收站。Mindwtr 只会从回收站永久删除任务。

### 🗑️ 回收站

已删除的任务和项目，最新的排在最前。你可以恢复项目、永久删除项目，也可以使用**清空回收站**一次清空全部内容。选择**选择**可挑选多个任务和项目（或选择**全选**），然后一步完成恢复或永久删除。永久删除始终会要求确认。

### 📝 每周回顾

包含以下步骤的 GTD 引导式回顾向导：

1. 处理收件箱
2. 回顾日历
3. 跟进等待事项
4. 回顾项目
5. 回顾将来/也许

使用回顾中的**处理收件箱**操作清理已收集的项目，然后再继续进行日历、等待事项、项目和将来事项的回顾。

详细指导请参阅[每周回顾](/zh-Hans/use/weekly-review)。

## AI 助手（可选）

在**设置 → AI 助手**中启用：

- **澄清**：将含糊的任务转化为具体的下一步行动
- **拆分**：为大型任务生成核对清单步骤
- **回顾分析**：在回顾期间突出显示搁置已久的任务
- **Copilot**：在输入时提供情境/标签/时间建议

AI 为可选功能，只有在你请求时才会运行。

- **重置核对清单**：取消选中所有项目，以便重复使用（采购杂货）

## 任务编辑器（查看 + 编辑）

- 单击任务可打开包含所有详情的**只读视图**。
- 按**编辑**（或 `e`）切换到编辑模式。
- 编辑器最初仅显示必要字段。使用**更多选项**显示高级字段。
- **项目分区**是针对分区项目内任务的组织字段。收件箱任务、未分配任务或没有分区的项目应将其留空。
- 状态、优先级和精力水平使用胶囊控件而不是下拉菜单，以便更快地通过键盘和指针进行选择。
- 打开日期字段的日历，可在月历旁选择快捷日期：**今天**、**明天**、**+2 天**、**+3 天**、**下周**、**下个月**和**无日期**。
- 在**设置 → GTD → 任务编辑器布局**中，选择桌面编辑器是以内联侧边速览还是居中弹出窗口的形式打开。

## 日历集成

Mindwtr 可以在日历视图中叠加显示外部日历。在 macOS 上，它可以通过 EventKit 读取 Apple 日历；在所有桌面平台上，它都可以读取直接的 ICS 订阅 URL。
需要 ICS 订阅时，请在**设置 → 日历**中添加 **ICS URL**。
外部事件为只读，但你可以根据日历事件在 Mindwtr 中创建一项独立任务；Mindwtr 会复制事件的标题、日期/时间、地点、描述，以及可用时的日历名称。
在日历中使用**规划下一步行动**，可将所选日期变成规划界面：查看未排期的工作，选择任务，并将其安排到空闲时间段，同时保留截止日期作为最后期限。
在 macOS 上，**将任务推送到日历**还可以把 Mindwtr 中已排期或到期的任务写入所选的可写 Apple 日历目标。设置详情请参阅[日历集成](/zh-Hans/use/calendar-integration)。

### 🔍 全局搜索

使用运算符进行强大搜索，即时找到任何内容。

**打开：**按 `/` 或 `Ctrl/Cmd + K`，或单击搜索图标。

搜索文本可留空。打开**筛选器**，可按状态、范围、领域、截止日期、地点、情境或标签列出匹配条目；如需查看已完成条目，请选择**已完成**和**已归档**（或**包含已完成和已归档的任务**）。

#### 搜索运算符

| 运算符      | 示例               | 说明                 |
| ----------- | ------------------ | -------------------- |
| `status:`   | `status:next`      | 按任务状态筛选       |
| `-status:`  | `-status:done`     | 排除某个状态         |
| `context:`  | `context:@home`    | 按情境筛选           |
| `tag:`      | `tag:#focused`     | 按标签筛选           |
| `assigned:` | `assigned:Tom`     | 按受委派人筛选       |
| `project:`  | `project:HomeReno` | 按项目名称或 ID 筛选 |
| `location:` | `location:office`  | 按任务地点筛选       |
| `where:`    | `where:office`     | 任务地点的别名       |
| `id:`       | `id:abc123`        | 查找确切的任务 ID    |
| `-id:`      | `-id:abc123`       | 排除确切的任务 ID    |
| `due:`      | `due:today`        | 在某日期到期的任务   |
| `due:<=`    | `due:<=7d`         | 7 天内到期的任务     |
| `start:`    | `start:>=tomorrow` | 从某日期开始的任务   |
| `created:`  | `created:>=30d`    | 最近 30 天创建的任务 |
| `OR`        | `@home OR @work`   | 匹配任一条件         |

**日期格式：**`today`、`tomorrow`、`7d`（7 天）、`2w`（2 周）、`1m`（1 个月）、`2025-01-15`

#### 已保存的搜索

已保存的搜索是可复用搜索查询的快捷方式：

1. 输入包含运算符的搜索查询
2. 单击**“保存搜索”**按钮
3. 为搜索命名（例如“即将到期的工作任务”）
4. 从侧边栏的**已保存的搜索**部分访问

**删除已保存的搜索：**打开已保存的搜索，然后单击回收站图标。

已保存的搜索不同于专注视图中的**已保存的筛选器**。专注视图的已保存筛选器会作为筛选条件同步，并通过专注视图的筛选控件进行编辑。

## 快速收集

### 全局热键

无需切换窗口，即可从桌面的任何位置收集任务：

1. 按全局热键（`Ctrl+Alt+M`）
2. 使用快速添加语法输入任务
3. 按 Enter 添加到收件箱
4. 继续你刚才在做的事情

你可以在**设置 → 通用 → 输入 → 全局快速添加快捷键**中更改或禁用此快捷键。
默认值遵循平台惯例：
- macOS：`Ctrl+Option+M`（默认）、`Ctrl+Option+N`、`Ctrl+Option+Q`，或旧版 `Cmd+Shift+A`
- Linux：`Ctrl+Alt+M`（默认）、`Ctrl+Alt+N`、`Ctrl+Alt+Q`，或旧版 `Ctrl+Shift+A`
- Windows：默认禁用，因此 Mindwtr 绝不会在未告知的情况下占用其他应用的组合键；启用快捷键时，推荐选择 `Ctrl+Alt+M`
- 避免使用旧版 `Ctrl+Shift+A` 选项：Chrome、Word 和 Excel 都在使用它，而全局热键会在整个系统中覆盖这些应用的快捷键

**Flatpak/Wayland 注意事项：**某些合成器不允许应用注册全局快捷键。在这种情况下，请将桌面快捷键绑定到 `flatpak run tech.dongdongbh.mindwtr --quick-add`。

### 托盘图标

单击系统托盘图标即可立即收集：

- 出现快速添加输入框
- 使用自然语言语法
- 任务直接进入收件箱

### 添加任务对话框快捷键

“添加任务”对话框完全可以通过键盘操作：

- `Enter`：保存并关闭
- `Shift+Enter`：保存并让对话框保持打开，以便添加下一个任务（批量收集）
- `Ctrl+Enter`（macOS 上为 `⌘Enter`）：保存并在完整编辑器中打开任务
- `Esc`：关闭且不保存
- 打开建议弹窗（`@`、`#`、`+`、`%`、`!`、`/`）时，`↑`/`↓` 可在建议间移动，`Enter` 或 `Tab` 会插入突出显示的建议

### 快速添加语法

Mindwtr 会在添加任务时解析自然语言：

| 语法             | 示例                         | 结果                                                         |
| ---------------- | ---------------------------- | ------------------------------------------------------------ |
| `@context`       | `@home`                      | 添加情境                                                     |
| `#tag`           | `#focused`                   | 添加标签                                                     |
| `+Project`       | `+HomeReno`                  | 分配到项目                                                   |
| `+Multi Word`    | `+New Project`               | 分配到“New Project”                                          |
| `+"Quoted Name"` | `+"New Project" call Bob`    | 引号用于界定句子中间的多词名称（也适用于 `!"Area Name"`）    |
| `!Area`          | `Plan roadmap !Work`         | 分配到领域                                                   |
| `%Person`        | `Ask %Jim for budget`        | 设置“分配给”（委派/等待事项的负责人）；新建多词名称时使用 `%"Full Name"` |
| `/area:<name>`   | `/area:Personal`             | 分配到领域（不能有空格）                                     |
| `/due:date`      | `/due:friday`                | 设置截止日期                                                 |
| `/energy:<level>` | `/energy:medium`            | 设置精力水平（`low`、`medium`、`high`）                       |
| `/note:text`     | `/note:call back`            | 添加描述                                                     |
| `/status`        | `/next`、`/waiting`、`/someday`、`/done`、`/archived`、`/inbox` | 设置状态 |

**日期格式：**today、tomorrow、friday、next week、in 3 days、2025-01-15

## 音频收集与转录

使用由 AI 驱动的转录，通过语音收集任务。

### 设置

1. 前往**设置 → AI 助手**。
2. 启用**语音转文字**。
3. 选择**提供商**：
   - **OpenAI / Gemini**：需要 API 密钥（基于云端）。
   - **离线（Whisper）**：在你的设备上本地运行。单击**下载**以获取一次模型。
4. 配置**处理模式**：
   - **智能解析**：从你的语音中提取日期（`tomorrow`）、优先级和项目。
   - **仅转录**：将文本逐字转录到任务中。

### 使用音频收集

- **快速添加**：在快速添加栏中将收集模式切换为**音频**（麦克风图标）。
- **录音**：单击麦克风开始录音。自然地说出你的任务。
- **完成**：单击停止进行转录。文本会填入输入框。
- **附件**：在**设置 → 通用**中启用“保存音频附件”，以保留原始语音笔记。

## 任务管理

### 创建任务

1. 使用任意列表视图底部的输入框
2. 在任何位置使用全局热键
3. 单击托盘图标进行快速收集
4. 在 Flatpak/Wayland 上，如果内置热键不可用，请将 `flatpak run tech.dongdongbh.mindwtr --quick-add` 绑定为自定义快捷键
5. 使用快速添加语法输入任务标题
6. 按 Enter 添加

### 编辑任务

- 单击任务可打开编辑面板
- 编辑：标题、状态、情境、标签、描述、地点
- 设置：截止日期、开始日期、回顾日期、预计用时、重复规则
- 管理核对清单项目
- 描述的 Markdown 支持无序列表和任务复选框（`- item`、`[ ] item`、`[x] item`）
- 开启列表的**详情**切换项后，任务行会以 Markdown 格式显示描述的第一行；展开一行可阅读完整描述
- 在任务描述或项目笔记中输入 `[[`，可从浮动选择器中链接另一个任务或项目
- 这些链接仅用于导航；它们不会在任务之间同步完成状态
- 描述中的 Markdown 复选框仅用作笔记；它们不会影响任务核对清单
- 将多行文本粘贴到核对清单项目中，可按每行创建一个项目（可识别项目符号、编号和 `[x]` 标记）
- 分配到项目

### 任务属性

| 属性                | 说明                                                  |
| ------------------- | ----------------------------------------------------- |
| **状态**            | inbox、next、waiting、someday、done、archived          |
| **优先级**          | low、medium、high、urgent                              |
| **情境**            | 地点/工具标签（例如 @home、@work）                     |
| **标签**            | 精力/模式标签（例如 #focused、#lowenergy）             |
| **截止日期**        | 任务应何时完成                                        |
| **开始日期**        | 何时开始处理任务                                      |
| **回顾日期**        | 用于回顾的提醒日期                                    |
| **预计用时**        | 5min、10min、15min、30min、1hr、2hr、3hr、4hr、4hr+   |
| **重复规则**        | daily、weekly、monthly、yearly + 策略                  |
| **核对清单**        | 多步骤任务的子项目                                    |
| **描述**            | 带预览的 Markdown 格式笔记                            |
| **附件**            | 附加到任务的文件和链接                                |
| **地点**            | 实际地点                                              |
| **受委派人 / 人员** | 等待事项的受委派人以及 `assigned:` 搜索中的人员       |
| **项目**            | 父项目分配                                            |
| **分区**            | 项目内的可选分组                                      |

**附件：** **添加链接**字段接受 URL 和本地文件路径（例如 `/home/user/doc.pdf`、`C:\Users\you\file.txt` 或 `file://...`）。
有关同步、清理和音频笔记的信息，请参阅[附件](/zh-Hans/use/attachments)。

### 重复任务

完成重复任务后，Mindwtr 会自动创建日期已更新的下一个实例。

- **严格**（默认）：保持固定的排期节奏。
  示例：每 5 天重复一次会始终锚定在规划的周期上。
- **完成后重复**：从完成时间起推算下一个截止日期。
  示例：现在完成，则下次在 5 天后到期。
- **结束：永不 / 指定日期 / N 次后**：选择该系列是永远持续、在特定日期停止，还是在固定的发生次数后停止。
  `After N occurrences` 会将当前任务计入该系列的总次数。

Mindwtr 会为重复任务保留一个活动实例。当该实例具有截止日期或开始时间时，日历会显示它；除非启用了**在日历中显示下一次发生**以提供仅供规划的预览，否则在当前实例完成前，不会预先填充未来的发生项。

你可以在任务编辑器的重复规则字段中使用**完成后重复**来切换此设置。
当你需要系列自动停止时，请使用同一个重复规则表单来设置结束日期或发生次数。

## 批量操作

选择多个任务以执行批量操作：

1. 单击列表标题栏中的**“选择”**按钮
2. 单击任务以选择/取消选择
3. 使用操作栏：
   - **移动**：更改所有所选任务的状态
   - **分配领域**：将所有所选任务移到某个特定领域下（或清除领域）
   - **添加标签**：为所有所选任务添加标签
   - **移除标签**：从所有所选任务中移除标签
   - **添加情境**：为所有所选任务添加情境
   - **移除情境**：从所有所选任务中移除情境
   - **删除**：删除所有所选任务
4. 单击**“完成”**退出选择模式

### 排序

使用排序下拉菜单按以下方式排列任务：
- 默认（基于状态）
- 截止日期
- 开始日期
- 回顾日期
- 标题（按字母顺序）
- 创建时间（最早/最新）

在项目内选择的排序方式会按项目记住，并同步到你的其他设备。未选择排序方式的项目使用手动顺序；项目处于默认排序时仍可拖动排序。

## 键盘快捷键

Mindwtr 支持**标准**（Gmail/Todoist 风格）、**Vim** 和 **Emacs** 按键绑定预设。可在“设置”中更改。

完整列表请参阅[桌面端键盘快捷键](/zh-Hans/use/keyboard-shortcuts)。

**快速参考（标准）：**
- `/`：打开搜索
- `?`：显示快捷键帮助
- `gi`：前往收件箱
- `gn`：前往下一步
- `gf`：前往专注视图
- `j/k`：向下/向上移动选择
- `Enter`：打开所选任务
- `Shift+Enter`：编辑所选任务
- `e`：标记为完成/未完成
- `x`：选择/取消选择任务以进行批量操作
- `#`：删除任务
- `z`：撤销上一次完成/删除操作

**快速参考（Vim）：**
- `j/k`：向下/向上移动选择
- `Enter`：打开所选任务
- `e`：编辑所选任务
- `x`：切换完成状态
- `dd`：删除任务

在每种预设中，`Ctrl+Z` / `Cmd+Z` 都可撤销上一次任务完成或删除操作。

在每种预设中，先按 `s` 再按一个字母，可直接设置所选任务的状态（`si` 收件箱、`sn` 下一步、`sw` 等待、`ss` 将来、`sd` 已完成、`sa` 已归档），并显示撤销提示消息。`Insert` 会跳转到添加任务输入框（如果列表没有输入框，则打开快速添加）。

## 设置

从侧边栏进入“设置”。

### 通用
- **外观**：浅色、深色或跟随系统
- **语言**：英语、简体中文、繁体中文、印地语、西班牙语、阿拉伯语、法语、葡萄牙语、俄语、德语、日语、越南语、土耳其语、韩语、意大利语、波兰语、荷兰语、捷克语
- **键盘快捷键**：标准、Vim 或 Emacs 预设
- **开机启动**：登录时自动启动 Mindwtr

### 通知

**任务提醒：**
- 启用/禁用截止日期和开始时间的任务通知

**每日摘要：**
- **晨间简报**：汇总今日到期、已逾期和专注任务
- **晚间回顾**：提示你回顾并为一天收尾
- 配置时间（例如上午 9:00、晚上 8:00）

**每周回顾：**
- **提醒**：接收开始每周回顾的通知
- **回顾日期/时间**：自定义你希望收到提醒的时间

### GTD
- **自动归档**：在设定的天数后自动将“已完成”任务移至“已归档”（默认：7 天），或选择**永不**，将已完成任务保留在“已完成”中。缩短延迟时间会立即归档现有“已完成”任务中已超过新期限的任务。
- **专注任务上限**：选择可以标记为“今日专注”的任务数量
- **功能**：可在需要时启用的可选提示：
  - **优先级**：在任务上显示优先级旗标
  - **预计用时**：添加用于时间分块的时长字段
- **收件箱处理**：保留用于教授 GTD 的引导式收件箱处理，或将默认模式切换为**快速**，使用紧凑的单屏流程
  - 选择是否显示 2 分钟捷径、提前询问项目、包含情境/标签、允许排期，以及在处理过程中提供参考资料选项
- **任务编辑器布局**：选择默认显示哪些字段、在各部分间移动字段，并重新排列字段
  - 选择编辑器的打开方式：使用**侧边速览**进行内联编辑，或使用**弹出窗口**进行专注编辑
  - 你可以将**截止日期**等字段移到**排期**中，并选择默认展开哪些部分
- **管理**：在一处维护可复用的元数据
  - 无需逐个查找任务，即可编辑**领域**、**人员**、**情境**和**标签**
  - 人员会保留可复用的受委派人姓名、笔记和参考链接，用于委派工作和建议
  - 有关在分配时从何处创建领域和人员，请参阅[领域与人员](/zh-Hans/use/areas-people)

### 数据与同步

**同步后端：**
- **云同步**：受支持的版本中提供 Dropbox，在可用的 macOS 设备上还提供 iCloud
- **文件夹 / 文件同步**：通过共享的 JSON 文件/文件夹进行文件同步（Google Drive、Syncthing、OneDrive 等）
- **高级 / 自定义服务器**：WebDAV 或自行托管的 Mindwtr Cloud

**设置同步选项：**
- 选择要在设备间同步的偏好设置（主题、语言/日期格式、GTD 默认值、外部日历 URL、AI 设置、已保存的筛选器）
- API 密钥和本地模型路径绝不会同步

**同步状态：**
- 侧边栏页脚会显示上次同步时间和在线/离线状态

对于 WebDAV，请配置：
- 服务器 URL（文件夹 URL；Mindwtr 会在其中存储 `data.json`）
- 用户名和密码

详细设置请参阅[数据与同步](/zh-Hans/data-sync/)。
