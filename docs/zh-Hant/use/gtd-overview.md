# GTD 概覽

Getting Things Done（GTD）是 David Allen 創立的個人生產力方法，記載於他的著作 *Getting Things Done: The Art of Stress-Free Productivity*。

---

## 心如止水

GTD 的核心理念，是達到**思緒清明**的狀態，讓大腦不必承受記住所有待辦事項的負擔。

> *「大腦是用來產生想法，而不是保存想法。」* — David Allen

當大腦相信所有事情都已收進可靠的系統，就能專注以創意而有效率的方式真正完成工作。

---

## 五個步驟

GTD 由五個彼此連結的步驟組成：

### 1. 收集

將所有事情從腦中移至可信賴的系統。

- 寫下每項任務、想法、承諾或提醒
- 先不要篩選或整理，只管收集
- 以**收集箱**作為所有項目的入口

### 2. 釐清

逐一處理收集到的項目，判斷它的意義。

對每個項目詢問：
- **這是什麼？**
- **可以採取行動嗎？**
  - 不行 → 垃圾、將來/也許或參考資料
  - 可以 → 想要的成果是什麼？下一步行動是什麼？

### 3. 整理

將事情放到適當的位置：

| 目的地          | 適用情況                         |
| --------------- | -------------------------------- |
| **下一步行動**  | 由自己執行的可行動任務           |
| **專案**        | 需要多個行動才能達成的成果       |
| **等待中**      | 已委派給他人的事項               |
| **行事曆**      | 必須在特定時間發生的事項         |
| **將來/也許**   | 留待未來的想法                   |
| **參考資料**    | 需要保留的資訊（筆記）           |
| **垃圾**        | 不再需要的內容                   |

### 4. 回顧

定期檢視你的系統。

- **每日回顧**：快速查看清單、行事曆與收集箱
- **每週回顧**：全面檢視所有清單、清空收集箱並更新專案

### 5. 執行

有信心地採取行動。

依照以下條件選擇要處理的工作：
- **情境**：你在哪裡？手邊有哪些工具？
- **時間**：有多少時間？
- **精力**：目前的身心狀態如何？
- **優先順序**：什麼最重要？

## 核心概念

### 下一步行動

下一步行動是推動事情前進的**下一個可見且具體的動作**。

❌ 不佳：「處理專案」
✓ 良好：「開啟專案資料夾並檢查狀態筆記」

### 專案

專案是任何需要多個行動才能達成的成果。它是對結果的承諾，而不是一份待辦清單。

### 專案分區

專案分區是同一個專案內有標題的群組。它能讓較長的專案任務清單保持易讀，而不必轉成巢狀專案或子任務。

例如名為**推出網站**的專案，可以有**設計**、**開發**與**內容**等分區。專案仍然只有一個成果；分區只是專案內任務的標題。

任務的**專案分區**欄位會將任務指派至目前專案的其中一個分區。只有任務已屬於某個設有分區的專案時，此欄位才有意義。若任務沒有專案，或專案沒有分區，請將「專案分區」留空。

### 領域、專案與任務

Mindwtr 採用靈活的「容器」模型，讓你能順暢整理：

- **領域**是持續承擔的責任（例如工作、家庭、健康）。
- **專案**是有終點的成果。它可以位於某個領域內，也可以獨立存在。
- **任務**可以位於專案內、直接位於領域內，也可以完全不指派（收集箱／無領域）。

如此便能將長期責任保留在領域中，同時迅速收集任務，不必強迫自己先選擇資料夾。

實用原則：

- **領域**回答：「這項責任與我的生活或工作中哪個部分有關？」
- **情境**回答：「考量目前所在位置與手邊資源，我現在能做什麼？」

例如，如果你是為多位客戶工作的承包商，可以為每位客戶各設一個**領域**。同一項任務可以同時屬於**領域：客戶 A**、**專案：網站改版**與**情境：@computer**。

### 2 分鐘法則

若一項行動不到 2 分鐘就能完成，請在處理時**立刻去做**。

### 情境

指出可在何處或使用什麼完成工作的標籤：
- `@home`、`@work`、`@errands`
- `@computer`、`@phone`
- `#focused`、`#lowenergy`

## GTD 為何有效

1. **外部可信賴系統**：大腦不再試圖記住所有事情
2. **明確的下一步行動**：不再對該做什麼感到含糊
3. **定期回顧**：系統保持最新且值得信任
4. **依情境整理的清單**：只看見當下相關的事情

---

## 學習 GTD

- **6 個月**：不再覺得使用系統笨拙
- **1 至 2 年**：真正內化原則
- **持續進行**：重讀時仍會發現新的體會
