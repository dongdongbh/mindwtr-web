# 如水是什么

如水管理你个人生活中的承诺与行动，并采用 David Allen 的 Getting Things Done 方法。本页介绍产品最核心的定位，帮助你判断它是否适合自己的工作方式。

## 原生支持 GTD

收集箱、项目、情境、等待中、将来/也许和回顾都是如水的一等功能。它们是应用的构建基础，而不是贴在通用任务清单上的标签。当你记录一件事、明确其含义并将其归类时，就是在直接运行 GTD 循环。[GTD 概览](/zh-Hans/use/gtd-overview)介绍这一方法，[GTD 工作流程](/zh-Hans/use/gtd-workflow)则说明每一步对应哪个界面。

## 笔记服务于行动

你可以为任务和项目附加笔记与文件，让所需背景资料紧邻相关行动。如水的定位到此为止：它不是知识库，也不试图成为知识库。当参考资料需要一个真正的归宿时，请把它们放在专门的工具里。[Obsidian 集成](/zh-Hans/power-users/obsidian)正是为此而设：笔记留在你的库中，如水从中读取任务。

## 面向个人，而非团队

如水承载一个人的承诺，也就是你的承诺。这里没有共享工作区，不能把任务分配给队友，也没有评论串和动态信息流。“等待中”用于记录别人尚未交付给你的事项，是个人记录，而非协作渠道。

## 默认简单，需要时强大

日常循环就是记录、明确和回顾，完成这些事情所需的一切无需配置即可看到。AI、本地 API、MCP 服务器、看板和番茄专注等较重的功能，在你主动开启前都不会打扰你；开启后也不会让基本流程变复杂。项目 README 说得很直白：

> 我只想骑自行车时，别给我一座驾驶舱。

## 自动优于手动

只要存在合理的默认选择，如水就会直接采用，而不是再问你一次。它会读取平台惯例和安装渠道，而不是为每个选择增加一项设置。开关越少，需要维护和可能出错的地方也越少。
