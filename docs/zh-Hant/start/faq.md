# 常見問題

Mindwtr 的常見問題。

---

## 一般

### Mindwtr 是什麼？

Mindwtr 是一款跨平台 Getting Things Done（GTD）生產力應用程式，協助你收集、釐清、整理並執行任務。提供桌面版（Windows、macOS、Linux）與行動版（Android、iOS）。

### Mindwtr 免費嗎？會一直免費嗎？

是。Mindwtr 是開放原始碼（AGPL-3.0）且可免費使用。同步或其他功能都沒有付費牆、訂閱或進階付費層級，計畫也會維持如此：好軟體應讓所有人都能使用，不受經濟狀況限制；支持應來自自願付出的人，而不是付費牆。

專案確實有持續支出：每年的 Apple 開發者費用、託管、開發工具，以及投入功能、修正與社群支援的時間。[捐款](https://mindwtr.app/donate)是維持專案永續的方式。

有一項必須坦白的但書：目前捐款無法支付所有成本，差額由維護者負擔。如果缺口日後大到難以維持，可能會增加選用的付費服務（例如供不想自行託管者使用的託管同步伺服器）以支應成本。無論如何，應用程式本身仍會保持免費與開放原始碼。今天能使用的任何功能都不會移至付費牆後。

### Mindwtr 會維護多久？

Mindwtr 是一個熱情專案，也是維護者管理自己生活的工具，因此他會維護它，因為自己也需要它持續受到維護。Commit 歷史最誠實地呈現實際維護狀況。

個人專案無法承諾「永遠」，所以 Mindwtr 的設計改為確保最糟情況仍然安全：它採用在地優先，資料以開放格式存放在你掌控的檔案中，程式碼使用 AGPL，並可在獨立管道上重現建置。即使開發停止，應用程式仍能離線運作，資料依然可讀，任何人也能分支程式碼。你可以放心將工作流程交給它，也能隨時帶著完整資料離開。

### Mindwtr 是由 AI 產生的嗎？專案如何看待 AI？

Mindwtr 是由 AI 輔助，而非 AI 產生。這項區別很重要。

Mindwtr 背後的架構、產品方向、功能設計與 GTD 理念，都由維護者設計及負責。AI 工具協助執行：更快草擬實作、減少樣板程式碼、改善文字，並加速錯誤調查。每項發布的變更仍會經過規格界定、審查及測試，結果也由維護者負責。

AI 輔助開發已是現今正常的軟體工程實務，包括大型科技公司與開放原始碼專案。AI 是一項工具，如同 IDE、自動完成、文件與 Stack Overflow。使用 AI 不會取代開發者的工作，只是將更多工作轉向方向、判斷、審查、整合與驗證。

Mindwtr 由一位具 10 年以上軟體工程經驗的開發者維護。正是這些經驗讓 AI 成為助力而非危險：知道該建置什麼、不該建置什麼、何時輸出有誤，以及如何保持系統一致。AI 協助一個範圍如此廣的專案由一人持續維護，但不會決定設計，也不對結果負責。

Issue 與 discussion 回覆由維護者撰寫。AI 可能用來潤飾英文，但沒有 agent 自動分類或回答 issue。

貢獻者請參閱 [CONTRIBUTING.md 中的 LLM 輔助程式設計章節](https://github.com/dongdongbh/Mindwtr/blob/main/docs/CONTRIBUTING.md#llm-assisted-coding-vibe-coding)。

### 為什麼有這麼多 commit 與 issue 回報？

Mindwtr 是跨平台的終端使用者應用程式，不是命令介面狹窄的小型 CLI 或函式庫。它包含桌面與行動用戶端、在地優先持久化、同步後端、匯入、通知、快速收集、小工具、翻譯、安裝管道及 GTD 工作流程決策。這類產品自然會有許多小型後續 commit，因為介面打磨、平台差異、封裝修正與邊界情況回報都是日常維護的一部分。

公開 issue 的涵蓋範圍也刻意保持廣泛。GitHub issue 包含功能要求、使用體驗改善、特定平台邊界情況、發行封裝回報、文件缺漏與已確認錯誤。許多回報不會阻擋主要的收集／整理／回顧工作流程，但仍公開追蹤，讓使用者看見已知事項與變更內容。迅速進行後續修正是維護模式的一部分，不表示問題受到忽略或隱瞞。

### 應該在哪裡回報錯誤或要求功能？

最佳位置是 **GitHub issue**。日誌、裝置詳細資料與後續回覆能集中在一個可追蹤、可搜尋的位置，不會遺失：
https://github.com/dongdongbh/Mindwtr/issues

其他聯絡方式：

- **應用程式內意見回饋**：組建提供時，可在**設定 → 關於**使用**傳送意見回饋**。不必有 GitHub 帳號，即可傳送錯誤回報、功能要求或其他備註，回覆電子郵件欄位為選用。回報錯誤時，**附上近期診斷**會加入一份有界快照，包含近期應用程式內檢視與動作，以及任何可用的近期診斷日誌；即使詳細日誌關閉也能使用，加入的快照不會儲存在裝置上。詳情請參閱[隱私權政策](https://mindwtr.app/privacy)。
- **電子郵件**：不適合公開 issue 的私人或帳號問題，請寄至 [support@mindwtr.app](mailto:support@mindwtr.app)。

回報錯誤時，請附上**平台**與**應用程式版本**（設定 → 關於），方便提供協助。

如果 Mindwtr 對你有幫助，[支持頁面](https://mindwtr.app/support)列出各種支持方式。

### 資料儲存在哪裡？

所有資料都儲存在你的裝置本機：
- **桌面資料（Linux）：**`~/.local/share/mindwtr/data.json`
- **桌面設定（Linux）：**`~/.config/mindwtr/config.toml`
- **Windows/macOS：**`%APPDATA%/mindwtr/` 或 `~/Library/Application Support/mindwtr/`
- **行動版：**應用程式內部儲存空間

詳情請參閱[資料與同步](/zh-Hant/data-sync/)。

### 有雲端同步嗎？

Mindwtr 支援檔案同步、WebDAV、自行託管 Cloud Sync，以及 Dropbox 同步（支援的組建）。請參閱[資料與同步](/zh-Hant/data-sync/)。

### 哪種同步方式最快：本機資料夾還是 WebDAV？

在硬體與網路距離相近的情況下，**本機資料夾同步通常比 WebDAV 快**。資料夾同步使用一般檔案系統呼叫讀寫快照；WebDAV 則為每項要求增加 HTTP 往返、驗證與伺服器處理。大型附件的差異最明顯，因為 WebDAV 會逐一上傳及下載。不過瓶頸通常是資料夾背後的儲存供應商（例如在背景同步的 Syncthing 或雲端硬碟用戶端），而不是 Mindwtr 本身。請選擇符合設定需求的後端；實際同步內容請參閱[資料生命週期](/zh-Hant/data-sync/data-lifecycle)。

### Mindwtr 直接支援 iCloud 同步嗎？

是，在支援的 Apple 組建上：

- **iPhone / iPad：**在**設定 → 同步**中提供原生 **iCloud** 後端
- **macOS 桌面版：**在**設定 → 同步**中提供原生 **iCloud** 後端
- **Android / Windows / Linux：**沒有原生 iCloud 後端

請參閱 [iCloud 同步](/zh-Hant/data-sync/icloud)與[資料與同步](/zh-Hant/data-sync/)。

### 如何與 OneDrive 同步（尤其是 Android）？

Mindwtr 已可透過**檔案同步**搭配 OneDrive：

- **Windows/macOS：**將 Mindwtr 的 `data.json` 放入 OneDrive 資料夾，OneDrive 會自動處理同步。
- **Android：**官方 OneDrive 應用程式**不提供**真正的雙向資料夾同步。
  請使用 **OneSync (Autosync for OneDrive)** 或 **FolderSync** 等「橋接」輔助應用程式，持續同步本機資料夾。
  再將 Mindwtr 指向**設定 → 同步**中的該本機資料夾（Mindwtr 會使用其中的 `data.json`）。

這與 Obsidian 等在地優先應用程式採用的方式相同。

### 為什麼沒有「使用 OneDrive / Google Drive 登入」按鈕？

Mindwtr 採用在地優先與離線優先。

Mindwtr 現在已在支援的組建中提供原生 **Dropbox OAuth** 同步，但**不提供**原生 OneDrive/Google Drive OAuth 整合。
擴充至每個供應商會增加大量維護與法規遵循負擔。

OneDrive/Google Drive 使用檔案同步，既能讓資料保持在你的掌控中，也避免龐大的維護／安全負擔。

### Mindwtr 能與電子郵件（Gmail/Outlook）整合，或接收轉寄郵件嗎？

不能直接整合。建置完整電子郵件用戶端需要：

- Gmail/Outlook OAuth 存取權（目前需要昂貴的安全稽核）
- 可靠的 MIME/HTML 解析與附件處理
- 持續因應各供應商維護

**目前替代方式：**
- **桌面版：**在任務中貼上 `message://` 或郵件連結；支援的用戶端也可將郵件拖入任務筆記。
- **行動版：**使用分享選單，將選取的電子郵件內容傳送至 Mindwtr。

Mindwtr**不提供** `add@mindwtr.com` 收件匣，因為這需要中央伺服器接收及儲存你的電子郵件。

## 功能

### GTD 是什麼？

Getting Things Done（GTD）是 David Allen 創立的生產力方法，由五個步驟組成：收集、釐清、整理、回顧與執行。請參閱 [GTD 概覽](/zh-Hant/use/gtd-overview)。

### GTD 視野如何對應 Mindwtr？

Mindwtr 原生呈現較低層級的視野：

- **視野 0（行動）：**下一步行動與任務清單。
- **視野 1（專案）：**明確的專案實體。
- **視野 2（領域）：**領域將相關專案分組。

視野 3 至 5（目標、願景、使命）目前沒有專用實體。大多數使用者會用以下方式追蹤：

- **參考資料**清單（或「目標」領域）
- 專案筆記，以及指向這些參考項目的連結
- 每週回顧檢查清單

如果你依賴明確的目標／願景物件，請提出 issue，說明所需工作流程與回顧節奏。

### 專案分區有什麼用途？

**專案分區**是單一專案內有標題的群組。可用分區讓較長的專案保持易讀，例如**推出網站**專案中的**設計**、**開發**與**內容**。

分區不是子任務，也不是獨立專案，只是同一項專案成果內任務的標題。

任務的**專案分區**欄位會將任務指派至目前專案的其中一個分區。只有任務已屬於設有分區的專案時才會產生作用。若任務沒有專案，或專案沒有分區，請留空。

### 為什麼一個專案可以有多項「下一步」任務？

在 Mindwtr 中，**下一步**是任務狀態，表示任務已釐清且可採取行動；不一定等同於專案「唯一的下一步行動」。

專案類型會控制呈現內容：

- **循序：**一次只呈現一項可用的「下一步」任務。其他「下一步」任務仍留在專案中等待輪到。
- **平行：**多項彼此獨立的「下一步」任務可以同時呈現，因為執行順序不限。

循序專案中較後面的步驟不是**參考資料**。參考資料用於不可執行的支援內容、筆記與文件。

### 如何將專案標示為已完成？

**將它歸檔**——歸檔就是在 Mindwtr 中完成專案的方式。開啟專案並使用**歸檔**按鈕（桌面版位於專案標題列，行動版位於專案詳細資料的**動作**下）。任何未完成任務都會一併完成；若太早歸檔，**重新啟用**可全部還原。完成專案最後一項行動時，「下一步行動是什麼？」提示也會直接提供**完成專案**，通常不必另外尋找按鈕。即使所有任務都已完成，專案仍會保持使用中，直到你自行關閉成果；這也會成為每週回顧中的實用提示。

### 可以將任務轉為專案嗎？

可以。最簡單的位置是在處理收集箱時：引導流程詢問項目是否需要多個步驟時，回答「是」，該收集項目就會成為專案，供你命名並定義第一項下一步行動。之後加入的其他行動會在已連結專案的狀態下回到收集箱，讓每一項各自經過釐清流程。請參閱 [GTD 工作流程](/zh-Hant/use/gtd-workflow)。

### 為什麼任務沒有顯示在「專注」？

「專注」刻意只呈現當下可執行的事情，因此任務可能因以下原因隱藏：

- 有**未來開始日期**，會在日期到來時重新出現。
- 是**循序專案中較後面的任務**，只會呈現第一項可用任務。
- 使用中的**情境篩選條件**將它排除。
- 狀態不可執行（收集箱、將來、等待中）。

請使用**專案**、**情境**或**搜尋**檢查完整任務清單。請參閱[「專注」如何排序可用行動](/zh-Hant/use/gtd-workflow#「專注」如何排序可用行動)。

### Mindwtr 支援開始日期或提醒嗎？

兩者都支援：

- **開始日期**會延後任務：未來開始日期會將任務從「專注」及「下一步行動」隱藏，並在日期到來時以原有狀態重新出現。「專注」與「下一步行動」清單會將延後任務隱藏至開始日；想提前查看，請使用**專案**或**搜尋**。
- **回顧日期**是提醒：日期到來時，Mindwtr 會顯示任務供你重新考慮。在你決定前，不會改變任何內容。
- **開始提前時間**會將開始時間綁定至截止日期（例如截止前兩天顯示）。

請參閱[日期與狀態的差異](/zh-Hant/use/gtd-workflow#日期與狀態的差異)。

### 如何啟用優先順序或預估時間？

Mindwtr 採用漸進式揭露，因此選用欄位預設隱藏。

在以下位置控制任務編輯欄位：

**設定 → GTD → 任務編輯器版面**

在該處開啟**優先順序**與**預估時間**（也可視需要重新排序欄位）。隱藏欄位仍會顯示在**更多**下方；任務已有該欄內容時也會顯示。

### 「已完成」與「已歸檔」任務有何差異？

**已完成**用於最近完成的任務。已完成任務會保留完成日期、繼續顯示於「已完成」檢視，並在每日或每週回顧中查看完成內容時發揮作用。

**已歸檔**用於想收存的完成任務。已歸檔任務不會顯示在一般任務清單，但仍可在「已歸檔」檢視中搜尋、還原或永久刪除。歸檔不會刪除任務。

實際使用時，將「已完成」視為短期完成記錄，「已歸檔」則是長期歷史。**自動歸檔**設定可在選定天數後，自動將已完成任務移至已歸檔；如果偏好將所有完成項目保留在「已完成」，則可設為**永不**。

### 重複任務如何運作？

Mindwtr 支援兩種重複策略：

- **固定排程**：下一個日期以排程模式本身為準。
  範例：即使延遲完成，每 5 天一次仍會保持該節奏。
- **完成後重複**（彈性）：下一個日期從實際完成任務的時間計算。
  範例：今天完成後，下一次於今天起 5 天後到期。

在任務編輯器中設定重複（每天／每週／每月／每年）；若想採用彈性方式，再啟用**完成後重複**。

Mindwtr 只保留一個使用中的重複任務實例，不會預先建立該系列的未來「行事曆」項目；只有完成目前任務後，才建立下一個任務實例。

### 能在 Google Calendar 或 Apple 行事曆中看見 Mindwtr 任務嗎？

可以，支援單向推送：

- **Android/iOS：**將有日期的任務推送至裝置行事曆。Android 已透過 Google Calendar 驗證；iOS 請使用 Apple 行事曆中已可用的行事曆。最好在該帳號中建立名為 `Mindwtr` 的專用行事曆。
- **macOS 桌面版：**透過 EventKit 推送至 Apple 行事曆。

重複任務的*規則*不會匯出為原生重複行事曆事件，只會推送具體實例。請參閱[行事曆整合](/zh-Hant/use/calendar-integration)。

### Mindwtr 能與 Obsidian 整合嗎？

可以，桌面版可匯入 vault，並保留回到原始筆記的 deep link，讓任務直接跳至 Obsidian 中的來源筆記。請參閱 [Obsidian](/zh-Hant/power-users/obsidian)。

### 如何收集錯誤回報所需的日誌？

日誌預設關閉，只在要回報錯誤時啟用。

**桌面版（Tauri）：**
1. 前往**設定 → 資料**。
2. 啟用**除錯日誌**。
3. 重現問題。
4. 複製**日誌檔案**路徑，並將檔案附加至 GitHub issue。

Linux 預設日誌位置：`~/.local/share/mindwtr/logs/mindwtr.log`

**行動版：**
1. 前往**設定 → 資料**。
2. 啟用**除錯日誌**。
3. 重現問題。
4. 點選**分享日誌**，並附加至 GitHub issue。

日誌只保留在本機，並會在寫入前遮蔽常見憑證（密碼／token）。

### 可以用自然語言加入任務嗎？

可以！Mindwtr 支援快速新增語法：
- `@context`：加入情境
- `#tag`：加入標籤
- `!Area` 或 `/area:<name>`：指派至領域
- `%Person`：設定「指派給」人員（新建多字名稱使用 `%"Full Name"`）
- `/due:date`：設定截止日期（today、tomorrow、friday、next week 等）
- `/energy:low`、`/energy:medium` 或 `/energy:high`：設定精力程度
- `/note:text`：加入描述
- `/status`：設定狀態（`/next`、`/waiting`、`/someday`、`/done`、`/archived`、`/inbox`）
- `+ProjectName`：指派至專案

範例：`Call client /due:friday @phone`

### 什麼是情境？

情境是指出可在何處或使用什麼完成任務的標籤。例如 `@home`、`@work`、`@phone`、`@computer`。依情境篩選後，只會看見目前能做的任務。請參閱[情境與標籤](/zh-Hant/use/contexts-tags)。

### 情境與領域有何差異？

兩者回答不同問題：

- **領域**回答：「這項責任與我的生活或工作中哪個部分有關？」（工作、家庭、健康、某位客戶……）
- **情境**回答：「考量目前所在位置與手邊資源，我現在能做什麼？」（`@computer`、`@errands`……）

一項任務可以同時有三者：**領域：客戶 A**、**專案：網站改版**、**情境：@computer**。請參閱 [GTD 概覽](/zh-Hant/use/gtd-overview)。

### 如何快速收集任務？

**桌面版：**
- 使用全域快速鍵，從任何位置開啟快速新增
- Flatpak/Wayland 無法使用內建快速鍵時，將 `flatpak run tech.dongdongbh.mindwtr --quick-add` 綁定為自訂系統快速鍵
- 按一下系統匣圖示立即收集
- 在任何檢視的輸入欄輸入

**行動版：**
- 使用分享選單，從任何應用程式收集
- 加入主畫面小工具，一點即可收集
- 使用「收集箱」分頁的輸入欄

## 桌面版

### 有哪些鍵盤快速鍵？

Mindwtr 支援 Vim 與 Emacs 按鍵綁定預設。按 `?`（Vim）或 `Ctrl-h`（Emacs）可查看所有快速鍵。請參閱[桌面版鍵盤快速鍵](/zh-Hant/use/keyboard-shortcuts)。

### 如何變更主題？

前往「設定 → 外觀」，選擇淺色、深色或跟隨系統。

### 如何與手機同步？

1. 在設定中配置同步資料夾（指向 Dropbox、Syncthing 等）
2. 在行動版的「設定 → 同步」中選取同步資料夾
3. 兩個平台都會在資料變更及切換應用程式時自動同步

請參閱[資料與同步](/zh-Hant/data-sync/)。

### 支援通知嗎？

支援！Mindwtr 會傳送以下桌面通知：
- 截止日期提醒
- 開始時間通知
- 重複任務提醒

可將通知延後。

第一次啟用通知時，**macOS** 會要求權限。**Linux** 請確認有通知 daemon 正在執行。

## 行動版

### 支援哪些平台？

- **Android：**透過 Google Play 或下載 APK 提供完整支援
- **iOS：**已在 App Store 上架，也可透過 TestFlight 測試版取得。維持 App Store 發行仍需每年支付 Apple Developer 費用，因此贊助有助於讓 iOS 版持續供應。

### 為什麼桌面版與行動版的編輯方式不同？

Mindwtr 遵循各平台慣例：
- **桌面版：**按一下切換詳細資料；按兩下開啟完整編輯模式（可從任務的 `⋯` 選單就地重新命名標題）；按一下右鍵開啟快顯選單。
- **行動版：**點選一次即開啟編輯模式，滑動動作則處理快速變更。

這些模式讓應用程式在各平台上維持快速且符合使用習慣。

### 如何在 Android 安裝？

從 Google Play 安裝，或從 [GitHub Releases](https://github.com/dongdongbh/Mindwtr/releases) 下載 APK。系統提示時，請啟用「從未知來源安裝」。請參閱[行動版安裝](/zh-Hant/start/mobile-installation)。

### 如何從其他應用程式收集？

使用**分享選單**！在任何應用程式（瀏覽器、電子郵件、筆記）中查看內容時，點選「分享」並選擇 Mindwtr。Mindwtr 會開啟收集畫面，將分享內容放入筆記，讓你加入標題、調整欄位並儲存至收集箱。

### 有小工具嗎？

有！將 Mindwtr 小工具加入主畫面，即可快速收集及查看專注項目。

### 必須使用 AI 助理嗎？

不必。AI 助理是選用功能，預設關閉。Mindwtr 不使用它也能完整運作。
啟用後會使用你自己的 API 金鑰（BYOK）。請參閱 [AI 助理](/zh-Hant/power-users/ai-assistant)。

### 滑動動作如何運作？

在收集箱中，將任務向右滑即可標示為「已完成」。其他檢視可能有不同的滑動動作。

### 如何與桌面版同步？

1. 將備份匯出至同步資料夾（Google Drive、Syncthing 等）
2. 在「設定 → 同步」中選取該資料夾
3. 應用程式會在資料變更及進入背景時自動同步

請參閱[資料與同步](/zh-Hant/data-sync/)。

### 行動版會傳送通知嗎？

會！Mindwtr 會傳送以下通知：
- 截止日期提醒
- 開始時間通知
- 每日摘要提示
- 每週回顧提醒

可以直接從通知延後。
點選通知本文會開啟**回顧**畫面。

## 同步與資料

### 同步會讓資料遺失嗎？

不會。Mindwtr 使用最後寫入優先合併，保留各項目最新版本。軟刪除項目也會在裝置間正確同步。

### 可以使用多個同步服務嗎？

建議只使用一個同步資料夾以避免衝突。請選擇一項服務（Dropbox、Google Drive、Syncthing）並持續使用。

### 如何備份資料？

**桌面版：**使用**設定 → 資料 → 匯出備份**，或備份應用程式資料資料夾中的 `data.json`。
**行動版：**使用**設定 → 資料 → 匯出備份**儲存副本。

請參閱[備份與還原](/zh-Hant/data-sync/backup-restore)。

### 可以還原已刪除任務嗎？

目前沒有還原單一任務的介面，但可從先前的備份或復原快照還原本機資料。

請參閱[備份與還原](/zh-Hant/data-sync/backup-restore)。

### 可以從 TickTick 匯入嗎？

可以。Mindwtr 可從**設定 → 資料 → 從 TickTick 匯入**，匯入 TickTick CSV 與 ZIP 備份。

- TickTick 資料夾會成為 Mindwtr 中的領域
- TickTick 清單會成為 Mindwtr 中的專案
- 可行時保留檢查清單內容、標籤、日期、優先順序與支援的重複規則

請參閱 [TickTick 匯入](/zh-Hant/import/ticktick)。

### 可以從 Todoist 匯入嗎？

可以。Mindwtr 可從**設定 → 資料 → 從 Todoist 匯入**，匯入 Todoist CSV 匯出與 ZIP 備份。

- Todoist 專案會成為 Mindwtr 中的專案
- 子任務會成為檢查清單項目
- 已指派專案的使用中任務會留在其專案；未指派的使用中任務則保留供收集箱處理

請參閱 [Todoist 匯入](/zh-Hant/import/todoist)。

### 可以從 DGT GTD 匯入嗎？

可以。Mindwtr 可從**設定 → 資料 → 從 DGT GTD 匯入**，匯入 DGT GTD JSON 匯出與 ZIP 封存。

- DGT 資料夾會成為 Mindwtr 中的領域
- DGT 專案會成為 Mindwtr 中的專案
- DGT 檢查清單會成為檢查清單任務
- DGT 情境與標籤會保留

不支援的 DGT 重複模式只會匯入一次，原始重複文字會保留在描述中。

請參閱 [DGT GTD 匯入](/zh-Hant/import/dgt-gtd)。

### 可以從 OmniFocus 匯入嗎？

可以。Mindwtr 可從**設定 → 資料 → 從 OmniFocus 匯入**，匯入 OmniFocus CSV 匯出及 Omni Automation JSON / ZIP 匯出。

- 中繼資料可用時，OmniFocus 資料夾可成為 Mindwtr 中的領域
- 需要時，OmniFocus 專案會成為 Mindwtr 中的專案
- 獨立 OmniFocus 行動會留在專案之外
- 從 JSON 路徑保留支援的筆記、標籤、延後日期、截止日期、完成狀態與重複規則
- 簡單的巢狀任務可成為檢查清單項目，較深的階層則扁平化並保留原始路徑

如果重複規則很重要，請使用 Omni Automation JSON / ZIP 匯出，而不是 CSV。Mindwtr 沒有直接對應欄位時，規劃日期與持續時間文字會保留在描述中。

請參閱 [OmniFocus 匯入](/zh-Hant/import/omnifocus)。

### 可以從 Apple 提醒事項匯入嗎？

可以，支援 iPhone 與 iPad。Mindwtr 可從**設定 → 資料 → 從 Apple 提醒事項匯入**，匯入尚未完成的 Apple 提醒事項。

- 選擇要匯入的「提醒事項」清單
- 匯入的提醒事項會成為 Mindwtr 任務
- Mindwtr 確認匯入後，可選擇從 Apple 提醒事項刪除提醒

Apple 提醒事項是單向匯入，而非同步。請參閱 [Apple 提醒事項匯入](/zh-Hant/data-sync/#apple-提醒事項匯入-ios)。

## 疑難排解

### 應用程式無法啟動（桌面版 Linux）

請確認已安裝 WebKitGTK：
```bash
# Arch
sudo pacman -S webkit2gtk-4.1

# Debian/Ubuntu
sudo apt install libwebkit2gtk-4.1-0
```

### 為什麼 AUR 安裝或建置目錄這麼大？

除非明確想從原始碼建置，否則 Arch 系列發行版請使用 `mindwtr-bin`：

```bash
yay -S mindwtr-bin
```

`mindwtr-bin` 會安裝預先建置的 GitHub 發行套件，應是體積小且快速的 AUR 路徑。原始碼套件 `mindwtr` 會在本機建置桌面應用程式，必須下載 Tauri、Rust、Bun 與 React 應用程式的建置相依套件，建置期間可能使用大量磁碟空間。

原始碼套件應擷取發行標籤封存，而不是完整 Git 歷史。如果 AUR helper 似乎下載了非常大的 Git checkout，請確認二進位套件路徑安裝的是 `mindwtr-bin`；或回報原始碼套件行為，讓 AUR 配方得以修正。

### 應用程式啟動時當機（行動版）

嘗試清除應用程式資料：
1. 前往「設定 → 應用程式 → Mindwtr 」
2. 點選「儲存空間 → 清除資料」
3. 重新開啟應用程式

注意：這會刪除本機資料。

### 任務沒有同步

1. 檢查同步資料夾是否可存取
2. 確認同步服務正在執行
3. 檢查檔案權限
4. 在設定中嘗試手動同步

### 通知無法運作

**桌面版：**
- 檢查系統通知設定
- 確認應用程式有通知權限

**行動版：**
- 在裝置設定中授予通知權限
- 檢查應用程式通知設定
