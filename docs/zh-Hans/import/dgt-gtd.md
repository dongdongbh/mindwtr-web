# 从 DGT GTD 导入

Mindwtr 可以导入 DGT GTD 导出文件，让你无需手动重建整个系统即可迁移。

支持的来源：

- DGT GTD **JSON** 导出
- 包含已导出 JSON 文件的 DGT GTD **ZIP** 归档

桌面端和移动端均可从**设置 → 数据 → 从 DGT GTD 导入**执行导入。

---

## 导入内容

Mindwtr 以 GTD 优先的方式将 DGT GTD 导出映射到自身模型：

- DGT 文件夹成为 **Mindwtr 中的领域**
- DGT 项目成为 **Mindwtr 中的项目**
- DGT 检查清单成为**检查清单任务**
- 检查清单项目仍为**检查清单项目**
- DGT 情境成为**情境**
- DGT 标签成为**标签**

当 Mindwtr 能安全表示原状态时，任务会保留映射后的状态。独立 DGT 任务可以不属于任何项目，方便你在导入后整理，而不是在导入过程中强行增加结构。

---

## 支持的 DGT 数据

- 任务标题
- 备注/描述
- 优先级
- 截止日期
- 检查清单项目
- 文件夹、项目、情境和标签
- 已完成任务
- 支持的重复规则，例如简单的每日/每周/每月/每年计划和部分按间隔重复

不受支持的 DGT 重复模式只会导入一次，并在描述中保留原始重复文字，便于你在 Mindwtr 中手动调整重复规则。

---

## 导入流程

1. 打开**从 DGT GTD 导入**
2. 选择 DGT GTD JSON 或 ZIP 文件
3. 检查预览摘要
4. 确认导入

如果平台支持，导入前 Mindwtr 会保存当前本地数据的恢复快照。

导入后：

- 按需创建新领域和项目
- 独立任务保留，供之后整理
- 对有损的重复规则映射或跳过的归档条目显示警告

---

## ZIP 导出说明

Mindwtr 会读取归档中的第一个有效 DGT JSON 导出。

Mindwtr 会跳过：

- 嵌套的 ZIP 文件
- 归档中的非 JSON 文件
- 无法安全解析的格式错误 JSON 文件

---

## 提示

- 如果想先验证映射，可从较小的 DGT GTD 导出开始
- 验证导入结果前请保留恢复快照
- 两次导入同一导出可能产生重复任务

另请参阅[数据与同步](/zh-Hans/data-sync/)以及[备份与恢复](/zh-Hans/data-sync/backup-restore)。
