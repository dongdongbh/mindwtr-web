# 从 TickTick 导入

如水可以导入 TickTick 备份，让你无需手动重建整个系统即可迁移。

支持的来源：

- TickTick **CSV** 备份
- 包含 CSV 导出的 TickTick **ZIP** 备份

桌面端和移动端均可从**设置 → 数据 → 从 TickTick 导入**执行导入。

## 从 TickTick 导出

TickTick 通过 Web 应用创建备份：

1. 在浏览器中打开 TickTick 并登录。
2. 点击左上角头像。
3. 打开**设置 → 账户 → 备份与导入**。
4. 选择**生成备份**。
5. 保存 TickTick 下载的 CSV 或 ZIP 文件。

然后在如水中打开**从 TickTick 导入**并选择该文件。

## 导入内容

如水以 GTD 优先的方式将 TickTick 备份映射到自身模型：

- TickTick 文件夹成为**如水领域**
- TickTick 清单成为**如水项目**
- 父任务与子行可成为**检查清单任务**
- 检查清单内容保留为检查清单项目
- TickTick 标签成为**如水标签**
- 存在优先级时会进行映射
- 在能够安全读取时，保留截止日期、开始日期、全天日期和时区
- 支持的重复规则导入为如水重复规则
- 存在完成和归档状态时会予以保留
- 任务备注/内容保留在描述中

导入的活跃任务仍可按 GTD 流程处理，不会被强制设为如水无法从备份推断的状态。

## 导入流程

1. 打开**从 TickTick 导入**
2. 选择 TickTick CSV 或 ZIP 备份
3. 检查预览摘要
4. 确认导入

如果平台支持，导入前如水会保存当前本地数据的恢复快照。

导入后：

- 按需创建新领域和项目
- 从备份添加任务、检查清单、标签、日期和重复规则
- 对跳过的行、不支持的归档条目，或如水无法安全映射的字段显示警告

## TickTick 备份说明

TickTick 备份会在实际 CSV 标头之前包含元数据行。如水会自动检测真正的标头，无需预先编辑文件。

如水会跳过：

- 嵌套的 ZIP 文件
- 归档中的非 CSV 文件
- 无法安全解析的格式错误行

TickTick 备份不包含应用状态的所有细节。附件和部分应用专属显示信息可能需要手动迁移。

## 提示

- 验证导入结果前请保留原始 TickTick 备份
- 先使用完整备份，但要通过预览检查映射是否符合预期
- 两次导入同一备份可能产生重复任务
- 如果要从其他应用迁移，请参阅[从其他应用导入数据](/zh-Hans/import/)了解备用路径

另请参阅[数据与同步](/zh-Hans/data-sync/)以及[备份与恢复](/zh-Hans/data-sync/backup-restore)。
