# 備份與還原

如水將工作資料儲存在本機，也能匯出 JSON 備份，便於攜帶、修復及遷移。

還原是一種**取代本機資料**的流程：

- 由你選擇備份 JSON 檔案
- 如水會在變更任何內容前驗證檔案
- 可行時先建立復原快照
- 所選備份會取代目前的本機資料集

這讓還原保持簡單、可預期；它不是合併操作。

---

## 匯出備份

### 桌面版

1. 開啟**設定 → 資料**
2. 在**資料傳輸**中選擇**匯出備份**
3. 將 JSON 檔案存至所需位置

### 行動版

1. 開啟**設定 → 資料**
2. 點選**匯出備份**
3. 儲存或分享 JSON 檔案

備份格式與如水內部 `data.json` 結構相容。

---

## 從備份還原

### 桌面版

1. 開啟**設定 → 資料**
2. 在**資料傳輸**中選擇**還原備份**
3. 選擇如水備份 JSON 檔案
4. 檢查摘要並確認還原

還原前，若 Tauri 執行階段可用，桌面版會在本機快照目錄建立資料快照。

### 行動版

1. 開啟**設定 → 資料**
2. 點選**還原備份**
3. 選擇如水備份 JSON 檔案
4. 檢查摘要並確認還原

還原前，行動版會在應用程式儲存空間保存本機復原快照。

---

## 復原快照

如水會在還原與匯入操作前自動建立復原快照。

- **桌面版：**快照位於**設定 → 同步 → 復原快照**
- **行動版：**快照位於**設定 → 同步 → 復原快照**

若還原了錯誤檔案，或想回復本機的匯入／還原操作，可使用這些快照。

---

## 驗證規則

如水會在還原前驗證所選 JSON 檔案：

- 檔案必須是有效 JSON
- 必須符合如水的資料形狀
- 可用時會顯示項目數量與備份中繼資料
- 版本不符會顯示警告，而不會默默失敗

驗證失敗時，還原會遭阻止，目前資料維持不變。

---

## 還原不會執行的操作

- **不會**將備份與目前本機資料合併
- **不會**只還原單一任務或專案
- 在下一次同步週期前，**不會**自行覆寫遠端同步服務

若使用同步，可將還原視為先取代目前的本機狀態；後續同步行為取決於所用後端及下一部執行同步的裝置。

---

## 提示

- 除同步外，也請定期手動匯出
- 只還原你信任的備份
- 使用檔案同步時，請等正確的 `data.json` 完成複製，再同步其他裝置

另請參閱[資料與同步](/zh-Hant/data-sync/)。
