# 情境与标签

情境会根据你所在的位置、拥有的工具或所需的精神状态，帮助你查看**此时此刻**能做什么。

---

## 什么是情境？

在 GTD 中，情境是完成任务所需的条件：地点、工具或人员。

按情境筛选时，你只会看到当前处境下实际可以完成的任务。

## 情境与领域

这是刚开始使用 GTD 时最容易混淆的概念之一。

| 概念 | 它回答的问题 | 经常变化吗？ | 示例 |
| ----------- | --------------------------------------------------- | -------------- | ----------------------------- |
| **情境** | 我现在身处这里、拥有这些条件，可以做什么？ | 是 | `@computer`、`@phone`、`@home` |
| **领域** | 这是哪项持续责任的一部分？ | 否 | 工作、家庭、健康、财务 |

关键区别：

- **情境**关注的是*执行限制*。它们会在一天中不断变化。
- **领域**关注的是*责任类别*。只要你生活或工作中的这部分仍然存在，它们就会一直保留。

### 示例：承包商或客户工作

如果你与多个客户合作，为每个客户使用一个**领域**完全合理。

- **领域：**客户 A
- **项目：**后端迁移
- **任务：**审查 API 规范
- **情境：**`@computer`

这样就能回答两个不同的问题：

- 如果想查看与某个客户相关的所有内容，请按**领域**筛选。
- 如果想查看现在能做什么，而不考虑客户，请按**情境**筛选。

要直接创建或编辑领域，请参阅[领域与人员](/zh-Hans/use/areas-people)。

## 地点情境（@）

如水包含以下预设地点情境：

| 情境 | 适用情况 |
| ----------- | ---------------------------------------- |
| `@home` | 需要在家完成的任务 |
| `@work` | 在办公室/工作场所完成的任务 |
| `@errands` | 外出时完成的任务（购物、赴约） |
| `@agendas` | 在会议或通话中讨论的事项 |
| `@computer` | 需要使用计算机的任务 |
| `@phone` | 需要打电话的任务 |
| `@anywhere` | 没有地点要求的任务 |

### 示例

- `Call mom @phone`
- `Fix leaky faucet @home`
- `Buy office supplies @errands`
- `Deploy update @computer @work`

---

## 标签（#）

如水包含用于按精神状态、模式或主题筛选的标签：

| 标签 | 适用情况 |
| ------------ | --------------------------------- |
| `#focused` | 需要集中注意力的深度工作 |
| `#lowenergy` | 疲惫时可以完成的简单任务 |
| `#creative` | 头脑风暴或构思 |
| `#routine` | 重复性/机械性任务 |

### 示例

- `Write proposal #focused @computer`
- `File receipts #lowenergy @home`
- `Brainstorm campaign ideas #creative`
- `Process expenses #routine @computer`

### 为什么使用标签？

你的工作效率会在一天中变化：
- **早晨：**注意力高度集中，处理 #focused 任务
- **午饭后：**精力较低，完成 #lowenergy 或 #routine 任务
- **创意时间：**有灵感时，处理 #creative 任务

## 在如水中使用情境

### 添加情境

**快速添加语法：**
```
Task title @context1 @context2
Research topic #focused @computer
```
情境和标签名称接受 Unicode 字母和数字（包括 CJK 字符和带重音符号的字符）。

**编辑任务：**
1. 打开任务编辑器
2. 在“情境”字段中添加情境（用逗号分隔）

### 按情境筛选

**桌面端：**
1. 前往**下一步行动**或**情境**视图
2. 单击情境标签块进行筛选

**移动端：**
1. 前往**下一步行动**标签页，或从抽屉菜单进入**情境**
2. 点按一个情境进行筛选

### 按情境或标签分组

任务列表工具栏还包含**分组**控件。在浏览较长的收集箱、等待中、将来/也许、参考资料、已完成或下一步行动列表时，可用它按情境、领域、项目或标签分组。

如果你偏好以标签为中心的工作流程，并将标签作为主要分类层，按标签分组会很有用。

## 自定义情境

你可以创建自己的情境：

- `@Bob`：要与 Bob 讨论的事项
- `@waiting-on-vendor`：等待供应商回复
- `@car`：在车里要做的事情
- `@morning`：晨间例行事项

### 创建自定义情境

添加任务时，只需输入新情境：
```
Review contract @legal
```

该情境会被添加，并可用于筛选。

### 管理已保存的情境和标签

需要清理或重命名可重复使用的元数据时：

- 打开**设置 → 管理**
- 在一个位置编辑**情境**、**标签**、**领域**和**人员**
- 用它合并重复项、统一命名或删除不再使用的值

也可以在分配领域和人员时创建它们。在**领域**选择器或**分配给**字段中输入内容，如果没有完全匹配项，请选择**创建**或**新建人员**。

### 批量编辑情境

在桌面端，你可以一次更新多项任务的情境：

1. 在列表视图中进入**选择**模式
2. 选择要编辑的任务
3. 使用批量操作栏中的**添加情境**或**移除情境**

如果想在每周回顾或项目规划后重新分类一整批任务，此功能会很有用。

## 分层情境与标签

使用斜杠表示法，以嵌套结构组织情境：

| 语法 | 示例任务 |
| ----------------- | --------------------------- |
| `@work/meetings` | Meeting prep @work/meetings |
| `@home/garage` | Fix shelf @home/garage |
| `#health/fitness` | Morning run #health/fitness |
| `#work/admin` | File reports #work/admin |

### 父级包含子级筛选

按父级情境筛选时，会包含所有子级：

| 筛选条件 | 显示内容 |
| --------- | -------------------------------------------- |
| `@work` | `@work`、`@work/meetings`、`@work/calls` |
| `#health` | `#health`、`#health/fitness`、`#health/diet` |

这样既能进行宽泛的高层级筛选，又能保持具体的组织结构。

### 优点

- **条理清晰**：对相关情境进行分组，避免杂乱
- **灵活**：既可宽泛筛选，也可具体筛选
- **向后兼容**：简单情境仍可正常使用

## 情境最佳实践

### 从简单开始

开始时只使用少量情境：
- @home
- @work
- @errands
- @computer

仅在需要时添加更多情境。

### 保持一致

使用相同的拼写和格式：
- ✓ `@home`（始终如此）
- ✗ `@Home`、`@house`、`home`

### 组合情境

任务可以有多个情境：
- `@computer @work`：需要在工作场所使用计算机
- `@phone @anywhere`：可以在任何地方打电话
- `#focused @home`：在家进行深度工作

### 人员情境

为议程事项添加人员情境：
```
Discuss project timeline @Sarah
Ask about budget @manager
```

见到 Sarah 时，搜索 `@Sarah` 即可找到所有事项。

## 情境工作流程

### 晨间规划

1. 确认今天会去哪些地方
2. 按相关情境筛选下一步行动
3. 选择你的重点任务

### 更换地点

当你移动到其他地点时：
1. 按新情境筛选（@work → @home）
2. 查看在这里能做什么
3. 选择下一个任务

### 意外互动

当有人意外来电/来访时：
1. 搜索对方的姓名/情境
2. 查看议程事项
3. 处理等待事项

## 情境与标签

在如水中，`@contexts` 和 `#tags` 都可以筛选任务，但它们回答的问题不同：

| 符号 | 它回答的问题 | 典型用途 | 示例 |
| ------ | ----------------------------- | ------------------------------------ | -------------------------- |
| `@` | 我可以在哪里/用什么完成它？ | 地点、工具或人员限制 | @home、@work、@phone |
| `#` | 这与什么有关？ | 主题、精力、模式或标签 | #focused、#lowenergy |

### 实用指南

- 使用 **`@` 情境**作为*执行筛选条件*（你需要的地点/工具/人员）。
- 使用 **`#` 标签**进行*分类*（精力水平、主题或跨项目分组）。
- 两者都支持**层级结构**（例如 `@work/meetings`、`#health/fitness`）。
