# 你可以信赖什么

软件项目会终结，公司可能被收购，资金也可能耗尽，应用会停止运营。当任务系统承载着你多年来的承诺时，在依赖它之前值得认真考虑这些风险。本页列出保护你的各项机制。每一项都是你今天就能验证的事实，而不是对未来的许诺。

## 许可证确保每个版本都可继续使用

如水采用 [AGPL-3.0](https://www.gnu.org/licenses/agpl-3.0.html) 发布。每个已发布版本都会永久遵循这些条款。无论项目今后如何，你机器上的版本都能继续运行，源代码也始终可供任何人构建、分叉或维护。

## 无需依赖账户

核心工作流程在你的设备上运行，无需账户。不必注册，也不用先通过登录才能记录、明确和回顾。如水运营的服务不会持有你的数据，因此不存在某台服务器关闭后把你挡在数据之外的风险。

## 数据是你能读取的普通文件

如水在你的设备上以 SQLite 和 JSON 保存数据。导出和备份是内置功能，也会一直如此：这里没有付费层级，自然不会把它们锁在付费墙后。[数据库结构](/zh-Hans/developers/database-schema)已有公开文档，因此数据格式始终开放，而非专有格式。在实际可行时，存储变更会随迁移一同发布，让升级能带着你的数据向前演进。

## 同步方式由你选择

是否同步，以及通过什么方式同步，都由你决定。可选项包括不同步、iCloud、Dropbox、WebDAV 或你自己托管的服务器。无论哪种方式，数据都只经过你已经掌控的基础设施，中间没有如水运营的服务留存副本。请参阅[数据与同步](/zh-Hans/data-sync/)了解各选项的工作方式。

## AI 在你开启前始终关闭

AI 助手是可选功能，默认关闭。它使用你自己的密钥和服务提供方；提供方也可以是 llama.cpp 或 Ollama 等本地端点，因此数据完全不必离开设备。请参阅 [AI 助手](/zh-Hans/power-users/ai-assistant)。

## 架构为何就是保障

订阅制笔记服务要求你相信背后的公司会持续健康经营并按时支付账单。如水需要的信任更少，因为它依赖的东西也更少。文件属于你，格式已有文档，许可证授权无法撤回，核心循环也不需要服务器。即使明天停止开发，你已经拥有的一切仍会继续工作。

如需详细了解如水如何处理数据，请阅读[隐私政策](https://mindwtr.app/privacy)。
