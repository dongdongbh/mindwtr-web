# Calendar integration

Mindwtr uses calendar information to help separate hard landscape from flexible next actions.

## Concepts

Calendar items are time-specific commitments. GTD next actions are usually flexible unless they truly belong on the calendar. Mindwtr keeps that distinction visible.

## Supported calendar paths

| Surface | Typical path |
| --- | --- |
| Mobile | Device calendars and mobile task-to-calendar actions where supported. |
| macOS | Apple Calendar integration where available. |
| Desktop and web | ICS URLs for readable external calendar feeds. |

## Scheduling workflow

Use dates and calendar placement deliberately. If a task can be done any time this week, it usually belongs in Next Actions or Focus rather than becoming a false calendar appointment.

## ICS requirements

Private calendar URLs should be treated as sensitive. They often include tokens. Do not commit them to git, paste them into public issues, or share them in screenshots.
