# 高级用户集成

Mindwtr 可以连接周边工具，同时不放弃本地优先模式。本页按你想完成的事情对集成进行分组。每项集成都用四个问题说明它需要你给予多少信任：

- **它能读取什么**数据
- **它能更改什么**
- **它在哪里运行**
- **哪些内容会离开设备**

这里的所有集成都是可选功能，在你设置之前保持关闭。完整说明请查看各链接页面。

## 从任何地方记录

### Apple 快捷指令

通过 Siri、“快捷指令”应用、操作按钮或自动化把任务发送到收集箱，也可直接跳到 GTD 清单。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 什么也不能读取。它只记录任务和打开清单。 |
| 它能更改什么 | 添加收集箱记录，不能编辑、完成或删除任务。 |
| 它在哪里运行 | 在你的 iPhone 或 iPad 上。 |
| 哪些内容会离开设备 | 没有。记录通过 Mindwtr 的常规存储和同步进行；Swift 绝不会直接写入任务。 |

[Apple 快捷指令指南](/zh-Hans/power-users/apple-shortcuts)

### 邮件收集

让桌面应用监视一个邮件文件夹，进入该文件夹的每封邮件都会成为收集箱任务。在手机上则可把邮件分享给 Mindwtr。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 只读取你指定的一个邮件文件夹，不接触邮箱其他部分。 |
| 它能更改什么 | 创建收集箱任务，绝不会修改、移动或删除邮件。 |
| 它在哪里运行 | 在桌面设备上，通过 IMAP 直接连接邮件服务提供方，没有第三方中转。 |
| 哪些内容会离开设备 | 不会发送给第三方。自托管方案只把你映射的字段发送到自己的服务器。 |

[邮件收集指南](/zh-Hans/power-users/email-capture)

## 连接个人工具

### Obsidian

从 Obsidian 库中读取任务，在文件变化时持续刷新，并在 Obsidian 中打开源笔记。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 桌面端中你允许访问的文件夹内的库文件。 |
| 它能更改什么 | 对受支持的格式切换复选框或 TaskNotes 状态，不会重写笔记。 |
| 它在哪里运行 | 仅限桌面端。它不是 Mindwtr 的同步后端。 |
| 哪些内容会离开设备 | 没有。笔记留在你的库中。 |

[Obsidian 指南](/zh-Hans/power-users/obsidian)

## 在本机上自动化和编写脚本

### 本地 API

为需要读写任务的脚本、快捷方式和小工具提供 REST API。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 所有任务数据，以及项目与领域的只读访问权限。 |
| 它能更改什么 | 创建、更新、完成、归档、软删除和恢复任务。项目和领域保持只读。 |
| 它在哪里运行 | 在桌面应用内部，绑定到 `127.0.0.1`，每个请求都需要 bearer token。 |
| 哪些内容会离开设备 | 没有。请求仅在 localhost 上传输。 |

[本地 API 指南](/zh-Hans/power-users/local-api)

## 连接 AI 代理

### MCP 服务器

让 Claude、Codex 或 Gemini 等 AI 客户端通过 Model Context Protocol 访问 Mindwtr 数据。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 任务、项目、分区、领域和人员。 |
| 它能更改什么 | 默认只读。使用 `--write` 后可添加、更新、完成和删除。 |
| 它在哪里运行 | 作为本地子进程直接访问 SQLite 文件，或连接你自己托管的 Cloud 服务器。 |
| 哪些内容会离开设备 | 只有所连接 AI 客户端收到的内容。Cloud 模式下，编辑通过经过验证的 API 发送到你自己的服务器。 |

[MCP 服务器指南](/zh-Hans/power-users/mcp)

## 应用内 AI

### AI 助手

可选的自备密钥功能，用于明确任务、拆解任务和回顾长期未更新的事项。

| 问题 | 答案 |
| --- | --- |
| 它能读取什么 | 只读取当前操作需要的任务数据。 |
| 它能更改什么 | 只更改你检查并批准的内容。 |
| 它在哪里运行 | 通过你配置的服务提供方运行，也可以是 llama.cpp 或 Ollama 等本地端点。 |
| 哪些内容会离开设备 | 只把范围受限的数据发送给所选服务提供方。如果服务在本地运行，则没有内容离开设备。 |

[AI 助手指南](/zh-Hans/power-users/ai-assistant)
