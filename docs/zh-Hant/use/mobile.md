# 使用指南：行動版

Mindwtr 行動版以 React Native 與 Expo 建置。Android 提供完整支援；iOS 已在 App Store 上架，也可透過 TestFlight 測試版取得。

## 概覽

行動版以底部分頁呈現核心流程，其他檢視則集中在「選單」頁面。

---

## 互動模式

- **點選**以開啟及編輯任務。
- **滑動**執行快速動作（見下方「滑動動作」）。
- **分享選單**將項目直接加入收集箱。

## 導覽

### 底部分頁

| 分頁               | 說明                           |
| ------------------ | ------------------------------ |
| 📥 **收集箱**      | 收集及處理傳入項目             |
| 🎯 **專注**        | 每日儀表板與下一步行動         |
| ➕ **快速收集**    | 迅速加入任務或音訊筆記         |
| 📝 **回顧**        | 每日 + 每週回顧                |
| ☰ **選單**         | 開啟專案、看板、行事曆等       |

### 選單分頁

點選**選單**分頁即可開啟其他檢視：

- 📋 **看板**：可手動拖曳排序、搜尋，並依情境／標籤／日期／專案篩選的看板檢視
- 🗓️ **行事曆**：依時間顯示
- 📁 **專案**：需要多個步驟的成果
- 🏷️ **情境**：依情境篩選
- ⏳ **等待中**：已委派項目
- 💭 **將來/也許**：未來想法
- 📚 **參考資料**：參考內容
- ✅ **已完成**：最近完成的任務
- 📦 **已歸檔**：從一般清單收存的完成任務
- 🗑️ **垃圾桶**：已刪除的任務與專案
- ⚙️ **設定**：應用程式偏好

開啟**已歸檔**並點選**選取**，即可管理多項任務。點選**全選**或挑選任務，再還原至收集箱或移至垃圾桶。Mindwtr 只會從垃圾桶永久刪除任務。

開啟**垃圾桶**並點選**選取**，即可管理多項已刪除任務與專案。點選**全選**或挑選項目，再一步完成還原或永久刪除。永久刪除一定會要求確認。

## 全域搜尋

點選標題列的**搜尋圖示**，開啟「全域搜尋」。

搜尋預設隱藏「已完成」與「已歸檔」項目。若隱藏項目符合查詢，會顯示可點選的「已完成及已歸檔中另有 N 項」提示。點選即可納入，或在搜尋篩選中啟用**包含已完成及已歸檔任務**。

### 搜尋運算子

使用運算子進行強大的篩選：

| 運算子      | 範例               | 說明                         |
| ----------- | ------------------ | ---------------------------- |
| `status:`   | `status:next`      | 依任務狀態篩選               |
| `-status:`  | `-status:done`     | 排除狀態                     |
| `context:`  | `context:@home`    | 依情境篩選                   |
| `tag:`      | `tag:#focused`     | 依標籤篩選                   |
| `assigned:` | `assigned:Tom`     | 依受指派者篩選               |
| `project:`  | `project:HomeReno` | 依專案篩選                   |
| `location:` | `location:office`  | 依任務地點篩選               |
| `where:`    | `where:office`     | 任務地點別名                 |
| `id:`       | `id:abc123`        | 尋找確切任務 ID              |
| `-id:`      | `-id:abc123`       | 排除確切任務 ID              |
| `due:`      | `due:today`        | 在該日到期的任務             |
| `due:<=`    | `due:<=7d`         | 未來 7 天內到期的任務        |
| `start:`    | `start:>=tomorrow` | 自該日開始的任務             |
| `created:`  | `created:>=30d`    | 過去 30 天建立的任務         |
| `OR`        | `@home OR @work`   | 符合任一條件                 |

### 已儲存搜尋

「已儲存搜尋」是可重複使用之搜尋查詢的捷徑。

1. 輸入搜尋查詢
2. 點選**「儲存搜尋」**
3. 為搜尋命名
4. 從**選單 → 已儲存搜尋**開啟

**刪除方式：**開啟已儲存搜尋，點選標題列的垃圾桶圖示。

「已儲存搜尋」與「專注」中的**已儲存篩選條件**彼此獨立。「專注」篩選條件會儲存專案、情境、標籤、優先順序、精力及預估時間等條件，供「專注」檢視使用。

## 快速收集

Mindwtr 在行動裝置上提供多種快速收集任務的方式。

收集畫面以輸入為先。語法說明收在小型「？」切換鈕後方，讓介面保持整潔。

需要連續收集時，開啟收集面板中的**繼續新增**切換鈕：每次按 Enter 都會儲存任務，並讓面板與鍵盤保持開啟，準備輸入下一項。此切換鈕會在多次收集之間保持開啟，直到你自行關閉。

### 分享選單

使用分享選單從任何應用程式收集任務：

1. 在任何應用程式（瀏覽器、電子郵件、筆記）找到想收集的內容
2. 點選**分享**按鈕
3. 從分享選項選擇**Mindwtr**
4. Mindwtr 會開啟收集畫面，並將分享內容附加為筆記
5. 加入標題、調整欄位，再儲存至收集箱

分享檔案（PDF、圖片、影片或錄音）時，會將它複製至 Mindwtr，成為新任務的附件，並以檔名作為初始標題。一次最多可分享六個檔案。

適合：
- 儲存日後閱讀的文章
- 將電子郵件收成任務
- 加入瀏覽網頁時找到的連結
- 將 PDF 或照片收為日後處理的收集箱任務

### 主畫面小工具

將 Mindwtr 小工具加入主畫面，以便快速開啟：

1. 長按主畫面
2. 選擇**小工具**
3. 找到並加入**Mindwtr**小工具
4. 點選小工具以開啟快速收集或查看專注項目

### iOS 鎖定畫面小工具

在 iOS 16 及更新版本，Mindwtr 提供鎖定畫面小工具，不必解鎖手機即可顯示目前專注任務：

1. 長按鎖定畫面並點選**自訂**
2. 選取小工具區域並加入**Mindwtr 專注**小工具
3. 選擇矩形小工具（顯示最優先專注任務與「專注」標題）、時鐘上方的行內單列小工具，或圓形專注任務計數器

點選任一小工具都會開啟「專注」畫面。鎖定畫面小工具會遵循系統的單色鎖定畫面樣式。

### Android 快速設定方塊

在 Android 上，將 Mindwtr 收集方塊加入快速設定，即可滑動一次收進收集箱：

1. 開啟 Android 快速設定編輯模式。
2. 加入**Mindwtr**方塊。
3. 點選方塊開啟快速收集。

### Android 通知列快速收集

若要從鎖定畫面以一次滑動最快收集，可釘選常駐的**快速收集**通知：

1. 開啟**設定 → 通知**。
2. 開啟**通知列快速收集**。
3. 隨時點選通知，開啟快速收集。

此通知不會發出聲音，並停留在通知欄底部而不造成干擾。關閉切換鈕即可移除。（僅限 Android；iOS 不支援常駐通知，請改用小工具或「捷徑」。）

### Android 語音 App Actions

Android 組建會向支援的助理公開收集動作，包括透過 Android App Actions 路由的 Gemini／Assistant 介面。語音建立的收集項目會開啟 Mindwtr 確認流程，讓你在儲存前檢查標題與筆記。

### Android 情境自動化 Intent

Tasker、MacroDroid 或 Phone Profiles 等自動化應用程式可以啟用 Mindwtr 中的情境。只在背景觸發時，請使用 Android broadcast 形式。啟用後，Mindwtr 會檢查目前可用且符合情境的 `/next` 行動；只有確實有工作時才傳送通知。點選通知會開啟相符的「情境」檢視。

Android broadcast 形式：

| 欄位 | 值 |
| --- | --- |
| Package | `tech.dongdongbh.mindwtr` |
| Class | `tech.dongdongbh.mindwtr.ContextAutomationReceiver` |
| Target | Broadcast Receiver |
| Activate action | `tech.dongdongbh.mindwtr.action.ACTIVATE_CONTEXT` |
| Deactivate action | `tech.dongdongbh.mindwtr.action.DEACTIVATE_CONTEXT` |
| String extra | `context=parents` or `context=@parents` |

ADB 範例：

```bash
adb shell am broadcast -n tech.dongdongbh.mindwtr/.ContextAutomationReceiver -a tech.dongdongbh.mindwtr.action.ACTIVATE_CONTEXT --es context parents
adb shell am broadcast -n tech.dongdongbh.mindwtr/.ContextAutomationReceiver -a tech.dongdongbh.mindwtr.action.DEACTIVATE_CONTEXT --es context parents
```

URL 形式：

| URL | 動作 |
| --- | --- |
| `mindwtr://contexts?token=%40parents&contextAction=activate` | 啟用 `@parents` |
| `mindwtr://contexts?token=%40parents&contextAction=deactivate` | 停用 `@parents` |

URL 範例：

```bash
adb shell am start -a android.intent.action.VIEW -d 'mindwtr://contexts?token=%40parents&contextAction=activate' tech.dongdongbh.mindwtr
```

注意事項：
- 透過 URL 啟動可能會開啟 Mindwtr。若自動化應留在背景，請使用 broadcast receiver 形式。
- 情境名稱會正規化為 `@context`，因此 `parents` 與 `@parents` 都會符合 `@parents`。
- 階層式情境會比對所選情境下的內容，因此 `@parents` 也符合 `@parents/errands`。
- 若目前沒有符合情境的可用 `/next` 行動，Mindwtr 不會發出通知。
- 停用目前是無聲的 no-op。它會確認自動化離開觸發，並保留給日後的使用中情境狀態；不會刪除、隱藏或變更任務。
- Android 上處理情境自動化 URL 與 Intent 後，Mindwtr 會回到背景。想開啟相符的「情境」檢視時，請點選通知。
- Mindwtr 不會自行偵測地點或裝置狀態；觸發條件由自動化應用程式負責。

### Apple 捷徑

在 iPhone 與 iPad 上，Mindwtr 提供原生 Apple 捷徑動作，供收集箱收集，以及開啟「專注」、「等待中」、「將來」、「專案」、「回顧」與「行事曆」等 GTD 檢視。設定、範例與 v1 限制請參閱 [Apple 捷徑](/zh-Hant/power-users/apple-shortcuts)。

### URL Scheme 快速收集（iOS 捷徑／Android 自動化）

Mindwtr 註冊了 `mindwtr://` URL scheme，因此可從 iOS 捷徑、Tasker 或其他自動化工具收集任務。

支援的 URL：

| URL | 動作 |
| --- | --- |
| `mindwtr://capture?title=Buy%20groceries` | 開啟帶有標題的收集畫面 |
| `mindwtr://capture?title=Buy%20groceries&note=From%20store` | 開啟帶有標題及筆記的收集畫面 |
| `mindwtr://capture?title=Buy%20groceries&project=Shopping&tags=errands,home` | 開啟帶有專案及標籤的收集畫面 |

注意事項：
- `title` 為必要欄位（別名：`text`）。
- `note` 為選用欄位（別名：`description`）。
- `project` 會以不區分大小寫的標題比對現有使用中專案，找不到時則建立。
- `tags` 以逗號分隔，儲存前正規化為 `#tag` 格式。

iOS 捷徑範例：
1. 開啟**捷徑**並建立捷徑。
2. 加入**詢問輸入內容**（提示：任務標題）。
3. 加入**打開 URL**：`mindwtr://capture?title=[Provided Input]`。
4. 執行捷徑；Mindwtr 會開啟收集畫面，供你檢查並儲存任務。

### 快速新增語法

Mindwtr 會在新增任務時解析自然語言：

| 語法              | 範例                         | 結果                                      |
| ----------------- | ---------------------------- | ----------------------------------------- |
| `@context`        | `@home`                      | 加入情境                                  |
| `#tag`            | `#focused`                   | 加入標籤                                  |
| `+Project`        | `+HomeReno`                  | 指派至專案                                |
| `+Multi Word`     | `+New Project`               | 指派至 "New Project"                      |
| `+"Quoted Name"`  | `+"New Project" call Bob`    | 引號界定句中多字名稱（領域也可用 `!"Area Name"`） |
| `!Area`           | `Plan roadmap !Work`         | 指派至領域                                |
| `%Person`         | `Ask %Jim for budget`        | 設定「指派給」（委派／等待對象）；新建多字名稱使用 `%"Full Name"` |
| `/area:<name>`    | `/area:Personal`             | 指派至領域（不能有空格）                  |
| `/due:date`       | `/due:friday`                | 設定截止日期                              |
| `/energy:<level>` | `/energy:medium`             | 設定精力程度（`low`、`medium`、`high`）   |
| `/note:text`      | `/note:call back`            | 加入描述                                  |
| `/status`         | `/next`, `/waiting`, `/someday`, `/done`, `/archived`, `/inbox` | 設定狀態 |

**日期格式：**today、tomorrow、friday、next week、in 3 days

## 音訊收集

使用 AI 轉錄，以語音收集任務。

### 設定

1. 前往**選單 → 設定 → AI 助理**。
2. 啟用**語音轉文字**。
3. 選擇**供應商**：
   - **OpenAI / Gemini**：雲端服務（需要 API 金鑰）。
   - **離線（Whisper）**：在本機執行。可直接在設定中下載模型（例如 Tiny 或 Base）。
4. 若偏好以音訊為先，請在**設定 → 一般**設定**預設收集方式**。

### 使用音訊收集

- **快速新增**：在「快速收集」畫面點選**音訊**分頁。
- **錄音**：點選麥克風開始。
- **轉錄**：停止錄音以處理音訊。
- **智慧解析**：啟用後，應用程式會自動擷取日期與欄位。

## 收集箱

供快速輸入任務的收集區域。

想透過工作、家庭、人物、跑腿與未完成事項等引導式提示收集，而不從空白輸入開始時，請使用**思緒清理**。

### 新增任務

1. 點選底部輸入欄
2. 從其他應用程式使用分享選單
3. 點選主畫面小工具
4. 使用快速新增語法輸入任務
5. 點選新增按鈕或按 Enter

### 處理收集箱

點選**處理收集箱**，開始釐清工作流程：

1. **可以採取行動嗎？**
   - 可以 → 繼續
   - 不行 → 垃圾或將來/也許

2. **能在 2 分鐘內完成嗎？**
   - 能 → 立刻去做，標示為「已完成」
   - 不能 → 繼續

3. **該由誰執行？**
   - 我來做 → 加入情境，移至下一步行動
   - 委派 → 移至等待中

4. **要在哪裡做？**
   - 選擇情境（@home、@work 等）
   - 加入自訂情境

5. **指派至專案？**（選用）
   - 選擇專案或略過
   - 如果收集項目包含多個行動，也可選擇**是，將它設為專案**：為專案命名並定義下一步行動。你輸入的任何後續行動都會在已連結專案的狀態下回到收集箱，準備各自進行釐清

## 專注

主要的執行儀表板。「專注」是執行儀表板，不是每項 `next` 狀態任務的完整清單。

### 區段

| 區段       | 內容                                                   |
| ---------- | ------------------------------------------------------ |
| **今日**   | 今日專注、今天到期／逾期或今天開始的任務              |
| **下一步** | 未受阻或延後的可用下一步行動                           |

「專注」會隱藏未來才開始的任務，以及循序專案中較後面的任務，讓清單只呈現目前可執行的事情。若想檢查更完整的任務清單，請使用**情境**、**專案**或**搜尋**。

預設下一步行動順序為：即將到期者優先、無日期行動其次、距離較遠的未來到期行動最後。同組內，「專注」依序使用已啟用的優先順序、開始時間、建立日期、標題與 id。完整邏輯請參閱[Mindwtr 的 GTD 工作流程](/zh-Hant/use/gtd-workflow#「專注」如何排序可用行動)。

### 功能

- **情境篩選條件**：點選情境方塊以篩選「下一步」清單。
- **已儲存篩選條件**：儲存可重複使用的專注條件，例如專案、情境、標籤、優先順序、精力程度與預估時間。
- **滑動以專注**：將任務向右滑以切換「專注」狀態（移至「今日」）。
- **快速狀態**：點選狀態徽章以變更狀態。
- **番茄鐘（選用）**：在**設定 → GTD → 功能 → 番茄鐘**中啟用，以顯示精簡的專注／休息計時器。可保留為**只使用計時器**，或開啟**將計時器連結至任務**，顯示「計時任務」選擇器與**將任務標示為已完成**動作。

## 回顧

回顧任務並更新狀態。

- 查看任務詳細資料（描述、開始時間、截止期限、情境）
- 迅速將任務標示為已完成
- 在任務之間導覽
- **選取模式**：批次選取並分享任務
- 每週回顧期間，先使用**處理收集箱**步驟清除收集項目，再繼續回顧行事曆、等待中、專案與將來項目。

## 任務編輯器（任務 + 檢視）

任務編輯器有兩種模式：

- **任務**：編輯欄位、檢查清單、日期、標籤與情境
- **檢視**：乾淨的唯讀摘要，並可點選檢查清單

向左／右滑動以切換**任務**與**檢視**。

以檢查清單為主的任務預設開啟「檢視」模式，方便快速勾選。

編輯器一開始保持精簡。點選**更多選項**顯示進階欄位；已有內容的欄位會保持可見。

描述 Markdown 支援無序清單與任務核取方塊（`- item`、`[ ] item`、`[x] item`）。
在任務描述或專案筆記輸入 `[[`，可從連結選擇面板連結另一項任務或專案。
這些連結只供導覽，不會同步任務之間的完成狀態。
描述中的 Markdown 核取方塊只屬於筆記，不會影響任務檢查清單。
將多行文字貼入檢查清單項目，會逐行建立一個項目（可辨識項目符號、編號與 `[x]` 標記）。
**受指派者／人員**欄位會儲存「等待中」、建議與 `assigned:` 搜尋所用的委派對象。可從**設定 → 管理**管理已儲存人員、筆記與參考連結。

重複任務支援兩種策略：
- **固定排程**（固定節奏）
- **完成後重複**（下一個日期從完成時間起算）
- **結束：永不／於日期／N 次後**

Mindwtr 只保留一個使用中的重複任務實例。有截止日期或開始時間時，「行事曆」會顯示目前實例；未來發生時間不會預先建立，直到目前任務完成。若啟用**在行事曆顯示下一次發生時間**，則會顯示僅供規劃的預覽。

使用任務編輯器中的重複欄位，再視需要切換**完成後重複**或**在行事曆顯示下一次發生時間**。
同一個設定面板也能讓系列在目標日期或固定總次數後停止。

### 附件

可從編輯器將檔案或連結附加至任務。啟用**儲存音訊附件**後，音訊筆記也能儲存為附件。

同步與清理詳情請參閱[附件](/zh-Hant/use/attachments)。

## 行事曆整合

Mindwtr 可疊加外部行事曆，並將有日期的 Mindwtr 任務推送至裝置行事曆。詳細設定請參閱[行事曆整合](/zh-Hant/use/calendar-integration)。

若要將任務推送至 Android/Google Calendar 或 iOS 上的 Apple 行事曆：

1. 前往**設定 → 行事曆**
2. 啟用**將任務推送至行事曆**
3. 授予行事曆權限
4. 展開**同步目標**
5. 選擇專用的 `Mindwtr` 行事曆或其他可寫入行事曆目標

Android Google Calendar 與 iOS Apple 行事曆目標的設定方式，請參閱[行事曆整合](/zh-Hant/use/calendar-integration)。

若要透過 ICS 訂閱疊加外部行事曆：

1. 前往**設定 → 行事曆**
2. 加入你的 **ICS URL**
3. 重新整理以擷取事件

外部事件只能檢視，不會同步回來源。點選外部事件並選擇**建立任務**，即可建立另一項 Mindwtr 任務；Mindwtr 會在可用時複製事件標題、日期／時間、地點、描述與行事曆名稱。

## 行事曆

具備排程功能、依時間顯示的檢視。

### 檢視

- **月檢視**：有截止日期任務的概覽
- **日檢視**：包含已排程任務與外部事件的詳細時間軸

### 排程任務

1. 在「行事曆」日檢視中點選**排程任務**
2. 從「下一步行動」（優先顯示）選取，或搜尋待辦任務
3. Mindwtr 會尋找最早的空閒時段（避開與外部事件的衝突）
4. 根據任務預估時間設定開始時間

排程面板是行動版的規劃介面：查看某一天時，可用它將尚未排程的下一步行動，或已到期但未排程的工作，轉成具體時間區塊。想專注查看行事曆格線時，可將它收合。

### 拖曳以重新排程

- 長按已排程任務區塊
- 拖至新時段（以 5 分鐘為單位貼齊）
- 放開以更新開始時間

### 外部行事曆（iCal/ICS）

訂閱外部行事曆，將事件與任務一起顯示：

1. 前往**設定 → 進階 → 行事曆**
2. 輸入行事曆 URL（ICS/webcal 格式）
3. 命名並點選**加入**
4. 外部事件會以灰色區塊顯示在日檢視

## 專案

管理需要多個步驟的成果。

從**選單 → 專案**開啟。

### 專案清單

- 查看所有使用中專案
- 查看各專案的任務數量
- 點選以查看專案詳細資料

### 專案詳細資料

- 查看專案中的所有任務
- 加入新任務
- 以**分區**在專案內為任務分組。分區是單一專案內的標題，不是子任務或獨立專案。
- 點選任務，在任務編輯器中指派**分區**
- 編輯專案設定（名稱、色彩、筆記）
- 指派**專注領域**（例如工作、個人）
- 加入**專案標籤**以供篩選
- 設定循序或平行模式
- 設定回顧日期
- 啟用自訂排序後，以拖曳把手重新排列專案任務。將任務放在其他分區標題下會移至該分區；放在第一個標題上方則會清除分區
- 將專案歸檔（這就是完成專案的方式：剩餘任務會一併完成；**重新啟用**可復原）

任務編輯器的**專案分區**欄位會將任務指派至目前專案中的其中一個分區。只有任務已屬於設有分區的專案時才有意義，否則請留空。

### 循序與平行

| 模式         | 行為                                             |
| ------------ | ------------------------------------------------ |
| **循序**     | 只有第一項可用專案任務會顯示在「專注」           |
| **平行**     | 所有可用專案任務都能顯示在「專注」               |

循序專案可依整個專案或逐分區執行。分區範圍的循序模式會顯示各專案分區中第一項可用任務，讓不同階段或工作流能平行推進，又不必顯示每一項任務。

## 滑動動作

以滑動手勢快速管理任務：

| 檢視         | 向右滑動 | 結果                   |
| ------------ | -------- | ---------------------- |
| **收集箱**   | 已完成   | 將任務標示為已完成     |
| **專注**     | 專注     | 切換專注狀態           |

完成任務時會記錄目前時間。如果實際完成得更早，可長按任何**已完成**控制項，選取真正完成時間：滑動後顯示的「已完成」按鈕、狀態選單中的「已完成」選項、任務編輯器中的「已完成」方塊，或任務預覽中的狀態徽章。之後也能點選已完成或已歸檔任務的**完成時間**加以修正。

## 通知與提醒

Mindwtr 會傳送通知，協助你掌握進度。

### 通知類型

- **截止日期提醒**：任務到期時通知
- **開始時間通知**：提醒該開始了
- **重複任務提醒**：重複項目的通知

點選通知本文可直接前往**回顧**畫面。

提醒在手機本身排程，不使用推播伺服器。如果在另一部裝置完成或重新排程任務（例如在桌面版勾選每日重複任務），變更同步至手機後，手機會立即更新鬧鐘；因此請在下一次提醒時間前讓手機同步（開啟應用程式一次即可）。

### 權限

確認已啟用通知：
1. 前往裝置的**設定 → 應用程式 → Mindwtr**
2. 啟用**通知**
3. 視需要允許提示與音效

## 設定

### 一般

- **外觀**：跟隨系統、淺色或深色
- **語言**：英文、簡體中文、繁體中文、印地文、西班牙文、阿拉伯文、法文、葡萄牙文、俄文、德文、日文、越南文、土耳其文、韓文、義大利文、波蘭文、荷蘭文、捷克文

### 通知

**任務提醒：**
- 啟用／停用截止日期與開始時間的通知

**每日摘要：**
- **早間簡報**：今日到期、逾期與專注任務摘要
- **晚間回顧**：提醒回顧並結束一天
- 分別設定時間

**每週回顧：**
- **提醒**：每週收到開始回顧的通知
- **時間／日期**：自訂回顧時間（例如星期五下午 4 點）

### GTD

自訂 Mindwtr 如何配合你的 GTD 工作流程：

**功能（選用）：**
- **優先順序**：在任務上顯示優先順序旗標
- **預估時間**：加入供時間區塊使用的持續時間欄位

**預估時間預設：**
- 選擇任務編輯器中顯示的預估時間
- 選項：5m、10m、15m、30m、1h、2h、3h、4h、4h+
- 預設：10m、30m、1h、2h、3h、4h、4h+

**自動歸檔：**
- 在設定天數後自動將已完成任務移至已歸檔（預設 7 天）
- 設為「永不」，可將完成項目永久保留在「已完成」清單

**收集箱處理：**
- 行動版保留卡片式收集箱處理流程
- 相同的共用設定可隱藏或顯示 2 分鐘捷徑、專案優先提示、情境／標籤區段、排程區段與參考資料選項

**任務編輯器版面：**
- 點選欄位以切換可見性（已有值的隱藏欄位仍會顯示）
- 長按拖曳把手以重新排序欄位
- 在**基本**、**排程**、**整理**與**詳細資料**等區段之間移動欄位
- 選擇預設開啟的可收合區段
- 隱藏欄位可透過編輯器的**更多**按鈕顯示

**管理：**
- 使用**設定 → 管理**編輯已儲存的**領域**、**人員**、**情境**與**標籤**
- 這是清理重複項目或重新命名可重複使用中繼資料最快的位置
- 指派時建立領域與人員的位置，請參閱[領域與人員](/zh-Hant/use/areas-people)

### 資料與同步

同步設定請參閱[資料與同步](/zh-Hant/data-sync/)。

**同步後端：**
- **雲端同步**：支援組建中的 Dropbox，以及可用時 iOS 上的 iCloud
- **資料夾／檔案同步**：透過共用 JSON 檔案／資料夾進行檔案同步（Google Drive、Syncthing、OneDrive 等）
- **進階／自訂伺服器**：WebDAV 或自行託管 Mindwtr Cloud

**其他選項：**
- **同步**：手動觸發同步
- **上次同步狀態**：查看資料上次同步時間
- **同步歷史**：預設收合；點選可展開最近項目
- **匯出備份**：將資料儲存至檔案
- **Apple 提醒事項匯入**：選擇「提醒事項」清單，將未完成提醒事項匯入收集箱。匯入項目會保留在 Apple 提醒事項中；已匯入、已完成或無標題的提醒事項會略過。
- **設定同步選項**：選擇跨裝置同步的偏好（主題、語言／日期格式、GTD 預設、外部行事曆 URL、AI 設定與已儲存篩選條件）。API 金鑰與本機模型路徑永遠不會同步。

**GTD 選項：**
- **專注任務上限**：選擇可標示為今日專注的任務數量。

### 進階

**AI 助理：**
- 選用的 BYOK 助理，可釐清及拆分任務

**行事曆（ICS/iCal）：**
- **加入行事曆**：輸入名稱與 URL
- **啟用／停用**：切換各行事曆的可見性
- **移除**：刪除訂閱
- **測試**：確認行事曆能正確載入
