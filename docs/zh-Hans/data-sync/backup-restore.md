# 备份与恢复

如水将工作数据保存在本地，并允许你导出 JSON 备份，以便携带、修复和迁移。

恢复被设计为**替换本地数据**的流程：

- 你选择一个备份 JSON 文件
- 如水在更改任何内容前先验证文件
- 如有可能，如水会先创建恢复快照
- 所选备份替换当前本地数据集

这样能让恢复过程简单且可预期。它不是合并操作。

---

## 导出备份

### 桌面端

1. 打开**设置 → 数据**
2. 在**数据传输**中选择**导出备份**
3. 将 JSON 文件保存到所需位置

### 移动端

1. 打开**设置 → 数据**
2. 点按**导出备份**
3. 保存或分享 JSON 文件

备份格式与如水内部 `data.json` 结构兼容。

---

## 从备份恢复

### 桌面端

1. 打开**设置 → 数据**
2. 在**数据传输**中选择**恢复备份**
3. 选择如水备份 JSON 文件
4. 检查摘要并确认恢复

如果 Tauri 运行时可用，桌面端会在恢复前于本地快照目录中创建数据快照。

### 移动端

1. 打开**设置 → 数据**
2. 点按**恢复备份**
3. 选择如水备份 JSON 文件
4. 检查摘要并确认恢复

移动端会在恢复前于应用存储中保存本地恢复快照。

---

## 恢复快照

如水会在恢复和导入操作前自动创建恢复快照。

- **桌面端：**快照显示在**设置 → 同步 → 恢复快照**中
- **移动端：**快照显示在**设置 → 同步 → 恢复快照**中

如果恢复了错误文件，或想撤销本地导入/恢复操作，可使用这些快照。

---

## 验证规则

如水会在恢复前验证所选 JSON 文件：

- 文件必须是有效 JSON
- 必须符合如水的数据结构
- 可用时会显示项目数量和备份元数据
- 版本不匹配会产生警告，而不是无提示失败

验证失败时，恢复会被阻止，当前数据保持不变。

---

## 恢复不会执行的操作

- **不会**把备份与当前本地数据合并
- **不会**只恢复一项任务或一个项目
- 在下一次同步周期前，**不会**自行覆盖远程同步服务

如果使用同步，请把恢复理解为先替换当前本地状态。之后的同步行为取决于所用后端，以及下一台执行同步的设备。

---

## 提示

- 除同步外，也请定期手动导出
- 只从你信任的备份恢复
- 如果使用文件同步，请等正确的 `data.json` 完成复制后，再同步另一台设备

另请参阅[数据与同步](/zh-Hans/data-sync/)。
