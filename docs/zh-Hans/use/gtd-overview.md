# GTD 概览

Getting Things Done（GTD）是 David Allen 创立的个人效率方法，详见他的著作 *Getting Things Done: The Art of Stress-Free Productivity*。

---

## 心如止水

GTD 的核心理念是达到一种**头脑清明**的状态，让大脑不再承担记住所有待办事项的负担。

> *“你的头脑是用来产生想法的，而不是保存想法的。”* — David Allen

当大脑相信所有事情都已记录在可靠系统中，就能把注意力用于真正有创意且高效地完成工作。

---

## 五个步骤

GTD 由五个相互关联的步骤组成：

### 1. 记录

把脑中的一切移到可信赖的系统中。

- 写下每项任务、想法、承诺或提醒
- 暂时不要筛选或整理，先记录下来
- 使用**收集箱**作为入口

### 2. 明确

处理每项记录，决定它究竟意味着什么。

逐项询问：

- **这是什么？**
- **可以采取行动吗？**
  - 不能 → 垃圾桶、将来/也许或参考
  - 能 → 期望的结果是什么？下一步行动是什么？

### 3. 整理

把事项放到正确位置：

| 去向 | 使用时机 |
| --- | --- |
| **下一步行动** | 你将亲自完成的可执行任务 |
| **项目** | 需要多个行动才能实现的成果 |
| **等待中** | 已委派给其他人 |
| **日历** | 必须在特定时间发生 |
| **将来/也许** | 留给未来的想法 |
| **参考** | 需要保留的信息（笔记） |
| **垃圾桶** | 不再需要 |

### 4. 回顾

定期检查你的系统。

- **每日回顾：**浏览清单、日历和收集箱
- **每周回顾：**全面检查所有清单，清空收集箱，更新项目

### 5. 执行

自信地采取行动。

根据以下条件选择当前工作：

- **情境：**你在哪里？手头有什么工具？
- **时间：**有多少时间？
- **精力：**当前身心状态如何？
- **优先级：**什么最重要？

## 关键概念

### 下一步行动

下一步行动是推动事情前进的**下一项看得见、可实际执行的活动**。

❌ 不好：“推进项目”
✓ 较好：“打开项目文件夹并检查状态备注”

### 项目

项目是任何需要不止一个行动才能实现的成果。它是对结果的承诺，而不只是一份待办清单。

### 项目分区

项目分区是单个项目内带标签的分组。它们让较长的项目任务列表保持易读，而不会变成嵌套项目或子任务。

例如，名为**发布网站**的项目可以包含**设计**、**开发**和**内容**分区。项目仍只有一个成果；分区只是项目内任务的标题。

任务上的**项目分区**字段会把任务分配到当前项目的某个分区。只有当任务已经属于一个包含分区的项目时，该字段才有意义。任务没有项目，或项目没有分区时，请留空。

### 领域、项目与任务

如水采用灵活的“容器”模型，让你可以顺畅整理：

- **领域**是持续的责任（例如工作、家庭、健康）。
- **项目**是有终点的成果，可以位于某个领域内，也可以独立存在。
- **任务**可以位于项目内、直接位于领域内，也可以完全未分配（收集箱/无领域）。

这样既能以领域承载长期责任，又能快速记录任务，而不必强迫自己先选择文件夹。

经验法则：

- **领域**回答：“这项责任与我生活或工作的哪一部分有关？”
- **情境**回答：“考虑到我现在的位置和手边条件，我能做什么？”

例如，如果你是为多个客户工作的承包商，可以为每个客户建立一个**领域**。同一任务可以同时属于**领域：客户 A**、**项目：网站改版**和**情境：@computer**。

### 2 分钟规则

如果一项行动不到 2 分钟就能完成，请在处理时**立即完成**。

### 情境

表示可以在何处或使用何种条件完成事情的标签：

- `@home`、`@work`、`@errands`
- `@computer`、`@phone`
- `#focused`、`#lowenergy`

另请参阅：[情境与标签](/zh-Hans/use/contexts-tags)

## GTD 为何有效

1. **外部可信系统：**大脑不再费力记忆
2. **明确的下一步行动：**该做什么不再含糊
3. **定期回顾：**系统始终保持最新、值得信赖
4. **基于情境的清单：**只看到当下相关的内容

---

## 学习 GTD

- **6 个月：**不再觉得系统用起来笨拙
- **1-2 年：**真正内化这些原则
- **持续学习：**每次重读都会获得新见解
