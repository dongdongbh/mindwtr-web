# Tâches récurrentes

Utilisez les tâches récurrentes pour les activités qui doivent revenir après leur achèvement : revues, corvées, routines, renouvellements et entretien.

La récurrence se règle dans l’éditeur de tâche. Si vous utilisez la capture rapide, créez d’abord la tâche, puis ouvrez-la pour la modifier.

## Configurer une tâche récurrente

1. Ouvrez la tâche.
2. Modifiez-la.
3. Repérez le champ **Récurrence** dans la section de planification.
4. Choisissez **Quotidienne**, **Hebdomadaire**, **Mensuelle** ou **Annuelle**.
5. Ajustez les détails de la répétition.
6. Enregistrez la tâche.

Les commandes de récurrence apparaissent dans l’éditeur sur ordinateur et mobile.

## Options de répétition

| Option | Utilisation |
| --- | --- |
| **Quotidienne** | Habitudes et routines quotidiennes. |
| **Hebdomadaire** | Revues hebdomadaires, réunions, courses ou entretien. |
| **Mensuelle** | Factures, revues mensuelles, abonnements ou listes mensuelles. |
| **Annuelle** | Renouvellements, impôts, inscriptions et anniversaires. |

Chaque type propose **Répéter tous les** pour sauter des intervalles, par exemple toutes les 2 semaines ou tous les 3 mois.

Les tâches hebdomadaires peuvent cibler des jours précis. Les tâches mensuelles peuvent revenir le même jour de chaque mois ou employer **Personnalisée** pour des motifs comme un jour précis de la semaine.

## Calendrier fixe ou répétition après achèvement

Par défaut, Mindwtr conserve un calendrier fixe. Utilisez-le pour les échéances et engagements qui doivent rester liés au calendrier.

Exemples :

- Payer le loyer tous les mois.
- Vérifier les finances le premier samedi.
- Renouveler l’assurance chaque année.

Activez **Répéter après achèvement** lorsque la prochaine tâche doit dépendre de la date à laquelle vous terminez l’actuelle.

Exemples :

- Arroser les plantes 3 jours après le dernier arrosage.
- Nettoyer les filtres 2 semaines après leur nettoyage.
- Relancer 1 mois après la relance précédente.

Lorsque cette répétition choisit aussi des jours de semaine ou un jour du mois, la prochaine tâche tombe le premier jour correspondant de la dernière semaine ou du dernier mois de l’intervalle. Si vous terminez un mardi une tâche « toutes les 2 semaines le dimanche », elle revient le dimanche de la semaine suivante, pas le dimanche immédiatement à venir.

Si vous avez terminé une tâche plus tôt mais ne la marquez comme terminée que plus tard, renseignez plutôt l’heure réelle : faites un clic droit sur le bouton de fin sur ordinateur ou un appui long sur **Terminé** dans le menu de statut mobile. La prochaine occurrence sera planifiée depuis l’heure d’achèvement réelle.

## Fin d’une récurrence

Utilisez **Se termine** pour arrêter la série :

| Réglage | Signification |
| --- | --- |
| **Jamais** | Continue à créer la tâche suivante après chaque achèvement. |
| **À une date** | S’arrête lorsque la prochaine occurrence dépasserait la date choisie. |
| **Après** | S’arrête après le nombre d’occurrences choisi. |

## Aperçu dans le calendrier

Mindwtr conserve une seule tâche active par série récurrente. La prochaine tâche réelle est créée lorsque vous terminez la tâche actuelle.

Activez **Afficher la prochaine occurrence dans le calendrier** pour voir un aperçu de planification. Cet aperçu facilite l’anticipation, mais ne constitue pas une seconde tâche active.

## Effet de l’achèvement

Lorsque vous terminez une tâche récurrente, Mindwtr crée la suivante avec le même titre, projet, domaine, contextes, tags, notes, rappels et autres détails. Les éléments de liste de contrôle sont réinitialisés.

Si la tâche possède une échéance, une date de début ou une date de revue, Mindwtr les avance selon la règle de récurrence. Les valeurs sans heure restent sans heure.

La tâche suivante reste hors de Focus et des Prochaines actions jusqu’à sa date. Une date de début future la reporte comme d’habitude ; sans date de début, une tâche récurrente est reportée jusqu’à sa prochaine date, la plus proche entre son échéance et sa date de revue. La nouvelle occurrence ne réapparaît donc pas immédiatement après l’achèvement de la précédente. Les occurrences reportées sont tout de même comptées dans l’indication « masquées (début futur) » ; utilisez **Afficher**, Projets ou Recherche pour les retrouver.
