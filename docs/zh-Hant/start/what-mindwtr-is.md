# Mindwtr 是什麼

Mindwtr 依照 David Allen 的 Getting Things Done 方法，管理你個人生活中的承諾與行動。本頁說明產品的核心，幫助你判斷它是否適合自己的做事方式。

## 原生支援 GTD

收集箱、專案、情境、等待中、將來/也許與回顧，都是 Mindwtr 的一等功能。它們是應用程式的設計基礎，而不是硬套在一般任務清單上的標籤。當你收集一件事、釐清它，再將它整理歸位，就是直接在執行 GTD 循環。[GTD 概覽](/zh-Hant/use/gtd-overview)會介紹這套方法，[GTD 工作流程](/zh-Hant/use/gtd-workflow)則說明每一步如何對應到應用程式畫面。

## 筆記為行動提供所需資料

你可以將筆記與檔案附加到任務及專案，讓執行工作所需的背景資料就在一旁。Mindwtr 的範圍止於此：它不是知識庫，也無意成為知識庫。參考資料需要真正的歸宿時，請交給專為此用途打造的工具。[Obsidian 整合](/zh-Hant/power-users/obsidian)正是為此而設：筆記留在你的 vault 中，Mindwtr 只從中讀取任務。

## 為個人而設，不是團隊工具

Mindwtr 只承載一個人的承諾：你的承諾。它沒有共享工作區、指派給隊友的任務、留言串或活動動態。「等待中」用來記錄別人尚未交付給你的事項，是個人紀錄，而非協作管道。

## 預設簡單，需要時依然強大

日常循環就是收集、釐清與回顧；完成這些步驟所需的一切都無須設定即可看見。較進階的功能（AI、本機 API、MCP 伺服器、看板、番茄鐘）會保持低調，直到你主動啟用；啟用後也不會讓基本流程變得複雜。專案 README 說得很直接：

> 我只是想騎腳踏車，別先給我一座駕駛艙。

## 能自動完成，就不要求手動設定

只要存在合理的預設值，Mindwtr 就會直接採用，而不把決定丟給你。它會依循平台慣例與安裝來源，不為每個選項再增加一項設定。開關越少，需要維護及可能設錯的地方也越少。
