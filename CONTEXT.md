# Domain glossary — mindwtr-web

This repo hosts Mindwtr's public web surfaces (not the app). Terms used in
code, commits, and design notes:

- **Landing** — the static marketing site at `mindwtr.app`, built by Vite from
  flat `landing/*.html` pages. No client framework; `landing/src/main.ts` adds
  progressive enhancement only (platform detection, copy buttons, coverflow).
- **Docs site** — the VitePress site at `docs.mindwtr.app` under `docs/`.
- **Page chrome** — the head boilerplate, header, and footer every landing
  page shares. Injected at build/serve time by `landing/chrome.ts`; pages
  carry only their content and per-page head data (title, description,
  og:title, og:description). Canonical URL, og:url, and nav `aria-current`
  are derived from the filename — never written by hand.
- **Page registry** — the list of landing pages, derived by globbing
  `landing/*.html` in `landing/vite.config.ts`. Rollup inputs and the set of
  indexable `sitemap.xml` URLs both come from it: adding a page means creating
  one file. Non-indexable utility pages such as `404.html` are emitted but
  explicitly excluded from the sitemap.
- **Dist verifier** — `scripts/check-dist.mjs`, the depth behind
  `bun run check`. It reads the built output of both sites (the same files
  Cloudflare serves) and checks internal/cross-site links, fragment targets,
  metadata and schema consistency, canonical/og:url correctness, social and
  priority images, optional `llms.txt` links, 404 behavior, and sitemap parity.
- **Shared assets** — `shared-assets/` is the only committed copy of brand
  assets, screenshots, and badges. `landing/public/assets/` and
  `docs/public/assets/` are generated by `scripts/sync-assets.mjs` (run
  automatically by every dev/build script) and are gitignored — edit
  `shared-assets/` only.
