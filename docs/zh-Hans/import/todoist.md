# 从 Todoist 导入

Mindwtr 可以导入 Todoist 导出文件，让你无需手动重建整个系统即可迁移。

支持的来源：

- 单个 Todoist **CSV** 导出
- 包含多个项目 CSV 文件的 Todoist **ZIP** 备份

桌面端和移动端均可从**设置 → 数据 → 从 Todoist 导入**执行导入。

---

## 导入内容

Mindwtr 以 GTD 优先的方式将 Todoist 导出映射到自身模型：

- Todoist 项目成为 **Mindwtr 中的项目**
- Todoist 分区成为 **Mindwtr 中的分区**
- Todoist 子任务成为**检查清单项目**
- Todoist 标签成为**标签**
- 已分配到项目的活跃任务成为 Mindwtr 对应项目中的**下一步行动**
- 未分配的活跃任务保留，供你在**收集箱**中处理

Mindwtr 不会把已经整理好的项目任务全部倒入收集箱，同时仍让零散任务可以按 GTD 流程处理。

---

## 支持的 Todoist 数据

- 任务标题
- 描述
- 优先级
- 能够安全解析的截止日期
- 分区
- 附加到任务的备注/评论
- 写在 Todoist 内容中的标签（例如 `@work`）

Todoist 的重复计划不会自动重建为 Mindwtr 中的重复规则。任务只导入一次，并在描述中保留原始 Todoist 重复文字，由你决定如何在 Mindwtr 中表示。

---

## 导入流程

1. 打开**从 Todoist 导入**
2. 选择 Todoist CSV 或 ZIP 文件
3. 检查预览摘要
4. 确认导入

如果平台支持，导入前 Mindwtr 会保存当前本地数据的恢复快照。

导入后：

- 按需创建新项目
- 已分配到项目的活跃任务作为**下一步行动**出现在相应项目中
- 未分配的活跃任务保留，供你在**收集箱**中处理
- 对重复任务、跳过的行或不受支持的归档条目显示警告

---

## ZIP 备份说明

Todoist ZIP 备份通常每个项目包含一个 CSV。Mindwtr 会读取每个 CSV 并分别导入各项目。

Mindwtr 会跳过：

- 嵌套的 ZIP 文件
- 归档中的非 CSV 文件
- 无法安全解析的格式错误 Todoist 行

---

## 提示

- 如果想先测试映射，可从较小的 Todoist 项目开始
- 验证导入结果前请保留恢复快照
- 两次导入同一导出可能产生重复任务

另请参阅[数据与同步](/zh-Hans/data-sync/)以及[备份与恢复](/zh-Hans/data-sync/backup-restore)。
